What does machine learning mean in biostatistics and medical research?

What does machine learning mean in biostatistics and medical research?

In this course, machine learning is not treated as magic.

It is treated as a set of statistical and computational tools that help us learn from data and make predictions.

In medical research, we may want to predict:

  • whether a patient has a disease;
  • whether a patient will be readmitted to hospital;
  • whether cancer may recur;
  • whether a patient may respond to treatment;
  • whether a patient is at high risk of clinical deterioration;
  • survival probability over time.

The goal is not just to build a model.

The goal is to build a model that is valid, interpretable, clinically meaningful, and useful on new patients.

Intuition first

Machine learning as structured pattern learning

Imagine a clinician seeing many patients over time.

The clinician starts noticing patterns:

  • patients with higher glucose often have higher diabetes risk;
  • patients with advanced tumour stage may have worse prognosis;
  • patients with repeated hospital admissions may have higher readmission risk;
  • some biomarkers may be linked with treatment response.

Machine learning tries to learn such patterns from data in a structured way.

But there is one important rule:

A model is only useful if it performs well on new patients, not just on the patients used to build it.

Medical machine learning

What makes medical machine learning different?

General machine learning often focuses on predictive performance.

Medical machine learning also cares about:

  • patient safety;
  • clinical interpretation;
  • calibration;
  • fairness;
  • generalisability;
  • reproducibility;
  • transparent reporting;
  • external validation;
  • decision consequences.

A model can perform well in one dataset but fail in another hospital, population, time period, or healthcare setting.

This is why a medical prediction model should not be judged only by whether the algorithm is advanced.

It should be judged by whether it is valid and useful in the intended setting.

Common mistake: thinking complex models are automatically better

A complex model is not automatically better than a simpler model.

For example, a random forest or boosting model may perform well, but if it is poorly validated or difficult to interpret, it may not be useful clinically.

A logistic regression model with good validation, good calibration, and clear interpretation may be more useful than a black-box model with weak reporting.

Detailed notes

Note 1: Prediction vs Inference

This note develops the central distinction between prediction, inference, association and causation in medical machine learning. The aim is to keep the statistical purpose clear before choosing an algorithm.

Learning objectives

By the end of this note, you should be able to:

Explain what machine learning means from a statistical learning perspective.

Describe why machine learning has become important in biostatistics, epidemiology, clinical prediction, and biomedical data science.

Distinguish clearly between prediction, inference, association, and causal inference.

Explain why a model that estimates an association well may not necessarily predict well.

Explain why a model that predicts well may not necessarily support causal or mechanistic interpretation.

Understand why medical machine learning must be evaluated using validation, calibration, discrimination, clinical usefulness, and transparent reporting.

Recognise common ways in which prediction and inference are confused in medical research.

1. Why start with prediction vs inference?

Machine learning is often introduced through algorithms: linear regression, logistic regression, decision trees, random forests, support vector machines, neural networks, and boosting. Although algorithms are important, they are not the best starting point for biostatistics. In medical statistics, the first question should not be “which algorithm should I use?” The first question should be “what scientific or clinical question am I trying to answer?”

This is why the distinction between prediction and inference is foundational. In biostatistics, the same dataset can often be used to answer very different questions. For example, a dataset containing smoking status, age, sex, lung cancer diagnosis, and other clinical variables could be used to ask whether smoking is associated with lung cancer. It could also be used to build a model that predicts an individual patient’s future risk of lung cancer. These two analyses may use similar variables and even similar statistical models, but the aims, modelling choices, evaluation criteria, and interpretation are different.

James et al. describe statistical learning as a set of tools for modelling and understanding complex datasets (James et al., 2023). That definition is deliberately broad. Some statistical learning methods are used to understand relationships between variables, while others are used primarily to predict outcomes. In medical research, both aims are valuable, but they should not be mixed without careful thought.

A major source of poor medical machine learning is not the use of the wrong algorithm. It is the use of the wrong framework: treating a prediction problem like an inference problem, or treating an inference problem like a prediction problem.

2. What is machine learning in biostatistics?

Machine learning can be described as a collection of statistical and computational methods that learn patterns from data and use those patterns for prediction, classification, clustering, or decision support. In a biostatistical context, machine learning usually means using patient-level, population-level, or biomedical data to predict clinically or scientifically meaningful outcomes.

Examples include predicting:

whether a patient has a disease;

whether a patient will be readmitted to hospital;

whether a tumour will recur;

whether a patient will respond to treatment;

whether an individual is at high risk of cardiovascular disease;

a patient’s survival probability at 1, 3, or 5 years;

disease status from gene expression or DNA methylation data.

The general supervised learning setup can be written as:

Y = f(X) + ε

Here, Y is the outcome, X represents predictors or features, f(X) is the unknown relationship between predictors and outcome, and ε represents random variation or unexplained error. In medical applications, Y might be disease status, mortality, relapse, response to treatment, or survival time. Predictors X might include age, sex, blood pressure, tumour stage, laboratory values, comorbidities, clinical history, imaging features, or molecular markers.

This mathematical form looks simple, but it hides several important medical questions. What exactly is the outcome? When is the prediction being made? Are all predictors available at that time? Is the target population clearly defined? Is the model intended for diagnosis, prognosis, screening, monitoring, or treatment decision-making? These questions matter because a model that looks statistically impressive can still be clinically inappropriate.

Steyerberg emphasises that prediction models in medicine require careful development, validation, and updating if they are to support clinical decision-making (Steyerberg, 2019). Similarly, TRIPOD+AI provides reporting guidance for clinical prediction models using either regression or machine learning methods, reflecting the idea that machine learning models should be held to transparent reporting standards (Collins et al., 2024).

3. Why machine learning matters in modern biostatistics

Biostatistics has traditionally been strongly associated with study design, estimation, hypothesis testing, regression modelling, survival analysis, causal inference, and uncertainty quantification. These areas remain central. Machine learning does not replace them. Instead, it adds tools for handling prediction, complex patterns, high-dimensional predictors, nonlinearities, interactions, and large-scale biomedical data.

Medical datasets are becoming more complex. A modern health data scientist may work with electronic health records, national registry data, clinical trial datasets, imaging-derived features, genomic data, transcriptomic data, methylation arrays, proteomics, metabolomics, wearable-device data, or linked administrative datasets. Some of these datasets contain thousands or millions of variables. Others contain repeated measurements over time. Some include rare outcomes, missing data, censoring, or measurement differences across hospitals.

Classical regression models remain extremely useful for many of these problems. Logistic regression, Cox regression, Poisson regression, mixed models, and generalised linear models are not outdated. In fact, they are often the most appropriate starting point. However, machine learning methods such as penalised regression, random forests, gradient boosting, support vector machines, and neural networks can sometimes capture structures that simpler models miss.

The central challenge in medical machine learning is that better flexibility can come with higher risk. Flexible models may fit nonlinear relationships and interactions, but they may also overfit, exploit leakage, learn site-specific artefacts, or produce poorly calibrated risk estimates. Therefore, the goal is not simply to use the most advanced algorithm. The goal is to build a model that answers a clearly defined medical question and performs reliably in the target setting.

4. Machine learning in medicine is not just algorithm fitting

A common beginner approach is to load a dataset, try many models, compare accuracy, and choose the model with the highest number. This approach may be acceptable for a simple classroom demonstration, but it is not sufficient for medical statistics.

A medical prediction model should be developed around a clinical question. Consider the difference between these two statements:

We trained an XGBoost model and achieved 91% accuracy.

and

We developed and internally validated a model to predict 30-day hospital readmission at the time of discharge, using predictors available before discharge, and evaluated discrimination, calibration, and potential clinical utility.

The second statement is much stronger because it defines the prediction setting. It tells us when the prediction is made, what outcome is being predicted, what type of validation was performed, and which aspects of performance were assessed. TRIPOD+AI emphasises transparent reporting of the data source, study design, predictors, outcome, model development, validation, performance, and intended use (Collins et al., 2024). PROBAST+AI similarly focuses on risk of bias and applicability in prediction model studies using regression or artificial intelligence methods (Moons et al., 2025).

A model in medicine should be assessed using questions such as:

  • What is the target population?
  • What decision or clinical process might the model support?
  • What is the outcome, and how is it measured?
  • What is the prediction time point?

Are the predictors available at the prediction time?

Was missing data handled appropriately?

Was the model internally validated?

Was it externally validated?

Does it show good discrimination?

Does it show good calibration?

Could it improve decisions at clinically meaningful thresholds?

  • Is there risk of bias?
  • Is it applicable to the intended setting?

These questions show why machine learning in biostatistics is best viewed as clinical prediction modelling plus responsible statistical learning, not as algorithm competition.

5. What is inference?

Inference is concerned with learning about relationships between variables. In medical statistics, inference often asks whether an exposure, treatment, biomarker, or patient characteristic is associated with an outcome. The aim may be explanatory, etiological, prognostic, or causal.

Examples of inference questions include:

  • Is smoking associated with lung cancer?
  • Is maternal smoking during pregnancy associated with offspring DNA methylation?
  • Is tumour stage independently associated with survival?
  • Is higher BMI associated with diabetes after adjustment for age and sex?

Does a treatment reduce mortality compared with standard care?

  • Is a biomarker associated with disease progression?

In these questions, we usually care about effect estimates, confidence intervals, uncertainty, confounding, model assumptions, and interpretation. If the question is causal, we also care about study design, exchangeability, temporality, selection bias, measurement error, and whether the adjustment set is appropriate.

For example, suppose we fit a logistic regression model to estimate the association between smoking and lung cancer. The coefficient for smoking may be transformed into an odds ratio. We might interpret this odds ratio as the adjusted association between smoking and lung cancer, conditional on the covariates in the model. The quality of this analysis depends on whether confounders were appropriately measured and adjusted for, whether the model form is suitable, and whether the study design supports the intended interpretation.

In inference, the model is often a tool for estimating a parameter or relationship. Prediction performance may still matter, but it is not the primary target.

6. What is prediction?

Prediction is concerned with estimating an outcome for new individuals. In prediction, the main question is not “what is the effect of this exposure?” but “how accurately and reliably can we estimate this person’s outcome or risk?”

Examples of prediction questions include:

  • What is this patient’s 10-year risk of cardiovascular disease?
  • What is the probability that this patient will be readmitted within 30 days?
  • What is this cancer patient’s predicted 5-year survival probability?
  • Which ICU patients are at high risk of death during admission?
  • Can baseline biomarkers predict response to therapy?
  • Can methylation profiles classify disease status?

In prediction, the focus shifts to performance in new data. We care about discrimination, calibration, sensitivity, specificity, positive predictive value, negative predictive value, clinical usefulness, and validation. A variable may be useful for prediction even if it is not causal. For example, previous hospital visits may help predict mortality because they summarise illness burden or healthcare use, even if hospital visits themselves are not a biological cause of death.

This distinction is very important. A prediction model may include variables that are good predictors but poor intervention targets. Conversely, a causal factor may not improve prediction much if other variables already capture similar information. For example, smoking may be causal for lung cancer, but in a short-term mortality prediction model among patients already diagnosed with advanced cancer, tumour stage and performance status may dominate prediction.

7. Prediction, association, and causation are not the same

Medical students and early researchers often use phrases such as “risk factor,” “predictor,” “associated factor,” and “cause” as if they mean the same thing. They do not.

A predictor is a variable that helps forecast an outcome. A risk factor is often used more loosely, sometimes meaning a predictor and sometimes implying a causal factor. An associated factor is statistically related to the outcome. A cause is a factor whose intervention would change the outcome under an appropriate causal framework.

Consider the variable “use of oxygen therapy” in hospital data. It may strongly predict mortality. Patients receiving oxygen are often sicker, so oxygen use may be associated with death. But that does not mean oxygen therapy causes death. It may be a marker of severity. If a prediction model includes oxygen use to estimate mortality risk at a given time point, that may be reasonable if oxygen use is available then. But it would be wrong to interpret the coefficient as evidence that oxygen therapy harms patients without considering confounding by indication.

Similarly, in an EWAS, a CpG site may be associated with an exposure or disease outcome. That does not automatically mean the methylation difference causes the disease. It may be a consequence of disease, a marker of cell composition, a technical artefact, or a correlate of another exposure.

Prediction answers “who is likely to have the outcome?” Causal inference answers “what would happen if we intervened?” These are different scientific questions.

9. Example: breast cancer prognosis

Suppose we have a dataset of breast cancer patients with age, tumour size, grade, lymph node status, receptor status, treatment information, comorbidities, follow-up time, and death or recurrence outcomes.

An inference question might be:

  • Is lymph node status independently associated with mortality after adjusting for age, tumour size, grade, receptor status, and treatment?

This question focuses on an association. We may use a Cox proportional hazards model and report a hazard ratio with a confidence interval. We would discuss adjustment, model assumptions, proportional hazards, missing data, and possible confounding.

A prediction question might be:

  • Can we predict a newly diagnosed patient’s 5-year survival probability using baseline clinical and tumour characteristics?

This question focuses on predicting individual risk. We may compare Cox regression, penalised Cox regression, random survival forests, or survival boosting. We would evaluate discrimination using the C-index or time-dependent AUC, calibration at 5 years, Brier score, and performance in internal or external validation.

Both analyses may use similar variables, but they are not the same analysis. In the inference analysis, the hazard ratio for lymph node status is central. In the prediction analysis, the overall risk estimate for the patient is central.

10. Example: EWAS and DNA methylation

In an epigenome-wide association study, the distinction between prediction and inference becomes especially important.

An inference-style EWAS question might be:

  • Which DNA methylation sites are associated with maternal smoking during pregnancy?

Here, the focus is on identifying CpG sites associated with an exposure. The analysis will likely involve regression models at many CpG sites, covariate adjustment, multiple testing correction, and careful interpretation of biological plausibility. The result may be a list of CpG sites with effect estimates and p-values.

A prediction question might be:

  • Can DNA methylation profiles predict whether an individual was exposed to maternal smoking during pregnancy?

Here, the focus is not on interpreting each CpG site as an exposure-associated locus. The focus is on classification performance in new individuals. The analysis may involve penalised regression, elastic net, random forests, or other machine learning methods. Evaluation should use nested cross-validation or external validation because the number of CpG sites is typically much larger than the number of samples.

This distinction matters because feature selection in high-dimensional data can easily create overfitting or leakage if performed before validation. A list of “important” CpGs from a predictive model should not automatically be interpreted as causal or mechanistic.

11. Training error and test error

A prediction model should not be judged only by how well it performs on the data used to train it. Training performance is often optimistic because the model has already seen the data. The real question is whether the model performs well for new patients.

Training error is the error measured in the same dataset used to fit the model. Test error is the error measured in data that were not used for fitting. James et al. emphasise that statistical learning is often concerned with estimating performance on unseen test data rather than only describing the training data (James et al., 2023).

Consider this example:

Training AUC = 0.95 Internal validation AUC = 0.78 External validation AUC = 0.62

The training AUC looks excellent. However, the external validation AUC suggests that the model performs only modestly in a new setting. This difference may reflect overfitting, differences between populations, changes in measurement practices, leakage during development, or a model that learned dataset-specific patterns rather than generalisable clinical signal.

In medical settings, test performance and external validation are particularly important because models are often intended to support decisions for future patients, not to describe past data.

12. Overfitting and why it matters in medicine

Overfitting occurs when a model learns noise or accidental patterns in the training data rather than generalisable signal. Overfitted models often look impressive during development but fail when applied to new data.

This problem is common in medical machine learning because many medical datasets have small sample sizes, rare outcomes, missing data, repeated measurements, many candidate predictors, or high-dimensional biomarkers. Omics datasets are especially vulnerable because the number of predictors may be far larger than the number of observations.

For example, if we have 200 patients and 450,000 methylation sites, some CpG sites will appear associated with the outcome purely by chance. If we select those sites using the full dataset and then evaluate the model using cross-validation, performance may be inflated because the test folds have indirectly influenced feature selection.

Overfitting is not just a technical inconvenience. In medicine, it can lead to false optimism, wasted validation studies, inappropriate clinical claims, and models that fail when used in real patients.

13. Bias-variance trade-off

The bias-variance trade-off is a useful way to think about model complexity. A very simple model may have high bias because it cannot capture the true relationship between predictors and outcome. For example, a simple linear model may miss a strong nonlinear relationship between age and risk. A very complex model may have high variance because it adapts too closely to the training data and changes substantially if trained on a different sample.

A useful way to think about prediction error is:

Prediction error = bias + variance + irreducible error

This is a simplification, but it is conceptually helpful. Bias reflects systematic error from an overly simple model. Variance reflects instability from an overly flexible model. Irreducible error reflects variation that cannot be predicted from the available data.

In biostatistics, this trade-off has practical consequences. A complex model such as gradient boosting may outperform logistic regression in some settings, but it may also require larger data, careful tuning, calibration assessment, and external validation. A simpler model may be preferable if it performs similarly and is easier to explain, validate, and implement.

14. Discrimination

Discrimination measures whether the model can separate individuals who experience the outcome from those who do not. In binary classification, discrimination is often assessed using the ROC curve and area under the ROC curve (AUC). In survival analysis, discrimination is often assessed using the C-index or time-dependent AUC.

If a mortality model gives higher predicted risks to patients who die than to patients who survive, it has good discrimination. However, discrimination is about ranking, not necessarily about accurate absolute probabilities.

For example, a model may correctly rank patients from low to high risk but systematically overestimate the actual risk. It may assign 70% risk to a group whose observed risk is only 30%. Such a model may have reasonable discrimination but poor calibration.

Discrimination is important, but it is not enough for clinical prediction.

15. Calibration

Calibration measures whether predicted probabilities agree with observed risks. If a model predicts that 100 patients each have about 20% risk, then approximately 20 of those patients should experience the outcome if the model is well calibrated.

Calibration is particularly important in clinical decision-making because decisions are often based on absolute risk thresholds. For example, a clinician may recommend additional monitoring if predicted risk exceeds 20%, or preventive treatment if predicted cardiovascular risk exceeds a guideline threshold. If the model systematically overestimates or underestimates risk, it can lead to overtreatment, undertreatment, patient anxiety, false reassurance, or inefficient use of healthcare resources.

Steyerberg places strong emphasis on evaluating both discrimination and calibration when assessing clinical prediction models (Steyerberg, 2019). TRIPOD+AI also expects transparent reporting of model performance, including relevant measures of discrimination and calibration when applicable (Collins et al., 2024).

16. Accuracy is not enough

Accuracy is often the first metric beginners learn, but it can be misleading in medical datasets. Suppose disease prevalence is 5%. A model that predicts “no disease” for everyone will be 95% accurate, but it will identify none of the diseased patients. Such a model is useless for screening.

The relevant metric depends on the clinical task. In screening, sensitivity may be prioritised because missing disease is harmful. In confirmatory testing, specificity may be prioritised because false positives may lead to unnecessary invasive procedures. In risk prediction, calibration may be essential because absolute risk estimates guide decisions. In treatment decision-making, decision curve analysis may help assess whether a model offers net benefit at clinically meaningful thresholds.

Therefore, medical prediction models should be evaluated using a combination of metrics rather than a single headline number.

17. Clinical usefulness

A model can be statistically impressive but clinically unhelpful. Clinical usefulness depends on whether the model can improve decisions.

For example, a model that predicts 30-day readmission may be useful only if the hospital has an intervention that can reduce readmission among high-risk patients. A survival prediction model may be useful if it supports follow-up planning, shared decision-making, or treatment stratification. A disease screening model may be useful if it detects cases earlier without producing excessive false positives.

Clinical usefulness depends on the prediction time point, availability of predictors, harms of false positives and false negatives, patient preferences, workflow integration, and threshold probabilities. Decision curve analysis is one approach for evaluating whether a model provides net benefit across a range of clinically relevant thresholds.

The key point is that a prediction model is not useful simply because it predicts. It is useful if its predictions can support better decisions.

18. Data leakage

Data leakage occurs when information is used during model development that would not be available when making predictions in real practice. Leakage can make a model appear much better than it really is.

Common examples include using variables measured after the prediction time, selecting features using the full dataset before train/test splitting, standardising predictors using the full dataset, imputing missing values using the test data, tuning hyperparameters on the test set, or allowing repeated records from the same patient to appear in both training and test sets.

Consider ICU mortality prediction at admission. Appropriate predictors might include age, admission diagnosis, baseline vital signs, and admission laboratory values. Inappropriate predictors would include length of ICU stay, discharge destination, final diagnosis after discharge, or lab values measured shortly before death. These variables may be strongly associated with mortality, but they are not available at admission. Including them would create leakage.

Leakage is one of the most serious threats to medical machine learning because it can produce models that look excellent in publications but fail in real clinical use.

19. Reporting and risk of bias

Transparent reporting is essential because readers need to understand how a model was developed, validated, and intended to be used. A prediction model report should describe the data source, study design, target population, inclusion and exclusion criteria, prediction time point, outcome definition, predictor definitions, missing data handling, sample size, model development process, hyperparameter tuning, validation, performance measures, calibration, model specification, and limitations.

TRIPOD+AI provides updated reporting guidance for prediction model studies using regression or machine learning methods (Collins et al., 2024). PROBAST+AI provides a structured approach to assessing quality, risk of bias, and applicability of prediction models using regression or artificial intelligence methods (Moons et al., 2025).

Risk of bias can arise from unrepresentative participants, poor outcome definitions, predictors unavailable at prediction time, inappropriate missing data handling, small sample size, overfitting, poor validation, selective reporting, or lack of calibration assessment. Applicability asks whether the model is relevant to the intended population and setting.

20. Practical framework for medical machine learning

A responsible medical machine learning workflow should begin with the clinical question, not the algorithm.

A useful workflow is:

Define the clinical prediction question.

Define the target population.

Define the prediction time point.

Define the outcome and outcome window.

Define candidate predictors available at prediction time.

Consider missing data and measurement quality.

Split data or define a resampling strategy.

Perform preprocessing inside the resampling process.

Fit a baseline model before complex models.

Tune models using appropriate validation.

Evaluate discrimination.

Evaluate calibration.

Consider clinical usefulness.

Assess internal validation.

Seek external validation where possible.

Report using TRIPOD+AI principles.

Assess risk of bias and applicability using PROBAST+AI principles.

This framework keeps the course focused on biostatistics. It ensures that machine learning is not treated as a black-box competition, but as a disciplined modelling process for medical data.

21. Summary

Machine learning in biostatistics is best understood as the development, validation, interpretation, and reporting of prediction models for medical and biomedical data.

The most important idea in this note is the distinction between prediction and inference. Inference asks what relationship exists between variables. Prediction asks how accurately outcomes can be estimated for new individuals. A model designed for inference should not automatically be judged by prediction metrics alone, and a model designed for prediction should not automatically be interpreted causally.

A good medical machine learning model should be well validated, well calibrated, clinically useful, free from major leakage, appropriate for the target population, transparently reported, and assessed for risk of bias and applicability.

Key takeaway

Prediction and inference are both central to biostatistics, but they answer different questions. Before fitting any model, define whether the aim is to estimate a relationship, estimate a causal effect, or predict outcomes for new individuals. In medical machine learning, responsible prediction modelling requires validation, calibration, clinical interpretation, transparent reporting, and careful avoidance of data leakage.

Exercise 5: Short answer

Write 3–5 sentences for each question before opening the suggested answer.

Why is prediction different from inference?

Click to see suggested answer

Prediction focuses on estimating outcomes or risks for new individuals. Inference focuses on estimating and interpreting relationships between variables. A prediction model is judged mainly by performance in new data, including discrimination, calibration and validation. An inference model is judged more by effect estimates, uncertainty, assumptions, confounding and interpretation.

Why is accuracy not enough in medical machine learning?

Click to see suggested answer

Accuracy can be misleading when outcomes are rare or class imbalance is present. A model can achieve high accuracy by predicting the majority class for everyone. In medicine, missing true cases may be very harmful, so sensitivity, specificity, predictive values, calibration and clinical usefulness are also needed.

Why is external validation important?

Click to see suggested answer

External validation tests whether a model works in data from a different setting, hospital, time period or population. A model may perform well in development data but fail elsewhere because of overfitting, population differences or measurement differences. External validation gives stronger evidence that the model can generalise to real use.

Why can data leakage make a model look better than it really is?

Click to see suggested answer

Data leakage occurs when information enters the model that would not be available at the real prediction time. This can allow the model to indirectly see the answer or future information. As a result, performance estimates become falsely high, and the model may fail when used prospectively on real patients.

Why should interpretability not be confused with causality?

Click to see suggested answer

An interpretable model may show which variables are associated with predictions, but this does not prove that those variables cause the outcome. A predictor can be useful because it is correlated with disease severity, healthcare use or another cause. Causal interpretation requires a causal question, appropriate design, temporality, confounding control and assumptions that are not guaranteed by interpretability alone.
