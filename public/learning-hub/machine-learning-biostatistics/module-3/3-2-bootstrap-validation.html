Lesson 7: Bootstrap Validation

Bootstrap validation estimates how much apparent model performance may be exaggerated. This lesson explains apparent performance, optimism, bootstrap sampling, optimism-corrected performance, and why bootstrap validation remains internal validation.

Today’s big question

How can bootstrap validation help estimate optimism in model performance?

Bootstrap validation is an internal validation method. It repeatedly samples from the development data, fits the model, and estimates how much apparent performance may be too optimistic.

Bootstrap validation helps adjust for overly optimistic performance estimates from the development data.

Intuition first

Performance on the development data is usually too good

When a model is evaluated on the same data used to build it, it usually looks better than it really is. This is called optimism.

Bootstrap validation tries to estimate that optimism.

Optimism =
apparent performance - expected new-data performance

Bootstrap sample

Sampling patients with replacement

A bootstrap sample is created by sampling patients from the original dataset with replacement.

This means:

  • some patients appear more than once;
  • some patients do not appear at all;
  • the sample has the same size as the original dataset.

The patients not selected in a bootstrap sample are sometimes called out-of-bag observations.

Curious Learner:
“So bootstrap creates new fake datasets?”

Prof Stat:
“Not fake, but resampled versions of the original data.”

Dr Clinic:
“And we use them to understand how stable the model performance is?”

Prof Stat:
“Yes. Especially how much apparent performance may be optimistic.”

Apparent performance and optimism

Why bootstrap validation is useful

Apparent performance

Same data for fitting and evaluation

Fit the model on all development data, then evaluate it on the same data.

This is often overly positive.

Optimism

The exaggeration amount

The difference between performance on the development data and expected performance on new data.

Apparent AUC = 0.84
Estimated optimism = 0.04
Optimism-corrected AUC = 0.84 - 0.04 = 0.80

This gives a more realistic estimate of performance.

Medical example

Cancer recurrence prediction

Suppose a logistic regression model for cancer recurrence has:

Apparent AUC = 0.84

After bootstrap validation, estimated optimism is:

Optimism = 0.04

Then optimism-corrected AUC is approximately:

0.84 - 0.04 = 0.80

This suggests that some of the apparent performance was due to overfitting. The corrected estimate is more cautious and more realistic than apparent performance alone.

Common mistake

Reporting only apparent performance

Common mistake: reporting only apparent performance

If you fit a model and report performance on the same data, the result is likely too optimistic.

Medical prediction models should include internal validation, such as bootstrap validation or cross-validation.

Clinical reality check

An optimism-corrected model can still fail externally. Bootstrap validation improves internal assessment, but it does not prove the model will work in another hospital or population.

External validation is still needed before clinical use.

Detailed notes

Note 7: Bootstrap Validation in Clinical Prediction Modelling

Bootstrap validation helps estimate how much a model’s apparent performance is likely to be too optimistic.

1. Why this note matters

In medical prediction modelling, performance estimated in the development dataset is usually too optimistic. A model may appear to discriminate well, classify patients accurately, or produce low prediction error simply because it has already been developed using the same data.

This is especially important in medical research because many datasets are small, outcomes may be rare, and the number of candidate predictors may be large. A model can easily learn dataset-specific patterns that do not apply to future patients.

2. Apparent performance

Apparent performance is the performance of a model when it is evaluated on the same data used to develop it.

Fit the model on the full development dataset. Evaluate the model on the same full development dataset.

The problem is that the model has already seen the data. It may have learned real patterns, but it may also have learned random noise, sample-specific patterns, or modelling choices that worked well in this dataset.

3. Why apparent performance is optimistic

Apparent performance is optimistic because model development is not a neutral process.

During development, analysts may:

  • try several models;
  • select predictors;
  • transform variables;
  • tune hyperparameters;
  • choose interactions;
  • remove outliers;
  • compare thresholds;
  • select the best-performing model.

Therefore, apparent performance reflects both true signal and overfitting.

4. Optimism

Optimism is the amount by which apparent performance overestimates performance on new data.

Optimism = apparent performance - expected performance on new data

For higher-is-better measures such as AUC or C-index, optimism is subtracted from apparent performance.

Apparent AUC = 0.86 Estimated optimism = 0.04 Optimism-corrected AUC = 0.82

5. What is bootstrap sampling?

Bootstrap sampling means sampling observations from the original dataset with replacement. If the original dataset contains 500 patients, a bootstrap sample also contains 500 rows.

However:

  • some patients may appear more than once;
  • some patients may not appear at all;
  • the sample is a resampled version of the original dataset.

6. Basic bootstrap validation process

1. Fit the original model in the full development data.
2. Calculate apparent performance.
3. Draw a bootstrap sample.
4. Fit the model in the bootstrap sample.
5. Evaluate performance in the bootstrap sample.
6. Evaluate the same bootstrap-fitted model in the original data.
7. Estimate optimism for that bootstrap sample.
8. Repeat many times.
9. Average the optimism.
10. Subtract average optimism from apparent performance.

7. Bootstrap validation versus train/test split

A train/test split holds out part of the dataset for evaluation. Bootstrap validation uses resampling and can allow the full dataset to contribute to model development. This can be attractive in clinical prediction modelling because sample sizes are often limited.

8. Bootstrap validation versus cross-validation

Both bootstrap validation and cross-validation are internal validation methods. Cross-validation repeatedly divides the dataset into training and validation folds. Bootstrap validation repeatedly samples with replacement and estimates optimism.

In clinical prediction modelling, bootstrap validation is often preferred for estimating optimism in a final model, especially when the sample size is limited. Cross-validation is often used for model comparison and tuning.

9. Include the full modelling process

Bootstrap validation should include all major steps in model development, such as:

  • feature selection;
  • transformation selection;
  • tuning;
  • imputation;
  • variable filtering;
  • model comparison.

If not, the optimism estimate may be too small.

10. Bootstrap validation and calibration

Bootstrap validation can assess more than discrimination. It can also assess:

  • calibration slope;
  • calibration intercept;
  • Brier score;
  • calibration plots;
  • C-index;
  • overall performance.

A calibration slope less than 1 often suggests overfitting and predictions that are too extreme.

11. How many bootstrap samples?

Common choices include 100, 200, 500 or 1000 bootstrap samples. More samples usually give more stable estimates but require more computation.

12. Bootstrap validation is internal validation

Bootstrap validation uses the development dataset. It gives information about expected performance in similar patients from the same source population. It does not prove that the model will work in another hospital, country, laboratory system, or time period.

13. What bootstrap validation does not solve

  • poor outcome definition;
  • poor predictor quality;
  • data leakage;
  • missing important predictors;
  • poor measurement quality;
  • unrepresentative data;
  • lack of external validation;
  • lack of clinical usefulness.

14. Common mistakes

  • Reporting only apparent performance.
  • Calling bootstrap validation external validation.
  • Not repeating feature selection inside the bootstrap process.
  • Ignoring calibration.
  • Using too few bootstrap samples.

15. Reporting bootstrap validation

A clear report should describe:

  • number of bootstrap samples;
  • performance measures evaluated;
  • apparent performance;
  • estimated optimism;
  • optimism-corrected performance;
  • whether feature selection was repeated;
  • whether tuning was repeated;
  • calibration assessment;
  • limitations.

16. Worked example: hospital mortality model

Suppose we develop a model to predict 30-day hospital mortality.

Apparent AUC = 0.88 Estimated optimism = 0.06 Optimism-corrected AUC = 0.82

This suggests that the model is likely less impressive than the apparent performance suggested. If the calibration slope is also low, the model may be overfitted and may require shrinkage or recalibration before further testing.

17. Summary

Bootstrap validation is a practical internal validation method for clinical prediction models. It estimates how much apparent performance may be exaggerated due to overfitting.

It is particularly useful when sample size is limited and when holding out a test set would be inefficient. However, bootstrap validation is not external validation.

R code

R practical: bootstrap validation for optimism in AUC

This script estimates optimism in AUC using a simple bootstrap approach. It fits a logistic regression model, calculates apparent AUC, repeatedly fits the model in bootstrap samples, estimates optimism, and calculates optimism-corrected AUC.

Apparent AUC

The model is fitted and evaluated on the full development dataset.

Bootstrap optimism

Each bootstrap sample estimates how much performance is exaggerated.

Corrected AUC

Estimated optimism is subtracted from apparent AUC.

# Machine Learning in Biostatistics
# Script 07: Bootstrap Validation
#
# This script estimates optimism in AUC using a simple bootstrap approach.
# It is written for teaching, so the steps are explicit.

library(tidyverse)
library(mlbench)
library(pROC)

set.seed(123)

data("PimaIndiansDiabetes")

diabetes_data <- PimaIndiansDiabetes %>%
  mutate(
    diabetes = factor(diabetes, levels = c("neg", "pos"))
  )

fit_model <- function(data) {
  glm(
    diabetes ~ pregnant + glucose + pressure + triceps +
      insulin + mass + pedigree + age,
    data = data,
    family = binomial
  )
}

calculate_auc <- function(model, data) {
  predicted_risk <- predict(model, newdata = data, type = "response")

  roc_object <- roc(
    response = data$diabetes,
    predictor = predicted_risk,
    levels = c("neg", "pos"),
    quiet = TRUE
  )

  as.numeric(auc(roc_object))
}

apparent_model <- fit_model(diabetes_data)
apparent_auc <- calculate_auc(apparent_model, diabetes_data)

cat("\nApparent AUC:\n")
print(apparent_auc)

n_boot <- 100
n <- nrow(diabetes_data)

bootstrap_results <- map_dfr(1:n_boot, function(b) {
  bootstrap_index <- sample(seq_len(n), size = n, replace = TRUE)

  bootstrap_data <- diabetes_data[bootstrap_index, ]

  bootstrap_model <- fit_model(bootstrap_data)

  auc_bootstrap <- calculate_auc(bootstrap_model, bootstrap_data)
  auc_original <- calculate_auc(bootstrap_model, diabetes_data)

  tibble(
    bootstrap_sample = b,
    auc_bootstrap = auc_bootstrap,
    auc_original = auc_original,
    optimism = auc_bootstrap - auc_original
  )
})

cat("\nFirst few bootstrap results:\n")
print(head(bootstrap_results))

optimism_estimate <- mean(bootstrap_results$optimism)
optimism_corrected_auc <- apparent_auc - optimism_estimate

summary_table <- tibble(
  apparent_auc = apparent_auc,
  mean_optimism = optimism_estimate,
  optimism_corrected_auc = optimism_corrected_auc
)

cat("\nBootstrap validation summary:\n")
print(summary_table)

plot(
  bootstrap_results$bootstrap_sample,
  bootstrap_results$optimism,
  type = "h",
  xlab = "Bootstrap sample",
  ylab = "Optimism",
  main = "Estimated Optimism Across Bootstrap Samples"
)

abline(h = optimism_estimate, lty = 2)

cat("\nLearning points:\n")
cat("1. Apparent performance is usually optimistic.\n")
cat("2. Bootstrap validation estimates optimism.\n")
cat("3. Optimism-corrected performance is more realistic.\n")
cat("4. Bootstrap validation is internal validation.\n")
cat("5. External validation is still needed before clinical use.\n")

Exercise 7: Bootstrap Validation

This exercise helps you understand apparent performance, optimism, and the role of bootstrap validation in clinical prediction modelling.

Question 1: Concept check

What is apparent performance?

Show suggested answer

Apparent performance is the performance measured on the same development data used to fit or develop the model. It is often too optimistic because the model has already seen the data.

Question 2: Optimism

Apparent AUC = 0.86 Estimated optimism = 0.05

What is the optimism-corrected AUC?

Show suggested answer

Optimism-corrected AUC = 0.86 - 0.05 = 0.81.

Question 3: Interpretation

Why is apparent performance usually too optimistic?

Show suggested answer

The model is evaluated on the same data used during development. It may have learned not only real signal but also random noise, sample-specific patterns, and modelling decisions that worked well only in that dataset.

Question 4: Bootstrap sample

What does it mean to sample with replacement?

Show suggested answer

Sampling with replacement means each patient can be selected more than once. As a result, some patients appear multiple times in the bootstrap sample, while others are not selected at all.

Question 5: Clinical interpretation

Apparent C-index = 0.78 Optimism-corrected C-index = 0.70

What does this suggest?

Show suggested answer

It suggests that the apparent model performance was substantially optimistic. The model may have overfitted the development data, so expected performance in similar new patients may be closer to 0.70 than 0.78.

Question 6: External validation

A model has good optimism-corrected performance after bootstrap validation. Is it ready for clinical use?

Show suggested answer

Not necessarily. Bootstrap validation is internal validation. The model still needs external validation, calibration assessment, clinical usefulness assessment, and careful evaluation in the intended setting before clinical use.

Question 7: Method choice

Why might bootstrap validation be attractive in studies with limited sample size?

Show suggested answer

Bootstrap validation can use the full development dataset while estimating optimism. This can be attractive when holding out a test set would leave too little data or too few events for stable model development and evaluation.

Reflection

Why is it risky to report only apparent performance in a medical prediction study?

Show reflection answer

Reporting only apparent performance can exaggerate how well the model will perform for new patients. This can lead researchers and clinicians to trust a model that may fail when applied outside the development data. In medical settings, that can affect diagnosis, treatment, monitoring, and patient safety.
