Lesson 12: Lasso Regression

Lasso regression is a regularisation method that can shrink some coefficients exactly to zero. This makes it useful for feature selection and sparse clinical prediction models, but selected features must be validated and interpreted cautiously.

Big idea

Lasso can shrink coefficients exactly to zero

Lasso regression adds a penalty to the model-fitting process. Like ridge regression, it shrinks coefficients. But lasso has one special feature: some coefficients can become exactly zero.

If a coefficient becomes zero, that predictor is effectively removed from the model.

This means lasso can perform feature selection while building a prediction model. In medical prediction, this can be attractive when we have many candidate predictors and want a smaller model.

Why lasso is attractive

Smaller models can be easier to use

Lasso can be useful when:

  • there are many candidate predictors;
  • some predictors may be weak or redundant;
  • a simpler model is preferred;
  • clinical measurement burden matters;
  • the dataset is too small for ordinary regression with all predictors;
  • the goal is prediction, not causal explanation.

Clinical interpretation

A smaller model may be easier to measure, explain and implement. But smaller does not automatically mean better. A sparse model can still be poorly validated, miscalibrated or clinically unhelpful.

Lambda and sparsity

The penalty controls how many predictors remain

Lasso uses a tuning parameter called lambda. Lambda controls the amount of shrinkage.

Small lambda

Less shrinkage

More predictors are retained. The model may still overfit if shrinkage is too weak.

Large lambda

More shrinkage

Fewer predictors are retained. The model may underfit if shrinkage is too strong.

Lasso versus ridge

Both shrink, but they behave differently

Method What it does Feature selection?
RidgeShrinks coefficients but usually keeps all predictorsUsually no

Ridge is useful when many predictors may each contain useful information. Lasso is useful when we want a smaller selected predictor set.

Correlated predictors

Lasso selection can be unstable

Suppose two biomarkers carry very similar information. Lasso may select biomarker A and remove biomarker B. In another sample, it may select biomarker B and remove biomarker A.

Common mistake: assuming selected means biologically superior

If predictors are highly correlated, lasso may choose one somewhat arbitrarily. This does not prove that the selected predictor is biologically more important.

When correlated predictors are common and feature stability matters, elastic net may be considered because it combines lasso and ridge behaviour.

Feature-selection leakage

Lasso must be inside the validation process

Leakage Monster:
“Select features using the full dataset first. The AUC will look amazing.”

Curious Learner:
“That sounds tempting.”

Prof Stat:
“And biased. The validation data would already have influenced the model.”

Dr Clinic:
“So the model may fail on real patients?”

Prof Stat:
“Exactly.”

Incorrect workflow

Run lasso on the full dataset, keep selected predictors, then cross-validate the reduced model.

This leaks information from validation folds into feature selection.

Correct principle

For each fold, run lasso inside the training fold only, then evaluate on the validation fold.

The validation fold must remain unseen during feature selection.

Detailed notes

Note 12: Lasso Regression

Lasso regression can perform feature selection, but selected features must be validated and interpreted cautiously.

Learning objectives

  • Understand the role of the tuning parameter lambda.
  • Explain why lasso-selected predictors should not automatically be interpreted as causal.

1. Why this note matters

Medical and biomedical datasets often contain many candidate predictors. Some predictors may be useful, while others may be weak, redundant, noisy, unavailable in practice, or not clinically meaningful.

Lasso regression is attractive because it can shrink some coefficients exactly to zero. This means lasso can produce a smaller prediction model.

2. What lasso regression does

Lasso regression is a regularisation method. It adds a penalty to the model-fitting process. This penalty encourages some coefficients to become exactly zero. If a predictor has a coefficient of zero, it is effectively removed from the model.

3. Lasso and coefficient shrinkage

Like ridge regression, lasso shrinks coefficients. However, lasso can shrink some coefficients all the way to zero.

This can create a smaller and more practical model, but the selected predictors must not be over-interpreted.

4. Why lasso is useful

Lasso can be useful when:

  • there are many candidate predictors;
  • only some predictors are expected to be useful;
  • a simpler model is desired;
  • clinical measurement burden matters;
  • the model needs to be easier to explain;
  • the dataset is too small for ordinary regression with all predictors.

5. Lasso as feature selection

Feature selection means choosing a subset of predictors. Lasso performs feature selection automatically by shrinking some coefficients to zero. This can help create a sparse model.

However, sparse does not automatically mean better. A smaller model still needs validation, calibration assessment and clinical usefulness assessment.

6. The tuning parameter lambda

Lambda controls how much shrinkage is applied.

LambdaShrinkagePredictors retainedMain risk
Small lambdaLessMoreMay still overfit
Large lambdaMoreFewerMay underfit

Different lambda values can lead to different selected predictors. Cross-validation is commonly used to choose lambda.

7. Choosing lambda carefully

An honest workflow is:

1. Split data into training and test sets.
2. Use cross-validation within the training set to choose lambda.
3. Fit the lasso model using the chosen lambda.
4. Evaluate once on the untouched test set.

This keeps the test set honest.

8. Lasso versus ridge

MethodMain behaviourFeature selection?
RidgeShrinks coefficients towards zeroUsually no
LassoShrinks some coefficients exactly to zeroYes

Ridge is often useful when many predictors each contribute small information. Lasso is often useful when we want a smaller predictor set.

9. Lasso and correlated predictors

Lasso can be unstable when predictors are strongly correlated. If two biomarkers measure a similar process, lasso may select one in one sample and the other in another sample.

This does not necessarily mean one is truly more important. It may simply reflect instability due to correlation.

10. Stability of selected predictors

Because lasso selection can be unstable, check whether selected predictors are stable across:

  • cross-validation folds;
  • bootstrap samples;
  • repeated analyses;
  • external datasets.

11. Prediction versus causation

Lasso is a prediction tool. A selected predictor may be:

  • causally related to the outcome;
  • correlated with a causal variable;
  • a proxy for disease severity;
  • a technical artefact;
  • selected by chance;
  • specific to one dataset.

Therefore, lasso selection does not prove causality.

12. Feature-selection leakage

An incorrect workflow is:

1. Run lasso on the full dataset.
2. Keep the selected predictors.
3. Perform cross-validation using only those predictors.
4. Report cross-validated performance.

This is biased because the validation folds helped select predictors. The performance estimate may be too optimistic.

13. Correct lasso validation workflow

A correct cross-validation workflow is:

For each fold:
1. Use only the training fold.
2. Select lambda using only the training fold.
3. Fit the lasso model using only the training fold.
4. Predict the validation fold.
5. Store validation performance.

Feature selection must happen inside the validation process.

14. Lasso in omics

Lasso is often used in gene expression prediction, methylation-based prediction, proteomic biomarker panels and metabolomic risk prediction.

For example, with 300 participants and 20,000 gene expression features, lasso may select a smaller set of genes. But selected genes should not automatically be treated as biological discoveries.

15. Medical examples

In diabetes prediction, lasso may remove weak predictors and keep variables such as glucose, BMI, age and pedigree. This can produce a simpler model, but AUC, calibration, Brier score, clinical usefulness and external validation are still needed.

In cancer biomarker prediction, lasso may select a small biomarker panel. Before claiming importance, check stability, biological plausibility, routine measurability, external validation, batch effects, calibration and clinical usefulness.

16. Limitations of lasso

  • Selected features may be unstable.
  • Correlated predictors can cause arbitrary selection.
  • Selected features are not automatically causal.
  • Performance may be optimistic if validation is incorrect.
  • A sparse model may underfit if many predictors each have small effects.

Elastic net may be useful when correlated predictors are common.

17. Reporting lasso regression

A clear report should describe:

  • outcome definition;
  • candidate predictors;
  • preprocessing steps;
  • missing data handling;
  • lambda selection method;
  • whether feature selection was inside validation;
  • selected predictors;
  • discrimination;
  • calibration;
  • external validation if available;
  • limitations of selected feature interpretation.

18. Summary

Lasso regression is a regularisation method that can perform feature selection. It creates sparse models by shrinking some coefficients to zero. This can be useful in medical prediction when we want a smaller model.

Lasso is a prediction method, not automatic causal discovery. Selected predictors must be validated and interpreted carefully.

R code

R practical: lasso regression for diabetes prediction

This script fits lasso regression for diabetes prediction, calculates test AUC, prints all lasso coefficients and shows which predictors are selected.

alpha = 1

In glmnet, alpha = 1 fits lasso regression.

lambda.min

Cross-validation chooses the lambda value with minimum cross-validated error.

Selected features

Predictors with non-zero coefficients are retained.

Download R script 

# Machine Learning in Biostatistics
# Script 12: Lasso Regression
#
# This script fits lasso regression for diabetes prediction
# and shows which predictors are selected.

library(tidyverse)
library(mlbench)
library(pROC)
library(glmnet)

set.seed(123)

data("PimaIndiansDiabetes")

diabetes_data <- PimaIndiansDiabetes %>%
  mutate(
    diabetes = factor(diabetes, levels = c("neg", "pos"))
  )

train_index <- sample(seq_len(nrow(diabetes_data)), size = round(0.80 * nrow(diabetes_data)))

train_data <- diabetes_data[train_index, ]
test_data <- diabetes_data[-train_index, ]

x_train <- model.matrix(
  diabetes ~ pregnant + glucose + pressure + triceps +
    insulin + mass + pedigree + age,
  data = train_data
)[, -1]

x_test <- model.matrix(
  diabetes ~ pregnant + glucose + pressure + triceps +
    insulin + mass + pedigree + age,
  data = test_data
)[, -1]

y_train <- ifelse(train_data$diabetes == "pos", 1, 0)
y_test_factor <- test_data$diabetes

lasso_model <- cv.glmnet(
  x = x_train,
  y = y_train,
  family = "binomial",
  alpha = 1
)

lasso_risk <- as.numeric(
  predict(
    lasso_model,
    newx = x_test,
    s = "lambda.min",
    type = "response"
  )
)

calculate_auc <- function(actual, risk) {
  roc_object <- roc(
    response = actual,
    predictor = risk,
    levels = c("neg", "pos"),
    quiet = TRUE
  )

  as.numeric(auc(roc_object))
}

lasso_auc <- calculate_auc(y_test_factor, lasso_risk)

cat("\nLasso model test AUC:\n")
print(lasso_auc)

lasso_coefficients <- coef(lasso_model, s = "lambda.min")

lasso_coefficients_table <- tibble(
  term = rownames(lasso_coefficients),
  coefficient = as.numeric(lasso_coefficients)
)

cat("\nAll lasso coefficients:\n")
print(lasso_coefficients_table)

lasso_selected <- lasso_coefficients_table %>%
  filter(coefficient != 0)

cat("\nFeatures selected by lasso:\n")
print(lasso_selected)

cat("\nNumber of selected terms including intercept:\n")
print(nrow(lasso_selected))

plot(lasso_model, main = "Lasso Cross-Validation")

cat("\nLearning points:\n")
cat("1. Lasso regression shrinks coefficients towards zero.\n")
cat("2. Lasso can shrink some coefficients exactly to zero.\n")
cat("3. Predictors with zero coefficients are removed from the model.\n")
cat("4. Lasso-selected features are prediction features, not automatically causal factors.\n")
cat("5. Feature selection must be included inside validation to avoid leakage.\n")

Part D: Correlated predictors

Question 10

Lasso selects Biomarker A and removes highly correlated Biomarkers B and C. Can we conclude A is biologically more important?

Show suggested answer

No. If the biomarkers are highly correlated, lasso may choose one of them somewhat arbitrarily. Biomarker A may be useful for prediction, but this does not prove it is biologically more important.

Question 11

Why can lasso be unstable when predictors are strongly correlated?

Show suggested answer

When predictors are strongly correlated, several predictors may contain similar information. Lasso may choose one and remove others, but small changes in the data may change which predictor is selected.

Question 12

What method might be considered if lasso is unstable because predictors are highly correlated?

Show suggested answer

Elastic net may be considered because it combines lasso and ridge behaviour and can handle correlated predictors more smoothly.

Part E: Feature-selection leakage

Question 13

A researcher runs lasso on the full dataset, keeps selected predictors, then performs 10-fold cross-validation. What is wrong?

Show suggested answer

The full dataset was used to select predictors before cross-validation. This means the validation folds influenced feature selection. The reported cross-validated AUC may be optimistic.

Question 14

Rewrite the workflow correctly.

Show suggested answer

For each cross-validation fold: use only the training fold, run lasso inside the training fold, select predictors inside the training fold, fit the model inside the training fold, and evaluate on the validation fold.

Question 15

Why is feature-selection leakage especially dangerous in biomedical datasets with many predictors?

Show suggested answer

With many predictors, some will appear related to the outcome by chance. If the full dataset is used to select features, random signals from validation data can enter model development and inflate performance.

Part F: Biomedical prediction scenario

What model performance measures should they report besides AUC?

Question 18

Why should selected biomarkers be externally validated?

Show suggested answer

External validation tests whether the selected biomarkers and model performance generalise to new patients, settings, platforms or populations.
