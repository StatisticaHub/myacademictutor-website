Big idea

Feature selection chooses a subset of predictors

In biomedical machine learning, feature selection is common because datasets may contain many clinical variables, biomarkers, imaging features, genes, methylation sites, proteins or metabolites.

Feature selection can make a model simpler, but if done incorrectly, it can create serious data leakage.

Why feature selection is used

Smaller models can be more practical

Feature selection may be used to:

  • reduce model complexity;
  • remove noisy predictors;
  • reduce measurement burden;
  • make the model easier to explain;
  • identify candidate biomarkers;
  • improve practicality;
  • create a smaller prediction model.

Clinical practicality

A model using routinely available predictors is often easier to implement than a model requiring expensive specialised tests.

Types of feature selection

Clinical, statistical and model-based selection

Clinical

Based on medical knowledge

Uses predictors that are clinically plausible and available at prediction time.

Statistical

Based on data patterns

Uses p-values, correlations, univariable screening or other data-driven criteria.

Model-based

Based on fitted model behaviour

Uses lasso coefficients, random forest importance or recursive feature elimination.

Feature-selection leakage

The most dangerous mistake

Feature-selection leakage happens when validation or test data influence predictor selection.

Incorrect workflow

  1. Use full dataset to select features.
  2. Split into train and test sets.
  3. Fit model on training data.
  4. Evaluate on test data.

Problem: the test set helped choose the features.

Correct workflow

  1. Split data into training and test sets.
  2. Select features using training data only.
  3. Fit model using training data only.
  4. Evaluate once on the untouched test set.

Principle: validation data must remain unseen.

Why biomedical data are risky

Thousands of predictors create many accidental patterns

300 patients
450,000 methylation sites

With so many features, some will appear associated with the outcome by chance. If those features are selected using the full dataset before validation, performance may look excellent but fail externally.

Common mistake

Do not select biomarkers using the full dataset before cross-validation. This leaks outcome information into validation and can seriously inflate performance.

Batch effects

Technical artefacts can masquerade as biological signal

Batch effects are technical differences unrelated to biology.

Example sources

  • lab batch;
  • sequencing run;
  • methylation array plate;
  • sample processing date;
  • scanner type;
  • technician;
  • hospital site.

Confounding problem

If most cases are processed on Plate 1 and most controls on Plate 2, feature selection may identify plate differences, not disease biology.

If a model learns batch effects instead of disease biology, it may perform well internally but fail in external data where the batch structure is different.

Feature stability

A selected feature list should not change wildly

Feature stability means selected features are consistent across bootstrap samples, cross-validation folds, repeated analyses or external datasets.

If selected features change dramatically across resamples, interpretation should be cautious. The model may still predict, but the specific biomarker list may not be reliable.

Prediction versus discovery

Selected does not automatically mean causal

A selected feature may be:

  • causally important;
  • correlated with a causal factor;
  • a proxy for disease severity;
  • a technical artefact;
  • selected by chance;
  • specific to one dataset.

Always separate the prediction question from the discovery question.

Medical example

Methylation prediction workflow

Suppose we predict exposure status using DNA methylation.

Poor workflow

  1. Select top 100 CpG sites using full data.
  2. Cross-validate classifier.
  3. Report AUC.

Better workflow

  1. Create folds first.
  2. Select CpG sites inside each training fold.
  3. Fit model inside each training fold.
  4. Evaluate on held-out fold.

Detailed notes

Note 14: Feature Selection in Biomedical Data

Feature selection can simplify a biomedical prediction model, but if done incorrectly, it can seriously bias model performance.

Learning objectives

  • Explain why feature selection must happen inside validation.

1. Why this note matters

Biomedical datasets often contain many possible predictors, including clinical variables, blood test measurements, imaging features, biomarkers, gene expression values, DNA methylation sites, proteins, metabolites and electronic health record codes.

Feature selection is the process of choosing a subset of predictors. It can make models simpler and more practical, but it is also one of the easiest ways to accidentally create data leakage.

2. Feature selection is not automatically good

Feature selection can help, but it can also harm. It may remove useful predictors, select unstable predictors, select variables that work only in one dataset, or create data leakage if performed before validation.

3. Clinical feature selection

Clinical feature selection uses subject-matter knowledge. It helps ensure that the model uses predictors that are meaningful, measurable and available at the intended prediction time.

4. Statistical and model-based feature selection

Statistical feature selection uses data-driven criteria such as p-values, correlation filtering, stepwise regression, lasso selection, recursive feature elimination or model-based variable importance.

Model-based methods identify predictors that help prediction in a specific dataset and modelling workflow. They do not automatically identify causal factors.

5. Feature-selection leakage

Feature-selection leakage occurs when validation or test data influence predictor selection. If the full dataset is used to select predictors before splitting or cross-validation, evaluation is no longer independent.

6. Correct feature selection workflow

A safer train/test workflow is:

1. Split data into training and test sets.
2. Select features using training data only.
3. Fit the model using training data only.
4. Evaluate once on the untouched test set.

For cross-validation, feature selection must happen inside each training fold so each validation fold remains genuinely unseen.

7. Why this matters in omics

Omics datasets often contain thousands or hundreds of thousands of predictors. With so many features, some will appear associated with the outcome by chance. If selected before validation, these chance associations can seriously inflate performance.

8. Batch effects

Batch effects are technical differences unrelated to biology. Examples include laboratory batch, sequencing run, methylation array plate, sample processing date, scanner, technician or hospital site.

If batch is associated with outcome, feature selection may select variables that reflect batch rather than disease biology. This can produce excellent internal performance and poor external performance.

9. Feature stability

Stability means that the same or similar features are selected across bootstrap samples, cross-validation folds, repeated analyses or external datasets. If selected features change dramatically across resamples, interpretation should be cautious.

10. Correlated predictors and instability

Biomedical predictors are often correlated. Nearby methylation sites, genes in the same pathway, related proteins and imaging features from the same region may carry overlapping information. Selection methods may choose one predictor from a group in one sample and another predictor from the same group in another sample.

11. Prediction versus discovery

Feature selection for prediction is not the same as biological discovery. A selected predictor may help prediction because it is correlated with the outcome, but that does not prove it causes the outcome.

Similarly, a biologically important predictor may not be selected if it does not improve prediction in that dataset.

12. Biomedical examples

In diabetes prediction, feature selection may identify glucose, BMI, age and pedigree. These are clinically plausible, but selection should still be validated.

In methylation prediction, selecting the top CpG sites using full data before cross-validation will bias performance. The correct workflow is to select CpG sites inside each training fold and use external validation where possible.

13. Practical checklist

  • Were features selected inside validation?
  • Was the final test set untouched?
  • Are selected features stable?
  • Are they clinically or biologically plausible?
  • Are they measured reliably?
  • Are they available at the intended prediction time?
  • Could batch effects explain them?
  • Do they validate externally?
  • Do they improve calibration or only AUC?
  • Are claims limited to what the study design supports?

14. Reporting feature selection

A clear report should describe:

  • all candidate predictors;
  • feature selection method;
  • when feature selection was performed;
  • whether it was inside validation;
  • number of selected features;
  • stability of selected features if assessed;
  • clinical or biological interpretation;
  • validation performance;
  • limitations.

15. Summary

Feature selection is common in biomedical machine learning. It can make models simpler and more practical. However, feature selection must be included inside the validation process.

Selected features should be interpreted as prediction features first, not automatically as causal or biologically validated discoveries.

R practical: feature selection leakage versus honest selection

This script demonstrates why feature selection should be done inside the validation process. It compares selecting predictors using the full dataset with selecting predictors using training data only.

Full-data selection

Shows the leakage-prone workflow where all data are used before testing.

Training-only selection

Shows the safer workflow where test data remain untouched.

AUC comparison

Compares performance from the two workflows.

# Machine Learning in Biostatistics
# Script 14: Feature Selection in Biomedical Data
#
# This script demonstrates why feature selection should be done
# inside the validation process, not before validation.

library(tidyverse)
library(mlbench)
library(pROC)

set.seed(123)

data("PimaIndiansDiabetes")

diabetes_data <- PimaIndiansDiabetes %>%
  mutate(
    diabetes = factor(diabetes, levels = c("neg", "pos"))
  )

candidate_predictors <- c(
  "pregnant",
  "glucose",
  "pressure",
  "triceps",
  "insulin",
  "mass",
  "pedigree",
  "age"
)

rank_predictors_by_auc <- function(data, predictors) {
  map_dfr(predictors, function(current_predictor) {
    roc_object <- roc(
      response = data$diabetes,
      predictor = data[[current_predictor]],
      levels = c("neg", "pos"),
      quiet = TRUE
    )

    auc_value <- as.numeric(auc(roc_object))

    tibble(
      predictor = current_predictor,
      univariable_auc = max(auc_value, 1 - auc_value)
    )
  }) %>%
    arrange(desc(univariable_auc))
}

evaluate_model <- function(train_data, test_data, selected_predictors) {
  model_formula <- as.formula(
    paste("diabetes ~", paste(selected_predictors, collapse = " + "))
  )

  model <- glm(
    model_formula,
    data = train_data,
    family = binomial
  )

  risk <- predict(
    model,
    newdata = test_data,
    type = "response"
  )

  roc_object <- roc(
    response = test_data$diabetes,
    predictor = risk,
    levels = c("neg", "pos"),
    quiet = TRUE
  )

  as.numeric(auc(roc_object))
}

train_index <- sample(seq_len(nrow(diabetes_data)), size = round(0.80 * nrow(diabetes_data)))

train_data <- diabetes_data[train_index, ]
test_data <- diabetes_data[-train_index, ]

full_data_ranking <- rank_predictors_by_auc(
  data = diabetes_data,
  predictors = candidate_predictors
)

leakage_selected_predictors <- full_data_ranking %>%
  slice_head(n = 4) %>%
  pull(predictor)

train_data_ranking <- rank_predictors_by_auc(
  data = train_data,
  predictors = candidate_predictors
)

honest_selected_predictors <- train_data_ranking %>%
  slice_head(n = 4) %>%
  pull(predictor)

leakage_auc <- evaluate_model(
  train_data = train_data,
  test_data = test_data,
  selected_predictors = leakage_selected_predictors
)

honest_auc <- evaluate_model(
  train_data = train_data,
  test_data = test_data,
  selected_predictors = honest_selected_predictors
)

feature_selection_comparison <- tibble(
  workflow = c(
    "Leakage-prone: features selected using full data",
    "Honest: features selected using training data only"
  ),
  selected_predictors = c(
    paste(leakage_selected_predictors, collapse = ", "),
    paste(honest_selected_predictors, collapse = ", ")
  ),
  test_auc = c(
    leakage_auc,
    honest_auc
  )
)

cat("\nFeature ranking using the full dataset:\n")
print(full_data_ranking)

cat("\nFeature ranking using training data only:\n")
print(train_data_ranking)

cat("\nComparison of feature selection workflows:\n")
print(feature_selection_comparison)

cat("\nLearning points:\n")
cat("1. Feature selection is part of model development.\n")
cat("2. Feature selection should use training data only.\n")
cat("3. Selecting features before validation can cause leakage.\n")
cat("4. Leakage can make model performance look better than it really is.\n")
cat("5. Selected features are prediction features, not automatically causal factors.\n")

Question 9

Why can feature-selection leakage make model performance look better than it really is?

Show suggested answer

Validation or test data have already influenced model development, so evaluation is no longer independent. Random patterns from validation or test data can be selected and make performance look better.

Part D: Cross-validation workflow

Question 10

Which workflow is correct: A, select features once using full data before cross-validation; or B, select features inside each training fold?

Show suggested answer

B is correct. Feature selection must happen inside each training fold so that the validation fold remains unseen.

Question 11

Why is feature selection before cross-validation especially problematic in omics data?

Show suggested answer

Omics data contain many thousands of features. Some will appear related to the outcome by chance. If selected before validation, those chance signals can inflate performance.

Part E: Batch effects

Question 12

A batch effect is technical variation unrelated to biology. Examples include sequencing run, methylation array plate, lab technician, scanner, processing date or hospital site.

Part F: Feature stability

Question 15

What does feature stability mean?

Show suggested answer

Feature stability means selected features are consistent across resamples, folds, bootstrap samples or external datasets.

Question 16

A lasso model selects different biomarkers in each bootstrap sample. What does this suggest?

Show suggested answer

It suggests the selected biomarker list is unstable. The model may not provide a reliable set of biomarkers for interpretation.

Question 17

Why can correlated predictors make selected feature lists unstable?

Show suggested answer

When predictors are correlated, several variables contain similar information. A selection method may choose one in one sample and another in a different sample.

Part G: Prediction versus discovery

Question 18

A feature is selected by a prediction model. Does this prove it causes the outcome?

Show suggested answer

No. A selected feature may be causal, correlated with a causal factor, a proxy, a batch artefact or selected by chance.

Question 19

A biologically important predictor is not selected by the model. Does this mean it is not biologically important?

Show suggested answer

No. A biologically important predictor may not improve prediction enough to be selected, or it may be redundant with other predictors.

Part H: Biomedical scenario

A team is building a recurrence prediction model with 400 patients, 120 recurrence events, 10 clinical predictors and 10,000 gene expression features. They select the top 100 genes using the full dataset before cross-validation.

Question 20

Identify the main problem.

Show suggested answer

The main problem is feature-selection leakage. The full dataset was used to select genes before cross-validation.

Question 21

Describe a corrected analysis plan.

Show suggested answer

Define cross-validation folds. Within each training fold, perform gene selection. Fit the prediction model using selected genes in that training fold. Predict the held-out validation fold. Repeat across folds. Estimate performance from held-out predictions. Use external validation if possible.
