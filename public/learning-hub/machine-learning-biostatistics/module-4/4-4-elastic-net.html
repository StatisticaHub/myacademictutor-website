Big idea

Elastic net combines ridge and lasso

Ridge regression shrinks coefficients but usually keeps all predictors. Lasso regression shrinks some coefficients to zero and performs feature selection. Elastic net sits between them.

Elastic net can shrink coefficients, select predictors, and behave more stably when predictors are correlated.

Method Main behaviour Useful when
RidgeShrinks coefficients, usually keeps predictorsMany predictors each contain small signal
LassoShrinks some coefficients to zeroA very sparse model is desired
Elastic netCombines shrinkage and selectionPredictors are correlated and some selection is desired

Why elastic net exists

Lasso can be unstable with correlated predictors

If three biomarkers measure a similar biological process, lasso may select one and discard the others. In another dataset, it may select a different one.

Elastic net solution

Elastic net can handle correlated groups more smoothly. It may keep related predictors together while still applying shrinkage.

Alpha and lambda

Elastic net has two tuning ideas

Alpha

Ridge-lasso mixture

Alpha controls whether the model behaves more like ridge or more like lasso.

Lambda

Penalty strength

Lambda controls how strong the overall regularisation penalty is.

alpha = 0    → ridge regression
alpha = 1    → lasso regression
0 < alpha < 1    → elastic net

Curious Learner:
“Ridge has lambda. Lasso has lambda. Elastic net has alpha and lambda?”

Prof Stat:
“Correct. Alpha controls the ridge-lasso mixture. Lambda controls how strong the penalty is.”

Dr Clinic:
“More tuning means more responsibility.”

Prof Stat:
“Exactly. Tuning must be done inside the validation process.”

Correlated predictors

Elastic net is useful for grouped biomedical signals

Biomedical predictors are often correlated. Examples include:

  • genes in the same pathway;
  • neighbouring methylation sites;
  • related proteins;
  • repeated clinical measurements;
  • imaging features from the same region.

Elastic net can be useful because it can select features while also keeping grouped information more stable than lasso alone.

Choosing between ridge, lasso and elastic net

The goal determines the method

Prefer elastic net over ridge when

Some feature selection is desired

Use it when many predictors may be irrelevant, measurement burden matters, or a smaller model is preferred.

Prefer elastic net over lasso when

Predictors are highly correlated

Use it when lasso selection is unstable or grouped biological signals are expected.

Common mistake: tuning on the final test set

Do not try many alpha and lambda values and choose the combination with the best final test AUC. The test set would become part of model development, making final performance optimistic.

Tuning and validation

Keep tuning inside the training data

An honest workflow is:

1. Split data into training and test data.
2. Use cross-validation inside training data to choose alpha and lambda.
3. Fit the final elastic net model on training data.
4. Evaluate once on the untouched test data.

When many tuning values are explored, nested cross-validation can be useful. The inner loop tunes alpha and lambda. The outer loop estimates model performance.

Interpretation

Selected features are not automatic biological discoveries

Elastic net-selected features are prediction features. Before making biological or clinical claims, ask:

  • Are the features stable?
  • Are they biologically plausible?
  • Are they measured reliably?
  • Are they available at prediction time?
  • Do they validate externally?
  • Could they reflect batch effects?

Clinical reality check

Elastic net may reduce overfitting, but it does not guarantee good calibration. After fitting an elastic net model, assess discrimination, calibration, Brier score and clinical usefulness.

Detailed notes

Note 13: Elastic Net

Elastic net is useful when we want both coefficient shrinkage and some feature selection, especially when predictors are correlated.

1. Why this note matters

Ridge and lasso are both useful regularisation methods, but each has limitations. Ridge is stable with correlated predictors, but usually keeps all predictors. Lasso can perform feature selection, but may be unstable when predictors are strongly correlated.

2. What elastic net does

Elastic net uses a mixture of ridge and lasso penalties. It can shrink coefficients, set some coefficients to zero, handle correlated predictors better than lasso alone, and produce a model that is more stable than pure lasso in some settings.

3. The alpha parameter

Alpha controls the balance between ridge-like and lasso-like behaviour. If alpha is closer to 0, the model behaves more like ridge. If alpha is closer to 1, the model behaves more like lasso.

AlphaMeaning
0Ridge regression
1Lasso regression
Between 0 and 1Elastic net

4. The lambda parameter

Lambda controls the overall strength of regularisation. Small lambda gives a weak penalty, while large lambda gives a strong penalty. Elastic net therefore has two tuning ideas: alpha for the type of penalty mixture and lambda for penalty strength.

5. Why elastic net helps with correlated predictors

Biomedical predictors are often correlated, such as genes in the same pathway, neighbouring methylation sites, related proteins, repeated clinical measurements, or imaging features from the same region. Lasso may select one predictor from a correlated group and ignore the others. Elastic net can allow groups of related predictors to contribute more smoothly.

6. Elastic net in omics

Elastic net is commonly useful in omics prediction because omics data often have many predictors, high correlation, limited sample size, noisy measurements, many weak signals and high overfitting risk.

7. Elastic net versus ridge and lasso

Elastic net may be preferable to ridge when we want some feature selection, some predictors are likely irrelevant, the final model should be smaller, or measurement burden matters.

Elastic net may be preferable to lasso when predictors are strongly correlated, lasso selection is unstable, groups of related predictors may be useful, or high-dimensional biomedical features are present.

8. Tuning elastic net

A simple workflow is:

1. Define candidate alpha values.
2. Define candidate lambda values.
3. Use cross-validation in the training data.
4. Select tuning values.
5. Fit the final model in the training data.
6. Evaluate once on the untouched test data.

The test set should not be used repeatedly to choose alpha or lambda.

9. Nested cross-validation

When many tuning values are explored, nested cross-validation can be helpful. The inner loop tunes alpha and lambda, while the outer loop estimates model performance. This avoids using the same validation data for both model selection and performance estimation.

10. Medical examples

In cancer recurrence prediction using age, stage, grade, treatment and thousands of gene expression features, ridge may keep all gene features and lasso may select a small but unstable set. Elastic net may select a group of related genes while shrinking coefficients.

In a small diabetes model with only a few clinical predictors, elastic net may not always be necessary. It becomes more useful when the number of candidate predictors increases or predictors are strongly correlated.

11. Interpretation of selected features

Elastic net-selected features should be interpreted cautiously. They may be:

  • true biological signals;
  • proxies for other variables;
  • correlated with selected pathways;
  • artefacts of preprocessing;
  • dataset-specific patterns;
  • selected due to random variation.

12. Calibration and validation

Elastic net may reduce overfitting, but it does not guarantee good calibration. After fitting an elastic net model, assess discrimination, calibration plot, calibration slope, calibration intercept, Brier score and decision curve analysis if linked to action.

13. Common mistakes

  • Tuning alpha and lambda on the final test set.
  • Selecting features before validation.
  • Interpreting selected features as causal.
  • Reporting only AUC.
  • Ignoring calibration.
  • Ignoring feature stability.
  • Failing to externally validate high-dimensional models.

14. Reporting elastic net models

A clear report should describe:

  • outcome definition;
  • candidate predictors;
  • preprocessing;
  • standardisation;
  • alpha selection;
  • lambda selection;
  • validation design;
  • selected predictors;
  • discrimination;
  • calibration;
  • feature stability if relevant;
  • external validation if available.

15. Summary

Elastic net combines ridge and lasso. It can shrink coefficients and perform feature selection. It is especially useful when predictors are correlated, which is common in biomedical data. Like ridge and lasso, elastic net requires careful validation and cautious interpretation.

R practical: elastic net across alpha values

This script fits elastic net models with different alpha values and compares their performance for diabetes prediction. It also prints the selected features from the best model.

Test AUC

Selected features

# Machine Learning in Biostatistics
# Script 13: Elastic Net
#
# This script fits elastic net models with different alpha values
# and compares their performance for diabetes prediction.

library(tidyverse)
library(mlbench)
library(pROC)
library(glmnet)

set.seed(123)

data("PimaIndiansDiabetes")

diabetes_data <- PimaIndiansDiabetes %>%
  mutate(
    diabetes = factor(diabetes, levels = c("neg", "pos"))
  )

train_index <- sample(seq_len(nrow(diabetes_data)), size = round(0.80 * nrow(diabetes_data)))

train_data <- diabetes_data[train_index, ]
test_data <- diabetes_data[-train_index, ]

x_train <- model.matrix(
  diabetes ~ pregnant + glucose + pressure + triceps +
    insulin + mass + pedigree + age,
  data = train_data
)[, -1]

x_test <- model.matrix(
  diabetes ~ pregnant + glucose + pressure + triceps +
    insulin + mass + pedigree + age,
  data = test_data
)[, -1]

y_train <- ifelse(train_data$diabetes == "pos", 1, 0)
y_test_factor <- test_data$diabetes

calculate_auc <- function(actual, risk) {
  roc_object <- roc(
    response = actual,
    predictor = risk,
    levels = c("neg", "pos"),
    quiet = TRUE
  )

  as.numeric(auc(roc_object))
}

alpha_values <- c(0, 0.25, 0.50, 0.75, 1)

elastic_net_results <- map_dfr(alpha_values, function(current_alpha) {
  elastic_net_model <- cv.glmnet(
    x = x_train,
    y = y_train,
    family = "binomial",
    alpha = current_alpha
  )

  elastic_net_risk <- as.numeric(
    predict(
      elastic_net_model,
      newx = x_test,
      s = "lambda.min",
      type = "response"
    )
  )

  elastic_net_auc <- calculate_auc(y_test_factor, elastic_net_risk)

  tibble(
    alpha = current_alpha,
    lambda_min = elastic_net_model$lambda.min,
    test_auc = elastic_net_auc,
    model_object = list(elastic_net_model)
  )
})

cat("\nElastic net comparison across alpha values:\n")
print(
  elastic_net_results %>%
    select(alpha, lambda_min, test_auc)
)

best_elastic_net <- elastic_net_results %>%
  arrange(desc(test_auc)) %>%
  slice(1)

best_alpha <- best_elastic_net$alpha
best_model <- best_elastic_net$model_object[[1]]

best_coefficients <- coef(best_model, s = "lambda.min")

best_selected <- tibble(
  term = rownames(best_coefficients),
  coefficient = as.numeric(best_coefficients)
) %>%
  filter(coefficient != 0)

cat("\nBest alpha based on test AUC:\n")
print(best_alpha)

cat("\nFeatures selected by the best elastic net model:\n")
print(best_selected)

plot(
  elastic_net_results$alpha,
  elastic_net_results$test_auc,
  type = "b",
  xlab = "Alpha",
  ylab = "Test AUC",
  main = "Elastic Net: Alpha versus Test AUC"
)

cat("\nLearning points:\n")
cat("1. Elastic net combines ridge and lasso behaviour.\n")
cat("2. Alpha controls the balance between ridge and lasso.\n")
cat("3. Alpha = 0 gives ridge regression.\n")
cat("4. Alpha = 1 gives lasso regression.\n")
cat("5. Elastic net can be useful when predictors are correlated.\n")

Part C: Biomedical scenarios

Question 7

A gene expression dataset has 250 patients and 15,000 features. Many genes belong to related pathways. Why might elastic net be useful?

Show suggested answer

Elastic net may be useful because the dataset has many predictors and many genes may be correlated through biological pathways. Elastic net can shrink coefficients, select features and handle correlated predictors better than pure lasso.

Question 8

A model contains only age, BMI, glucose and blood pressure. Would elastic net always be necessary?

Show suggested answer

No. With only four clinically meaningful predictors, ordinary logistic regression or ridge may be sufficient. Elastic net becomes more useful when there are many candidate predictors or strong correlation with a need for feature selection.

Question 9

A methylation study includes neighbouring CpG sites that are highly correlated. Why might pure lasso selection be unstable?

Show suggested answer

Neighbouring CpG sites may contain similar information. Lasso may select one CpG site and remove others, but a small change in the sample could lead to a different CpG being selected.

Part D: Tuning and validation

Question 10

A student tried many alpha and lambda values and selected the combination with the best final test set AUC. What is wrong?

Show suggested answer

The final test set was used for model tuning. This means the test set became part of model development, so the final test AUC may be optimistic.

Question 11

Write a better workflow for tuning elastic net.

Split data into training and test sets. Use cross-validation inside the training data to choose alpha and lambda. Fit the final model on the training data. Evaluate once on the untouched test set.

Question 12

What is nested cross-validation, and why may it be useful for elastic net?

Show suggested answer

Nested cross-validation separates tuning from performance estimation. The inner loop tunes alpha and lambda. The outer loop estimates model performance. This is useful when many tuning combinations are tested.

Part E: Interpretation

Question 13

An elastic net model selected 25 genes for predicting cancer recurrence. Can we say these 25 genes cause recurrence?

Show suggested answer

No. Elastic net-selected genes are predictors in a model. They may be causal, correlated with causal factors, proxies, artefacts or selected by chance. Causality requires additional evidence.

Question 14

List four questions before making biological claims about elastic net-selected features.

Show suggested answer

Ask whether the features are stable, biologically plausible, externally validated, reliably measured, available at prediction time and not explained by batch effects.

Question 15

Why should calibration still be assessed after fitting an elastic net model?

Show suggested answer

Elastic net may improve stability, but it does not guarantee well-calibrated probabilities. Calibration must be checked because clinical decisions may depend on accurate risk estimates.
