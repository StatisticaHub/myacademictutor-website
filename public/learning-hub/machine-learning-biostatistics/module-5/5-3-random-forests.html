Lesson 18: Random Forests

A random forest builds many decision trees and averages their predictions. Instead of trusting one unstable tree, it combines many trees to improve stability and prediction performance.

Lecture

Lecture 18: Random Forests

Opening scene: Many imperfect trees

Curious Learner:
“A single tree is easy to understand, but it can be unstable. Can we build many trees instead?”

Prof Stat:
“Yes. That is the idea behind random forests.”

Dr Clinic:
“So instead of one clinical rule, we average many possible rules?”

Prof Stat:
“Exactly. Averaging many trees can reduce instability.”

Confusion Bug:
“So if glucose is important in the forest, that proves glucose causes the outcome?”

Prof Stat:
“No. Importance means useful for prediction in the model, not automatic causality.”

Today’s big question

How can averaging many trees improve clinical prediction?

Random forests improve stability by building many trees on slightly different data and averaging their predictions.

A single decision tree is sensitive to changes in the development data. A random forest reduces this sensitivity by combining many trees. Each tree may be imperfect, but the average prediction can be more stable.

Intuition first

A forest is more stable than one tree

Imagine asking one clinician for a risk estimate based on one decision pathway. Now imagine asking many clinicians, each considering a slightly different view of the data, and averaging their opinions.

The forest prediction is not the opinion of one tree. It is the average of many tree predictions. This averaging is why random forests often perform better than a single tree.

Bagging

Each tree sees a bootstrap sample

Random forests use bootstrap samples. A bootstrap sample is created by sampling patients from the training data with replacement. Some patients appear more than once. Some patients are left out.

Training data → bootstrap sample 1 → tree 1
Training data → bootstrap sample 2 → tree 2
Training data → bootstrap sample 3 → tree 3

This process is called bagging, short for bootstrap aggregation. It creates many related but different trees.

Random predictor selection

The trees are deliberately made different

Random forests add another source of randomness. At each split, the tree is not allowed to consider all predictors. It considers only a random subset of predictors.

This helps because if one predictor is very strong, many trees might otherwise make very similar splits. Random predictor selection makes the trees less correlated with each other.

Why this helps

Averaging works best when the trees make somewhat different errors. If all trees are nearly identical, averaging adds little benefit.

Out-of-bag error

Some patients are left out of each tree

Because each tree is trained on a bootstrap sample, some training patients are not used to build that particular tree. These are called out-of-bag patients for that tree.

The model can predict each patient using only trees for which that patient was out-of-bag. This gives an internal estimate of prediction error without needing a separate validation set.

Important caution

Out-of-bag error is useful internal validation, but it does not replace external validation. External data may differ in patient mix, measurement process, clinical practice or site effects.

Variable importance

Importance is predictive, not automatically causal

Random forests can produce variable importance measures. These measures describe which predictors were useful for prediction in the fitted model.

Reasonable interpretation

Glucose was important for prediction in this dataset and model.

Overclaim

Glucose importance proves a causal mechanism in this study.

Variable importance can be affected by correlation between predictors, measurement scale, missingness, noise, and dataset-specific patterns.

Detailed notes

Note 18: Random Forests

A random forest improves on a single decision tree by building many trees and averaging their predictions. This often improves stability, but interpretation and validation remain essential.

Learning objectives

  • Explain why a single decision tree can be unstable.
  • Describe the idea of an ensemble model.
  • Explain bootstrap sampling and bagging.
  • Describe random predictor selection at each split.
  • Explain why averaging many trees can reduce variance.
  • Understand out-of-bag error.
  • Interpret random forest variable importance cautiously.

1. Why this note matters

Decision trees are intuitive, but a single tree is often unstable. A small change in the dataset can change the first split and produce a different tree. This instability matters in medical prediction because we want predictions that are reliable for future patients, not only for the development sample.

Random forests address this problem by building many trees and averaging them. The result is usually more stable than a single tree, although it becomes less directly interpretable.

2. Ensemble learning

An ensemble model combines many individual models. A random forest is an ensemble of decision trees. Each tree gives a prediction, and the forest combines the predictions.

For binary clinical prediction, each tree can estimate a probability of the outcome. The random forest averages these probabilities across trees to produce a final predicted risk.

3. Bootstrap sampling

Each tree in a random forest is usually trained on a bootstrap sample of the training data. A bootstrap sample is created by sampling patients with replacement from the original training dataset.

Because sampling is with replacement, some patients appear multiple times in a bootstrap sample, while others are not included at all. This means each tree sees a slightly different version of the training data.

4. Bagging

Bagging means bootstrap aggregation. The method builds many models on bootstrap samples and then averages their predictions.

Bagging reduces variance. In simple terms, one tree may be highly sensitive to noise, but the average of many trees is usually less sensitive to any single noisy split.

5. Random predictor selection

Random forests are not just bagged trees. They also randomly select a subset of predictors at each split.

This matters because if all predictors are available at every split, a very strong predictor may dominate many trees. The trees may then become too similar. Random predictor selection makes trees more diverse, which improves the benefit of averaging.

6. Why averaging helps

A single tree can have high variance. High variance means that the fitted model changes substantially if the training data change slightly.

Random forests reduce variance by averaging many trees. If the trees make partly different errors, their average prediction is often more stable than the prediction from one tree.

7. Bias and variance intuition

A single deep tree may have low bias because it can fit complex patterns, but high variance because it is sensitive to the training data. Random forests keep much of the flexibility of trees while reducing variance through averaging.

This is why random forests often perform better than a single decision tree in prediction tasks.

8. Out-of-bag patients

For each bootstrap sample, some patients are left out. These are called out-of-bag patients for that tree.

A patient can be predicted using only the trees for which that patient was not included in the bootstrap training sample. This provides an internal estimate of prediction error called out-of-bag error.

9. Out-of-bag error is not external validation

Out-of-bag error is useful because it gives an internal validation estimate without needing a separate test set. However, it is still based on the same development dataset.

External validation remains important because real future data may differ in patient population, measurement systems, clinical practice, outcome definition or missing data patterns.

10. Important tuning choices

Random forests have several tuning choices:

  • number of trees;
  • number of predictors considered at each split;
  • minimum terminal node size;
  • maximum depth, depending on implementation;
  • sampling strategy if classes are imbalanced.

In many settings, increasing the number of trees stabilises the forest but increases computation. The number of predictors considered at each split affects how similar the trees are.

11. Random forests and non-linearity

Random forests can capture non-linear patterns because each tree uses splits rather than assuming a straight-line relationship.

For example, diabetes risk may rise sharply after a glucose threshold. A tree-based model can represent this without requiring the analyst to specify the threshold in advance.

12. Random forests and interactions

Random forests can also capture interactions. For example, BMI may be more predictive among patients with high glucose than among patients with low glucose.

Because each tree can split on different variables in different branches, interactions can arise naturally from the tree structure.

13. Variable importance

Random forests can rank predictors using variable importance measures. These measures can help identify which predictors the model used strongly.

However, importance is not the same as causality. A variable can be important because it is a proxy for another variable, because it is correlated with a causal factor, or because it reflects data collection patterns.

14. Correlated predictors and importance

If two predictors are highly correlated, importance can be shared or distorted. One predictor may appear less important because another correlated predictor already carries similar information.

In biomedical data, this is common. Related biomarkers, repeated clinical measurements, neighbouring genomic features and imaging variables may contain overlapping information.

15. Random forests and calibration

Random forests can have good discrimination but imperfect calibration. A model may rank patients well but produce risk estimates that are too high or too low.

Therefore, calibration plots, calibration slope, calibration intercept and Brier score remain important.

16. Random forests and clinical usefulness

A random forest should be evaluated in relation to the clinical decision it is intended to support. If the model is used to decide who receives additional screening, the relevant threshold probabilities and consequences of false positives and false negatives should be considered.

Decision curve analysis may be useful when the model is linked to a clinical action.

17. Strengths of random forests

  • They can model non-linear relationships.
  • They can capture interactions.
  • They reduce instability compared with a single tree.
  • They can handle mixed types of predictors.
  • They provide internal out-of-bag error estimates.
  • They provide variable importance summaries.

18. Limitations of random forests

  • They are less interpretable than a single tree.
  • Predicted probabilities may require calibration assessment.
  • Variable importance can be misleading.
  • They do not automatically solve leakage problems.
  • They can perform poorly if the development data are biased.
  • They still need external validation before strong clinical claims.

19. Medical example: hospital readmission

A random forest could be used to predict hospital readmission using age, prior admissions, medications, comorbidities, blood tests and vital signs.

The model may detect complex patterns, such as a combination of age, recent admission history and abnormal laboratory values. However, if hospital site or discharge coding practices are strong predictors, the model may learn system-level artefacts rather than patient-level clinical risk.

20. Medical example: diabetes prediction

In diabetes prediction, a random forest may use glucose, BMI, age, pedigree, insulin and other variables. Glucose may appear highly important, which is clinically plausible.

However, the correct interpretation is that glucose was useful for prediction in this model and dataset. The importance measure should not be written as proof of causality.

21. Common mistakes

  • Assuming random forests cannot overfit.
  • Reporting only AUC and ignoring calibration.
  • Treating variable importance as causal importance.
  • Ignoring correlated predictors when interpreting importance.
  • Assuming out-of-bag error replaces external validation.
  • Using predictors that are not available at the intended prediction time.

22. Reporting random forest models

A clear report should describe:

  • outcome definition;
  • prediction time point;
  • candidate predictors;
  • number of trees;
  • number of predictors considered at each split;
  • node-size settings;
  • validation strategy;
  • discrimination;
  • calibration;
  • variable importance method;
  • clinical interpretation and limitations.

23. Summary

A random forest is an ensemble of decision trees. It uses bootstrap samples and random predictor selection to create many different trees. The final prediction is an average across trees.

Random forests often improve stability compared with a single tree, but they are not automatically clinically useful. They still require honest validation, calibration assessment, cautious interpretation and external testing where possible.

# Machine Learning in Biostatistics
# Script 18: Random Forests
#
# This script fits a random forest for diabetes prediction.
# It compares a single decision tree with a random forest and
# demonstrates AUC, Brier score, variable importance and calibration.
#
# Main ideas:
# 1. A random forest averages many decision trees.
# 2. Bootstrap samples are used to build different trees.
# 3. Random predictor selection makes trees less similar.
# 4. Averaging many trees can reduce instability.
# 5. Variable importance is not causal importance.

library(tidyverse)
library(mlbench)
library(pROC)
library(rpart)
library(rpart.plot)
library(randomForest)

set.seed(123)

data("PimaIndiansDiabetes")

diabetes_data <- PimaIndiansDiabetes %>%
  mutate(
    diabetes = factor(diabetes, levels = c("neg", "pos"))
  )

cat("\nDataset summary:\n")
cat("Number of patients:", nrow(diabetes_data), "\n")
cat("Outcome distribution:\n")
print(table(diabetes_data$diabetes))

train_index <- sample(
  seq_len(nrow(diabetes_data)),
  size = round(0.80 * nrow(diabetes_data))
)

train_data <- diabetes_data[train_index, ]
test_data <- diabetes_data[-train_index, ]

cat("\nTraining patients:", nrow(train_data), "\n")
cat("Test patients:", nrow(test_data), "\n")

calculate_auc <- function(actual, risk) {
  roc_object <- roc(
    response = actual,
    predictor = risk,
    levels = c("neg", "pos"),
    quiet = TRUE
  )

  as.numeric(auc(roc_object))
}

calculate_brier <- function(actual, risk) {
  observed <- as.numeric(actual == "pos")
  mean((risk - observed)^2)
}

single_tree <- rpart(
  diabetes ~ pregnant + glucose + pressure + triceps +
    insulin + mass + pedigree + age,
  data = train_data,
  method = "class",
  control = rpart.control(
    cp = 0.01,
    maxdepth = 4,
    minsplit = 20
  )
)

single_tree_risk <- predict(
  single_tree,
  newdata = test_data,
  type = "prob"
)[, "pos"]

random_forest_model <- randomForest(
  diabetes ~ pregnant + glucose + pressure + triceps +
    insulin + mass + pedigree + age,
  data = train_data,
  ntree = 500,
  mtry = 3,
  importance = TRUE
)

forest_risk <- predict(
  random_forest_model,
  newdata = test_data,
  type = "prob"
)[, "pos"]

model_comparison <- tibble(
  model = c("Single decision tree", "Random forest"),
  test_auc = c(
    calculate_auc(test_data$diabetes, single_tree_risk),
    calculate_auc(test_data$diabetes, forest_risk)
  ),
  test_brier = c(
    calculate_brier(test_data$diabetes, single_tree_risk),
    calculate_brier(test_data$diabetes, forest_risk)
  )
)

cat("\nModel comparison:\n")
print(model_comparison)

cat("\nRandom forest model summary:\n")
print(random_forest_model)

cat("\nOut-of-bag error by number of trees is stored in the model object.\n")
cat("Final out-of-bag error estimate:\n")
print(random_forest_model$err.rate[nrow(random_forest_model$err.rate), ])

importance_table <- importance(random_forest_model) %>%
  as.data.frame() %>%
  rownames_to_column("predictor") %>%
  arrange(desc(MeanDecreaseGini))

cat("\nVariable importance table:\n")
print(importance_table)

varImpPlot(
  random_forest_model,
  main = "Random Forest Variable Importance"
)

forest_predictions <- test_data %>%
  mutate(
    predicted_risk = forest_risk,
    observed_numeric = as.numeric(diabetes == "pos")
  )

calibration_table <- forest_predictions %>%
  mutate(risk_group = ntile(predicted_risk, 5)) %>%
  group_by(risk_group) %>%
  summarise(
    mean_predicted_risk = mean(predicted_risk),
    observed_risk = mean(observed_numeric),
    n = n(),
    .groups = "drop"
  )

cat("\nGrouped calibration summary for random forest:\n")
print(calibration_table)

plot(
  calibration_table$mean_predicted_risk,
  calibration_table$observed_risk,
  xlim = c(0, 1),
  ylim = c(0, 1),
  xlab = "Mean predicted risk",
  ylab = "Observed risk",
  main = "Grouped Calibration: Random Forest"
)

abline(0, 1, lty = 2)

forest_roc <- roc(
  response = test_data$diabetes,
  predictor = forest_risk,
  levels = c("neg", "pos"),
  quiet = TRUE
)

plot(
  forest_roc,
  main = "Random Forest ROC Curve",
  legacy.axes = TRUE
)

rpart.plot(
  single_tree,
  type = 2,
  extra = 104,
  fallen.leaves = TRUE,
  main = "Single Decision Tree"
)

cat("\nLearning points:\n")
cat("1. A random forest averages predictions from many decision trees.\n")
cat("2. Bootstrap sampling makes each tree use a slightly different dataset.\n")
cat("3. Random predictor selection makes the trees less similar to each other.\n")
cat("4. Averaging can reduce instability compared with a single tree.\n")
cat("5. Variable importance should be interpreted as predictive importance, not causal importance.\n")
cat("6. Random forests still need calibration, clinical usefulness assessment and external validation.\n")
