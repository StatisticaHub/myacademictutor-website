Lesson 19: Gradient Boosting

Gradient boosting builds trees sequentially. Each new tree makes a small correction to the current model. This can produce strong clinical prediction models, but tuning, calibration and validation are essential.

How does gradient boosting build a strong model from many small trees?

Gradient boosting creates a strong prediction model by adding weak learners one at a time, with each learner correcting the current model.

Boosting is powerful because it is tunable

Gradient boosting has several tuning choices. These choices determine whether the model is too simple, appropriately flexible, or overfitted.

Detailed notes

Note 19: Gradient Boosting

Gradient boosting builds a strong prediction model by adding small trees sequentially. It can be powerful, but it requires careful tuning, validation and calibration assessment.

1. Why this note matters

Gradient boosting is one of the most important machine learning methods for prediction. It is popular because it can learn complex relationships, handle non-linear effects and capture interactions between predictors.

In biomedical data, relationships are rarely perfectly linear. For example, diabetes risk may change sharply at high glucose values, and the effect of BMI may differ by age or glucose level. Boosting can learn such patterns without the analyst specifying every interaction in advance.

However, this flexibility also creates risk. A boosted model can adapt strongly to the development dataset. Without careful tuning and validation, it may learn noise rather than stable clinical signal.

2. Boosting versus random forests

Random forests build many trees mostly independently and then average their predictions. Gradient boosting is different because trees are built sequentially.

In a random forest, each tree is like a separate opinion. In boosting, each tree is like a correction to the previous opinion. This makes boosting more directed and often more powerful, but also more sensitive to tuning choices.

3. The additive model idea

A boosted model is additive. It starts with an initial prediction and then adds many small trees. Each tree contributes a small adjustment.

The final model can be understood as a sum of many small corrections. This is why a single tree in a boosting model is usually not interpreted on its own. The whole ensemble produces the prediction.

4. Weak learners

A weak learner is a simple model that captures limited structure. In gradient boosting, weak learners are often shallow trees.

One shallow tree may not be very accurate. But many shallow trees, added carefully, can create a strong model. This is the central strength of boosting.

5. Loss function intuition

A loss function measures how wrong predictions are. For binary outcomes, the loss function penalises predicted probabilities that do not match the observed outcome.

Gradient boosting adds a new tree in a direction that reduces the loss. You do not need to focus on the technical calculus here. The practical idea is that each tree tries to improve the current model.

6. Learning rate

The learning rate controls the size of each correction. A smaller learning rate means each tree has a smaller effect. This usually requires more trees, but can produce smoother learning.

A larger learning rate makes each tree more influential. This can learn quickly, but may overfit quickly or skip over better solutions.

7. Number of trees

The number of trees controls how many corrections are added. Too few trees may underfit because the model has not learned enough. Too many trees may overfit because the model begins to learn random noise.

In practice, the number of trees is usually chosen using cross-validation or a validation set. The final test set should not be used repeatedly to decide this value.

8. Tree depth

Tree depth controls how complex each individual tree can be. A depth of 1 allows simple split-based corrections. Larger depths allow interactions.

In clinical prediction, deeper trees may learn complex patient subgroups. Some of these may be real, but some may be unstable. This is why depth should be tuned carefully.

9. Minimum node size

Minimum node size controls how small terminal nodes can become. If node sizes are too small, the model may learn very specific subgroup patterns based on few patients.

In medical data, small unstable subgroups are risky because they may not appear in future patients.

10. Overfitting in boosting

Boosting can overfit when the model continues adding corrections after the stable signal has been captured. The training error may keep decreasing, but test error may start increasing.

This is why a performance curve is useful. The best model is not necessarily the model with the lowest training error. It is the model that performs best on validation data and generalises to new patients.

11. Early stopping

Early stopping means stopping the boosting process when validation performance no longer improves.

Early stopping is a practical way to prevent too many trees from being added. It should be based on appropriate validation data, not repeated use of the final test set.

12. Internal cross-validation

Internal cross-validation can be used to choose the number of trees and other tuning values inside the training data.

This keeps the final test set untouched. The test set should only be used once at the end for a more honest performance estimate.

13. Boosting and clinical interpretation

Boosted models are usually less interpretable than single decision trees. The final prediction comes from many trees, not one simple pathway.

Interpretation tools such as variable influence, partial dependence and calibration plots can help, but they do not make the model automatically clinically valid.

14. Variable influence

Boosting algorithms often provide relative influence measures. These indicate which predictors contributed strongly to the model’s prediction process.

Variable influence should not be interpreted as causality. A variable may be influential because it is a proxy, correlated with another variable, affected by measurement practices or specific to the development dataset.

15. Calibration

Boosted models can produce predicted probabilities. These probabilities must be checked. A model can have strong AUC but still give probabilities that are too high or too low.

Calibration matters because clinical decisions often use absolute risk thresholds. For example, a patient predicted to have 30% risk may be treated differently from a patient predicted to have 5% risk.

16. Brier score

The Brier score measures the average squared difference between predicted probabilities and observed outcomes.

It complements AUC because AUC measures ranking, while Brier score reflects probability prediction accuracy.

17. Decision curve analysis

If a boosted model is intended to support a clinical action, decision curve analysis can be used to examine clinical usefulness across threshold probabilities.

A boosted model should not be called clinically useful only because its AUC is high. Usefulness depends on the consequences of decisions.

18. External validation

External validation is especially important for flexible models. A boosted model may adapt to local measurement practices, patient mix or coding patterns.

External validation tests whether performance holds in another setting.

19. Strengths of gradient boosting

  • It can model non-linear relationships.
  • It can capture interactions.
  • It often gives strong predictive performance.
  • It can use many predictors.
  • It provides variable influence summaries.

20. Limitations of gradient boosting

  • It requires careful tuning.
  • It can overfit.
  • It is less transparent than a single tree.
  • Variable influence does not prove causality.
  • Calibration may be imperfect.
  • External validation is still needed.

21. Reporting gradient boosting models

A clear report should describe:

  • outcome definition;
  • prediction time point;
  • candidate predictors;
  • number of trees;
  • learning rate;
  • tree depth;
  • minimum node size;
  • tuning procedure;
  • validation design;
  • discrimination;
  • calibration;
  • Brier score;
  • clinical usefulness if relevant;
  • external validation if available;
  • limitations.

22. Common mistakes

  • Assuming boosting is automatically better than simpler models.
  • Tuning repeatedly on the final test set.
  • Reporting only AUC.
  • Ignoring calibration.
  • Using too many trees without validation.
  • Interpreting variable influence as causality.
  • Claiming clinical usefulness without decision-based evaluation.

23. Summary

Gradient boosting builds a prediction model sequentially. Each new tree improves the current model by targeting remaining errors.

This makes boosting powerful, but also sensitive to tuning choices. In clinical prediction, boosted models require honest validation, calibration assessment and cautious interpretation.

# Machine Learning in Biostatistics
# Script 19: Gradient Boosting
#
# This script fits a gradient boosting model for diabetes prediction.
# It compares gradient boosting with logistic regression and shows
# AUC, Brier score, relative influence and grouped calibration.
#
# Main ideas:
# 1. Gradient boosting builds trees sequentially.
# 2. Each new tree tries to improve the current model.
# 3. Learning rate controls how strongly each new tree contributes.
# 4. Tree depth controls how complex each tree can be.
# 5. Boosting can overfit if tuning is careless.

library(tidyverse)
library(mlbench)
library(pROC)
library(gbm)

set.seed(123)

data("PimaIndiansDiabetes")

diabetes_data <- PimaIndiansDiabetes %>%
  mutate(
    diabetes = factor(diabetes, levels = c("neg", "pos")),
    diabetes_numeric = ifelse(diabetes == "pos", 1, 0)
  )

cat("\nDataset summary:\n")
cat("Number of patients:", nrow(diabetes_data), "\n")
cat("Outcome distribution:\n")
print(table(diabetes_data$diabetes))

train_index <- sample(
  seq_len(nrow(diabetes_data)),
  size = round(0.80 * nrow(diabetes_data))
)

train_data <- diabetes_data[train_index, ]
test_data <- diabetes_data[-train_index, ]

cat("\nTraining patients:", nrow(train_data), "\n")
cat("Test patients:", nrow(test_data), "\n")

calculate_auc <- function(actual, risk) {
  roc_object <- roc(
    response = actual,
    predictor = risk,
    levels = c("neg", "pos"),
    quiet = TRUE
  )

  as.numeric(auc(roc_object))
}

calculate_brier <- function(actual, risk) {
  observed <- as.numeric(actual == "pos")
  mean((risk - observed)^2)
}

boost_model <- gbm(
  diabetes_numeric ~ pregnant + glucose + pressure + triceps +
    insulin + mass + pedigree + age,
  data = train_data,
  distribution = "bernoulli",
  n.trees = 1500,
  interaction.depth = 2,
  shrinkage = 0.01,
  n.minobsinnode = 10,
  cv.folds = 5,
  verbose = FALSE
)

best_trees <- gbm.perf(
  boost_model,
  method = "cv",
  plot.it = TRUE
)

cat("\nBest number of trees selected using internal cross-validation:\n")
print(best_trees)

boost_risk <- predict(
  boost_model,
  newdata = test_data,
  n.trees = best_trees,
  type = "response"
)

logistic_model <- glm(
  diabetes ~ pregnant + glucose + pressure + triceps +
    insulin + mass + pedigree + age,
  data = train_data,
  family = binomial
)

logistic_risk <- predict(
  logistic_model,
  newdata = test_data,
  type = "response"
)

model_comparison <- tibble(
  model = c("Gradient boosting", "Logistic regression"),
  test_auc = c(
    calculate_auc(test_data$diabetes, boost_risk),
    calculate_auc(test_data$diabetes, logistic_risk)
  ),
  test_brier = c(
    calculate_brier(test_data$diabetes, boost_risk),
    calculate_brier(test_data$diabetes, logistic_risk)
  )
)

cat("\nModel comparison:\n")
print(model_comparison)

relative_influence <- summary(
  boost_model,
  n.trees = best_trees,
  plotit = FALSE
)

cat("\nRelative influence table:\n")
print(relative_influence)

boost_predictions <- test_data %>%
  mutate(
    predicted_risk = boost_risk,
    observed_numeric = as.numeric(diabetes == "pos")
  )

calibration_table <- boost_predictions %>%
  mutate(risk_group = ntile(predicted_risk, 5)) %>%
  group_by(risk_group) %>%
  summarise(
    mean_predicted_risk = mean(predicted_risk),
    observed_risk = mean(observed_numeric),
    n = n(),
    .groups = "drop"
  )

cat("\nGrouped calibration summary for gradient boosting:\n")
print(calibration_table)

plot(
  calibration_table$mean_predicted_risk,
  calibration_table$observed_risk,
  xlim = c(0, 1),
  ylim = c(0, 1),
  xlab = "Mean predicted risk",
  ylab = "Observed risk",
  main = "Grouped Calibration: Gradient Boosting"
)

abline(0, 1, lty = 2)

boost_roc <- roc(
  response = test_data$diabetes,
  predictor = boost_risk,
  levels = c("neg", "pos"),
  quiet = TRUE
)

plot(
  boost_roc,
  main = "Gradient Boosting ROC Curve",
  legacy.axes = TRUE
)

cat("\nLearning points:\n")
cat("1. Gradient boosting builds trees sequentially rather than independently.\n")
cat("2. Each new tree improves the current model by focusing on remaining errors.\n")
cat("3. Important tuning choices include number of trees, tree depth and learning rate.\n")
cat("4. Internal cross-validation can help choose the number of trees.\n")
cat("5. Boosting can overfit if too many trees or too much complexity are used.\n")
cat("6. A boosted model still needs calibration and clinical usefulness assessment.\n")
