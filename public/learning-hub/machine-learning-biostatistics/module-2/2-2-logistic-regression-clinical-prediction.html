Today’s big question

Why is logistic regression still so useful for clinical prediction?

Logistic regression is used when the outcome is binary.

Examples:

  • Diabetes: yes/no
  • Cancer recurrence: yes/no
  • Hospital readmission: yes/no
  • Treatment response: yes/no
  • Mortality: yes/no

It estimates the probability of the positive outcome.

Intuition first

For a binary outcome, we want probabilities between 0 and 1

For a binary outcome, we do not want predictions like:

Diabetes = 1.8

That would not make sense.

Instead, we want a probability between 0 and 1.

Predicted diabetes risk = 0.72

This means the model estimates a 72% probability that the patient is diabetes positive.

Logistic regression turns patient information into an estimated probability of a binary outcome.

The model idea

Logistic regression models the log-odds of the outcome

Conceptually:

logit(P(outcome positive)) =
β0 + β1X1 + β2X2 + β3X3 + ...

For diabetes prediction, this might look like:

logit(P(diabetes positive)) =
β0 + β1(glucose) + β2(BMI) + β3(age)

The model then converts the log-odds into a predicted probability.

Does the full model predict well for new patients?

That means we need to evaluate:

  • discrimination;
  • calibration;
  • sensitivity;

Odds ratios versus predicted probabilities

Two useful but different perspectives

Logistic regression coefficients are often converted to odds ratios.

Odds ratios are useful for interpretation.

But for prediction, predicted probabilities are often more directly useful.

Coefficient interpretation

This may be statistically meaningful.

Clinical prediction

Predicted diabetes risk = 72%

Threshold choice

A threshold converts probabilities into predicted classes

Commonly, people start with:

Threshold = 0.50

But 0.50 is not automatically correct.

For screening, a lower threshold may be preferred because missing true disease cases can be harmful.

For confirmatory decisions, a higher threshold may be preferred to avoid too many false positives.

Clinical reality check

A threshold is not just a statistical choice. It reflects clinical consequences.

Ask:

  • What happens if we miss a true case?
  • What happens if we create a false positive?
  • Is the model being used for screening, diagnosis, triage, or treatment decisions?

Common mistakes

Two common mistakes in logistic regression prediction

Common mistake: treating 0.50 as the automatic threshold

A threshold of 0.50 may be reasonable in some settings, but it is not universal.

In a screening setting, we may choose a lower threshold to improve sensitivity.

In a treatment decision with serious side effects, we may choose a higher threshold.

Threshold choice should depend on clinical consequences, not habit.

Common mistake: judging the model only by p-values

A prediction model is not evaluated only by whether predictors are statistically significant.

A model can contain statistically significant predictors and still perform poorly for prediction.

For clinical prediction, we need to evaluate the whole model using validation, discrimination, calibration, and clinical usefulness.

Why logistic regression is a strong baseline

A serious benchmark before complex models

Logistic regression is often a good first model because it is:

  • interpretable;
  • familiar in medical statistics;
  • suitable for binary outcomes;
  • able to produce predicted probabilities;
  • easier to validate and explain than many black-box models;
  • useful as a benchmark before trying more complex methods.

A complex machine learning model should usually be compared against a logistic regression baseline.

If the complex model does not improve meaningful performance, it may not be worth the added complexity.

Note 4: Logistic Regression for Clinical Prediction

Logistic regression is one of the most important models in medical statistics, epidemiology, and clinical prediction modelling. Although it is often taught as a classical statistical method, it is also a supervised learning method when used to predict a binary outcome.

1. Introduction

Many medical outcomes are binary. A patient may develop diabetes or not. A patient may be readmitted within 30 days or not. A tumour may recur within 5 years or not. A patient may respond to treatment or not. A participant may be classified as a case or control. These are all examples where logistic regression can be used.

In machine learning courses, logistic regression is sometimes treated as a simple baseline model before moving to random forests, support vector machines, boosting, or neural networks. In biostatistics, however, logistic regression deserves deeper attention. It is interpretable, clinically familiar, statistically well understood, and often performs surprisingly well for structured tabular medical data.

2. Where logistic regression fits in supervised learning

Logistic regression is used when the outcome is binary.

Y = 0 or 1

where:

  • Y = 1 means the event occurs;
  • Y = 0 means the event does not occur.
Clinical questionOutcome coded as 1Outcome coded as 0
Diabetes predictionDeveloped diabetesDid not develop diabetes
Readmission predictionReadmitted within 30 daysNot readmitted
Cancer recurrence predictionRecurrence within 5 yearsNo recurrence
Treatment responseRespondedDid not respond
Disease classificationCaseControl

The model uses predictors such as age, sex, BMI, blood pressure, biomarkers, tumour features, or methylation markers to estimate the probability that Y = 1.

P(Y = 1 | X)

This means the probability that the event occurs, given the predictors.

3. Why not use ordinary linear regression?

Ordinary linear regression is not ideal for binary outcomes because it can produce predicted values below 0 or above 1. But probabilities must lie between 0 and 1.

Predicted probability = 1.18 Predicted probability = -0.12

These values are not meaningful probabilities.

Logistic regression handles this by modelling the log-odds of the outcome rather than the probability directly.

4. Probability, odds, and log-odds

Probability

The chance that an event occurs. If the probability is 0.20, this means 20%.

Odds

Odds = probability / (1 - probability). If probability is 0.20, odds = 0.25.

Log-odds

Logistic regression models the log of the odds, also called the logit transformation.

Odds = p / (1 - p) logit(p) = log(p / (1 - p))

5. The logistic regression model

logit(p) = β0 + β1X1 + β2X2 + ... + βpXp

where:

  • p is the probability of the outcome;
  • β0 is the intercept;
  • β1, β2, ..., βp are regression coefficients;
  • X1, X2, ..., Xp are predictors.

For example:

logit(p) = β0 + β1(age) + β2(BMI) + β3(HbA1c)

The model is linear on the log-odds scale, not on the probability scale.

After estimating the log-odds, we can convert back to probability using:

p = exp(logit) / (1 + exp(logit))

6. Interpreting coefficients

If a coefficient is positive, higher values of the predictor are associated with higher odds of the outcome. If a coefficient is negative, higher values of the predictor are associated with lower odds of the outcome.

β_age = 0.04 Odds ratio = exp(β) = exp(0.04) = 1.04

This means each one-year increase in age is associated with approximately 4% higher odds of the outcome, assuming other predictors are held constant.

7. Odds ratios: useful but easy to misunderstand

An odds ratio is not the same as a risk ratio, especially when the outcome is common. For rare outcomes, odds ratios and risk ratios may be similar. For common outcomes, odds ratios can exaggerate the impression of effect size compared with risk ratios.

In prediction, odds ratios may be less central because the main goal is not necessarily to interpret each coefficient. Instead, we often care about predicted risks, discrimination, calibration, and clinical usefulness.

8. Logistic regression for inference vs prediction

Inference

Is BMI associated with diabetes?

The focus is the coefficient, odds ratio, confidence interval, p-value and confounding control.

Prediction

Can we predict diabetes risk?

The focus is predicted probabilities, discrimination, calibration, clinical usefulness and validation.

9. Breast cancer prediction example

Logistic regression can be relevant if a survival question is converted into a fixed-time binary prediction problem, such as death within 5 years or recurrence within 5 years.

However, if patients have different follow-up times or censoring is present, converting survival data into a binary outcome may discard information or introduce bias. In that case, Cox models or survival machine learning methods may be more appropriate.

The model should match the outcome structure.

10. EWAS and methylation example

Methylation data are high-dimensional. A dataset may contain hundreds of thousands of CpG sites and far fewer participants.

n = 800 participants p = 450,000 CpG sites

A standard logistic regression model cannot be fitted safely with so many predictors. Penalised logistic regression methods such as lasso or elastic net are more appropriate because they shrink coefficients and can perform variable selection.

Even then, feature selection must be done inside the resampling process to avoid data leakage.

11. Choosing predictors in logistic regression

In clinical prediction, predictor choice should be guided by the intended use of the model.

A predictor should generally be:

  • available at the prediction time point;
  • measured reliably;
  • clinically meaningful or operationally available;
  • not a consequence of the outcome;
  • not a leakage variable;
  • feasible to collect in the intended setting.

12. Events per predictor and sample size

Logistic regression can overfit when there are too many predictors relative to the number of outcome events.

Total patients = 200 Outcome events = 20 Candidate predictors = 30

This is a risky setting for ordinary logistic regression.

Possible solutions include:

  • reducing the number of predictors;
  • using penalised logistic regression;
  • increasing sample size;
  • using clinical knowledge to pre-specify predictors;
  • applying internal validation;
  • avoiding data-driven variable selection without validation.

13. Separation and unstable estimates

Separation occurs when a predictor or combination of predictors perfectly separates outcome groups. The model may try to estimate an infinitely large coefficient.

This can lead to:

  • very large standard errors;
  • unstable coefficients;
  • convergence warnings;
  • extreme predicted probabilities;
  • poor generalisation.

Possible solutions include penalised logistic regression, Firth correction, reducing model complexity, or collecting more data.

14. Logistic regression and calibration

Logistic regression often produces reasonably calibrated probabilities when the model is well specified and validated. However, it is not automatically well calibrated.

Poor calibration can occur if:

  • important predictors are missing;
  • relationships are nonlinear but modelled as linear;
  • interactions are ignored;
  • the model is overfitted;
  • the model is applied to a different population;
  • outcome prevalence differs between development and validation settings.

15. Logistic regression and discrimination

Discrimination measures how well the model separates patients who experience the outcome from those who do not. For binary outcomes, discrimination is commonly assessed using the ROC curve and AUC.

AUC = 0.82

This suggests that the model has good ability to rank patients by risk, but AUC alone is not enough.

16. Model specification: linearity and interactions

Standard logistic regression assumes that continuous predictors have a linear relationship with the log-odds of the outcome. This may not always be realistic.

Possible ways to handle nonlinear effects include:

  • transforming variables;
  • using splines;
  • categorising variables carefully, though this often loses information;
  • using more flexible models;
  • including clinically justified interaction terms.

17. Variable selection: be careful

Stepwise variable selection can lead to:

  • unstable predictor sets;
  • biased coefficient estimates;
  • overfitting;
  • optimistic performance estimates;
  • poor reproducibility;
  • exaggerated importance of selected predictors.

18. Logistic regression vs machine learning models

Complex models may sometimes outperform logistic regression, especially when there are nonlinear relationships or complex interactions. However, complex models are not automatically better.

Do not use a more complex model unless it improves clinically meaningful performance after proper validation.

19. Reporting a logistic regression prediction model

A report should include:

  • clinical prediction question;
  • target population;
  • prediction time point;
  • outcome definition;
  • candidate predictors;
  • missing data handling;
  • sample size and number of events;
  • model specification;
  • validation strategy;
  • discrimination;
  • calibration;
  • clinical usefulness where relevant;
  • limitations;
  • model equation or sufficient details for implementation.

20. Practical workflow

  1. Define the clinical prediction question.
  2. Define the target population.
  3. Define the prediction time point.
  4. Define the binary outcome clearly.
  5. Identify candidate predictors available at prediction time.
  6. Explore missing data and data quality.
  7. Split data into training and test sets or define resampling strategy.
  8. Fit logistic regression as a baseline model.
  9. Generate predicted probabilities.
  10. Evaluate discrimination using ROC AUC.
  11. Evaluate calibration using calibration plots.
  12. Examine sensitivity and specificity at clinically relevant thresholds.
  13. Compare with more complex models only after the baseline is understood.
  14. Report the model transparently.

21. Summary

Logistic regression is a foundational model for binary clinical prediction. It estimates the probability of an outcome using a linear model on the log-odds scale.

For prediction, logistic regression should be evaluated using discrimination, calibration, sensitivity, specificity, predictive values, and validation on new data. Accuracy alone is not enough.

Logistic regression is not just an old statistical method. In biostatistics, it is a strong, interpretable, and clinically meaningful baseline for supervised machine learning.

# Machine Learning in Biostatistics
# Script 04: Logistic Regression for Clinical Prediction
#
# This script uses logistic regression as a clinical prediction model.
# The focus is on predicted probabilities, odds ratios, and test-set performance.

library(tidyverse)
library(mlbench)
library(pROC)
library(broom)

set.seed(123)

data("PimaIndiansDiabetes")

diabetes_data <- PimaIndiansDiabetes %>%
  mutate(
    diabetes = factor(diabetes, levels = c("neg", "pos"))
  )

train_index <- sample(seq_len(nrow(diabetes_data)), size = round(0.80 * nrow(diabetes_data)))

train_data <- diabetes_data[train_index, ]
test_data <- diabetes_data[-train_index, ]

logistic_model <- glm(
  diabetes ~ pregnant + glucose + pressure + triceps +
    insulin + mass + pedigree + age,
  data = train_data,
  family = binomial
)

cat("\nLogistic regression coefficients:\n")
print(tidy(logistic_model))

odds_ratio_table <- tidy(logistic_model) %>%
  mutate(
    odds_ratio = exp(estimate),
    conf_low = exp(estimate - 1.96 * std.error),
    conf_high = exp(estimate + 1.96 * std.error)
  ) %>%
  select(term, estimate, odds_ratio, conf_low, conf_high, p.value)

cat("\nOdds ratio table:\n")
print(odds_ratio_table)

test_data <- test_data %>%
  mutate(
    predicted_risk = predict(logistic_model, newdata = test_data, type = "response")
  )

cat("\nPredicted risks for first 10 test patients:\n")
print(
  test_data %>%
    select(diabetes, glucose, mass, age, predicted_risk) %>%
    head(10)
)

risk_summary <- test_data %>%
  summarise(
    min_risk = min(predicted_risk),
    median_risk = median(predicted_risk),
    mean_risk = mean(predicted_risk),
    max_risk = max(predicted_risk)
  )

cat("\nPredicted risk summary:\n")
print(risk_summary)

threshold <- 0.50

test_data <- test_data %>%
  mutate(
    predicted_class = factor(
      ifelse(predicted_risk >= threshold, "pos", "neg"),
      levels = c("neg", "pos")
    )
  )

cat("\nConfusion matrix:\n")
print(table(Predicted = test_data$predicted_class, Actual = test_data$diabetes))

roc_object <- roc(
  response = test_data$diabetes,
  predictor = test_data$predicted_risk,
  levels = c("neg", "pos"),
  quiet = TRUE
)

cat("\nTest AUC:\n")
print(auc(roc_object))

plot(
  roc_object,
  lwd = 2,
  main = paste0("ROC Curve: Logistic Regression, AUC = ", round(auc(roc_object), 3))
)

abline(a = 0, b = 1, lty = 2)

cat("\nClinical interpretation prompts:\n")
cat("1. Which predictors have clinically sensible directions?\n")
cat("2. Does a predictor being statistically significant prove good prediction? No.\n")
cat("3. Are predicted probabilities useful for risk stratification?\n")
cat("4. Should threshold 0.50 always be used? No.\n")
cat("5. Would this model need external validation? Yes.\n")

Appropriate predictors may include age at diagnosis, tumour stage, tumour grade, treatment type if known at prediction time, and baseline biomarker level.

Leakage risks include recurrence status during follow-up and end-of-life care referral. These occur after baseline and are closely related to the outcome.

The predictor list should include only variables available at the defined prediction time point. The model should clearly state whether prediction is made at diagnosis, after treatment planning, or at another time.
