Feature Selection in Biomedical Data

Big idea

Feature selection means choosing a subset of predictors for a model.

In biomedical machine learning, feature selection is common because datasets may contain many clinical variables, biomarkers, imaging features, genes, methylation sites, proteins, or metabolites.

Feature selection can make a model simpler.

But if done incorrectly, it can create serious data leakage.

Dr. Clinic: So the AUC may be inflated?

1. Why feature selection is used

Feature selection may be used to:

  • reduce model complexity;
  • remove noisy predictors;
  • reduce measurement burden;
  • make the model easier to explain;
  • identify candidate biomarkers;
  • improve practicality;
  • create a smaller prediction model.

In medicine, fewer predictors can be helpful.

A model using routinely available predictors is often easier to implement than a model requiring expensive specialised tests.

2. Clinical feature selection

Clinical feature selection uses medical knowledge.

For example, a diabetes prediction model may include:

  • glucose;
  • BMI;
  • age;
  • family history;
  • blood pressure.

These predictors are clinically sensible and usually available.

Clinical selection helps ensure that predictors are meaningful and available at the intended prediction time.

3. Statistical feature selection

Statistical feature selection uses the data.

Examples include:

  • selecting predictors with low p-values;
  • selecting predictors most correlated with the outcome;
  • stepwise regression;
  • lasso selection;
  • random forest importance;
  • recursive feature elimination.

These methods may be useful, but they must be validated carefully.

4. Model-based feature selection

Model-based feature selection uses a fitted model to identify useful predictors.

Examples:

Lasso selected predictors
Random forest variable importance
Gradient boosting feature importance
Recursive feature elimination

These approaches identify features that help prediction in a specific dataset.

They do not automatically identify causal variables.

5. Feature-selection leakage

Feature-selection leakage happens when validation or test data influence predictor selection.

Incorrect workflow:

1. Use full dataset to select features.
2. Split into train and test sets.
3. Fit model on training data.
4. Evaluate on test data.

The problem is that the test set helped choose the features.

So the test set is no longer independent.

Performance may look better than it really is.

6. Correct workflow

A safer workflow is:

1. Split data into training and test sets.
2. Select features using training data only.
3. Fit the model using training data only.
4. Evaluate once on the untouched test set.

For cross-validation:

Feature selection must happen inside each training fold.

This keeps validation honest.

Learner: But selecting features before validation feels easier.

Prof. Stat: Exactly.

7. Why biomedical data are risky

Biomedical datasets can contain thousands of predictors.

For example:

300 patients
450,000 methylation sites

With so many features, some will appear associated with the outcome by chance.

If we select features using the full dataset before validation, the model may look excellent but fail externally.

This is one of the most common problems in biomedical prediction.

8. Batch effects

Batch effects are technical differences unrelated to biology.

Examples:

  • lab batch;
  • sequencing run;
  • methylation array plate;
  • sample processing date;
  • scanner type;
  • technician;
  • hospital site.

If batch is related to outcome, feature selection may pick technical artefacts rather than biological signal.

This can produce poor generalisation.

9. Feature stability

A useful selected feature should ideally be stable.

Ask:

Is this feature selected across different resamples?

If selected features change dramatically across folds or bootstrap samples, interpretation should be cautious.

The model may still predict, but the specific feature list may not be reliable.

10. Correlated predictors

Biomedical predictors are often correlated.

Examples:

  • genes in the same pathway;
  • nearby methylation sites;
  • related blood biomarkers;
  • imaging features from the same region.

When predictors are correlated, feature selection may choose one feature from a group and ignore the others.

Another dataset may select a different feature from the same group.

This makes biological interpretation difficult.

11. Prediction versus discovery

Feature selection for prediction is not the same as biological discovery.

A selected feature may be:

  • causally important;
  • correlated with a causal factor;
  • a proxy for disease severity;
  • a technical artefact;
  • selected by chance;
  • specific to one dataset.

Always separate the prediction question from the discovery question.

12. Medical example: methylation prediction

Suppose we predict exposure status using DNA methylation.

The dataset contains:

300 participants
450,000 CpG sites

A poor workflow is:

Select top 100 CpG sites using full data
Cross-validate classifier
Report AUC

A better workflow is:

Create folds first
Select CpG sites inside each training fold
Fit model inside each training fold
Evaluate on held-out fold

This avoids feature-selection leakage.

13. Practical checklist

Before trusting selected features, ask:

  • Were features selected inside validation?
  • Was the final test set untouched?
  • Are selected features stable?
  • Are they clinically or biologically plausible?
  • Are they measured reliably?
  • Are they available at prediction time?
  • Could batch effects explain them?
  • Do they validate externally?
  • Do they improve calibration or only AUC?
  • Are the claims appropriately cautious?

Common mistake

Do not select biomarkers using the full dataset before cross-validation. This leaks outcome information into validation and can seriously inflate performance.

Key takeaway

Feature selection can make biomedical models simpler and more practical, but it must be performed inside validation. Selected features should be interpreted as prediction features first, not automatic biological discoveries.
