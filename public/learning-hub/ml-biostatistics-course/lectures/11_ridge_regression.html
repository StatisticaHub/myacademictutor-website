Ridge Regression

Big idea

Ridge regression is a regularisation method.

It helps when an ordinary regression model becomes too unstable or too confident.

Instead of allowing coefficients to become very large, ridge regression shrinks them towards zero.

In medical prediction, this can help reduce overfitting and improve performance on new patients.

Scene: the model is too confident

Learner: I fitted a logistic regression model, but the coefficients look unstable. Some are very large.

Prof. Stat: That can happen when the dataset is small, predictors are correlated, or there are too many predictors for the number of events.

Dr. Clinic: Clinically, that worries me. A model that changes too much may not be reliable.

Prof. Stat: Exactly. Ridge regression can help by shrinking the coefficients and making the model more stable.

1. Why ordinary regression may overfit

Ordinary logistic regression tries to fit the training data as well as possible.

This is useful when the data are strong enough to support the model.

But in medical prediction, we often have:

  • limited sample size;
  • rare outcomes;
  • many candidate predictors;
  • correlated clinical variables;
  • noisy biomarkers;
  • missing data;
  • local hospital-specific patterns.

In this situation, ordinary regression may learn random patterns in the development data.

This is overfitting.

2. What overfitting looks like

An overfitted model often looks excellent in the training data.

For example:

Training AUC = 0.91
Test AUC = 0.73
External validation AUC = 0.65

The training result looks impressive, but the model does not generalise well.

This is not just a technical problem.

In medical prediction, overfitting can make a model appear clinically useful when it may fail on future patients.

3. What ridge regression does

Ridge regression adds a penalty to the model.

The model is still trying to predict the outcome, but it is also discouraged from using very large coefficients.

Ordinary regression says:

Fit the training data as well as possible.

Ridge regression says:

Fit the training data well, but avoid overly large coefficients.

This usually makes the model more stable.

4. Shrinkage

Shrinkage means pulling coefficients towards zero.

Suppose ordinary logistic regression gives:

glucose coefficient = 1.20
BMI coefficient = 0.75
age coefficient = 0.45

Ridge regression may shrink them to:

glucose coefficient = 0.95
BMI coefficient = 0.58
age coefficient = 0.32

The predictors remain in the model, but their influence is reduced.

This can prevent the model from becoming too extreme.

Scene: shrinking without deleting

Learner: So ridge removes weak predictors?

Prof. Stat: Usually no. Ridge shrinks coefficients, but it normally keeps all predictors in the model.

Dr. Clinic: So if I include glucose, BMI, blood pressure, and age, ridge keeps them but reduces their coefficient sizes?

Prof. Stat: Correct. It is more about stabilisation than feature selection.

5. Why ridge can help prediction

Ridge regression may fit the training data slightly less perfectly.

But this can be a good thing.

A model that fits training data perfectly may have learned noise.

Ridge sacrifices some training fit to gain better generalisation.

This is the bias-variance trade-off:

Slightly more bias
Less variance
Better generalisation

In prediction modelling, better generalisation is often more important than perfect training fit.

6. Ridge and correlated predictors

Medical predictors are often correlated.

Examples:

  • BMI and waist circumference;
  • systolic and diastolic blood pressure;
  • tumour size and tumour stage;
  • multiple inflammatory biomarkers;
  • neighbouring methylation sites;
  • genes in the same biological pathway.

When predictors are correlated, ordinary regression coefficients can become unstable.

Ridge regression handles correlated predictors more smoothly by shrinking coefficients.

This can make the model less sensitive to small changes in the dataset.

7. Clinical example: diabetes prediction

Suppose we predict diabetes using:

  • glucose;
  • BMI;
  • age;
  • insulin;
  • blood pressure;
  • pedigree.

Some predictors may overlap in the information they provide.

Ordinary logistic regression may give unstable coefficients if the sample size is limited.

Ridge regression can reduce coefficient instability.

It may produce more reliable predicted risks for new patients.

8. Biomedical example: biomarkers

Suppose we predict cancer recurrence using 40 biomarkers.

Many of these biomarkers may be correlated because they belong to similar pathways.

Ordinary regression may struggle.

Ridge regression can allow all biomarkers to contribute while controlling coefficient size.

This is useful when many predictors may each carry small predictive information.

9. Lambda

Ridge regression uses a tuning parameter called lambda.

Lambda controls the strength of the penalty.

Small lambda  → weak shrinkage
Large lambda  → strong shrinkage

If lambda is too small, the model may still overfit.

If lambda is too large, the model may underfit.

Cross-validation is commonly used to choose lambda.

Scene: choosing lambda

Learner: Can I just try many lambda values and choose the one with the best test AUC?

Prof. Stat: Be careful. The test set should not be used repeatedly for tuning.

Dr. Clinic: So tuning should happen inside training data?

Prof. Stat: Yes. Use cross-validation in the training data, then evaluate once on the test data.

10. Ridge is not enough by itself

Ridge regression can reduce overfitting, but it does not solve everything.

It does not fix:

  • poor outcome definition;
  • predictors measured after the prediction time;
  • data leakage;
  • missing important predictors;
  • poor data quality;
  • lack of external validation;
  • poor calibration.

A ridge model still needs proper validation.

11. Ridge and calibration

Because ridge reduces overly extreme coefficients, it may improve calibration.

But this is not guaranteed.

After fitting a ridge model, we should still check:

  • AUC;
  • calibration plot;
  • calibration slope;
  • calibration intercept;
  • Brier score;
  • clinical usefulness.

A model can be regularised and still poorly calibrated.

12. When ridge may be useful

Ridge may be useful when:

  • many predictors are clinically plausible;
  • predictors are correlated;
  • we do not want to remove predictors;
  • many predictors may each carry small signals;
  • stable prediction is more important than a short feature list.

Ridge may be less useful when the goal is to produce a very small model.

For feature selection, lasso may be more appropriate.

Common mistake

Do not call ridge regression a feature selection method. Ridge usually keeps predictors in the model. It shrinks coefficients rather than removing variables.

Key takeaway

Ridge regression improves stability by shrinking coefficients towards zero. It is especially useful when predictors are correlated or when many predictors may each contribute modest predictive information.
