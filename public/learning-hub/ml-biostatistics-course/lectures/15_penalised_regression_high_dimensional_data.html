Penalised Regression for High-Dimensional Data

Big idea

High-dimensional data often contain far more predictors than patients.

This is common in biomedical research.

Penalised regression methods, such as ridge, lasso, and elastic net, help build prediction models when ordinary regression would be unstable or impossible.

But high-dimensional prediction is risky.

Without careful validation, models can look excellent and still fail on new data.

Scene: too many columns, too few patients

Learner: I have 200 patients and 20,000 gene expression features. Can I fit ordinary logistic regression?

Prof. Stat: No, not safely. You have far more predictors than patients.

Dr. Clinic: But those genes may contain useful biological information.

Prof. Stat: They might. Penalised regression helps use that information while controlling overfitting.

1. What high-dimensional means

High-dimensional data means there are many predictors.

Often:

p >> n

where:

p = number of predictors
n = number of observations

Examples:

300 patients and 20,000 genes
500 patients and 450,000 methylation sites
150 patients and 5,000 proteins

This is common in omics, imaging, and electronic health record research.

2. Why ordinary regression fails

Ordinary regression becomes unstable when there are too many predictors.

Problems include:

  • overfitting;
  • unstable coefficients;
  • perfect or near-perfect separation;
  • inability to estimate all coefficients;
  • excellent training performance but poor validation performance.

With thousands of features, many will appear predictive by chance.

3. Penalised regression

Penalised regression adds a penalty to the model.

The model is asked to:

predict the outcome well
and
avoid overly complex coefficients

This helps control overfitting.

Common penalised regression methods include:

  • ridge regression;
  • lasso regression;
  • elastic net.

4. Ridge for high-dimensional data

Ridge regression shrinks all coefficients.

It can handle many correlated predictors.

It is useful when many features may each contain small predictive information.

However, ridge usually keeps all predictors, so it may not produce a short biomarker list.

5. Lasso for high-dimensional data

Lasso can set many coefficients to zero.

This makes it attractive for feature selection.

For example, from:

20,000 genes

lasso may select:

30 genes

But selected features can be unstable, especially when predictors are correlated.

6. Elastic net for high-dimensional data

Elastic net combines ridge and lasso.

It can select features while also handling correlated predictors more smoothly than lasso alone.

This makes it useful in omics data, where groups of features often move together.

Scene: choosing the method

Learner: Ridge, lasso, elastic net — which one should I choose?

Prof. Stat: It depends. Ridge is stable, lasso is sparse, elastic net is a compromise.

Dr. Clinic: So we compare them using proper validation?

Prof. Stat: Exactly. Validation should guide the choice.

7. Tuning matters

Penalised regression depends on tuning parameters.

The most important is often lambda.

Low penalty  → more complex model
High penalty → simpler model

Elastic net also uses alpha, which controls the balance between ridge and lasso.

Tuning must be performed using training data or inner validation.

The final test set should remain untouched.

8. Nested validation

High-dimensional workflows often need nested validation.

The inner loop selects features and tunes parameters.

The outer loop estimates performance.

This avoids using the same data to both choose the model and evaluate the model.

Nested validation is especially important when many tuning choices are explored.

9. Preprocessing leakage

High-dimensional data often require preprocessing.

Examples:

  • normalisation;
  • filtering;
  • batch correction;
  • PCA;
  • feature scaling;
  • missing data handling.

If preprocessing is learned from the full dataset before validation, leakage can occur.

For example:

1. Perform PCA using all samples.
2. Cross-validate a model using PCA components.

This is biased because the validation folds influenced the PCA components.

10. Batch effects

Batch effects are technical differences unrelated to biology.

Examples:

  • sequencing batch;
  • methylation array plate;
  • laboratory technician;
  • processing date;
  • scanner;
  • hospital site.

If batch is associated with outcome, the model may learn technical artefacts instead of biological signal.

This can produce excellent internal performance and poor external performance.

11. External validation

High-dimensional models can be fragile.

A model trained on one platform, cohort, or laboratory may fail elsewhere.

External validation is essential before making strong claims.

For biomedical prediction, internal performance alone is not enough.

12. Calibration still matters

High-dimensional models should not be evaluated only by AUC.

A model may rank patients well but produce risk estimates that are too high or too low.

For clinical prediction, calibration matters because predicted probabilities may guide action.

Assessment should include:

  • calibration plot;
  • calibration slope;
  • calibration intercept;
  • Brier score;
  • clinical usefulness where relevant.

13. Interpretation of selected features

Selected features from penalised regression should be interpreted cautiously.

They may represent:

  • true biological signal;
  • correlated proxies;
  • batch effects;
  • cohort-specific patterns;
  • noise.

A prediction model can suggest candidates for further study, but it does not prove biological mechanism.

Scene: discovery or prediction?

Learner: My penalised model selected 25 genes. Can I say these genes explain the disease?

Prof. Stat: Not from prediction modelling alone.

Dr. Clinic: So they are candidate predictors, not confirmed mechanisms?

Prof. Stat: Exactly.

14. Medical example: methylation prediction

Suppose we want to predict exposure status using DNA methylation.

The dataset contains:

300 participants
450,000 CpG sites

A penalised model may select a small set of CpG sites.

Before trusting the result, ask:

  • Was feature selection inside validation?
  • Was preprocessing inside validation?
  • Were batch effects assessed?
  • Was calibration assessed?
  • Was external validation performed?
  • Are selected features stable?
  • Are biological claims cautious?

15. Practical checklist

Before trusting a high-dimensional penalised model, ask:

  • What is the exact prediction question?
  • What is the prediction time point?
  • How many patients are available?
  • How many outcome events are available?
  • How many candidate predictors are available?
  • Was preprocessing done inside validation?
  • Was feature selection done inside validation?
  • Was tuning performed without touching the test set?
  • Were batch effects assessed?
  • Was calibration assessed?
  • Was external validation performed?
  • Are selected features stable?

Common mistake

Do not trust a high-dimensional model simply because it has a high internal AUC. High-dimensional models are very vulnerable to leakage, overfitting, and batch effects.

Key takeaway

Penalised regression makes high-dimensional prediction possible, but credible biomedical use requires careful tuning, leakage prevention, validation, calibration, external testing, and cautious interpretation of selected features.
