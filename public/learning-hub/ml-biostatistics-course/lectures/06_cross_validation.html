Today’s big question

Why do we use cross-validation in medical machine learning?

Cross-validation is a method for estimating how well a model may perform on unseen data.

Instead of using only one training/test split, cross-validation repeatedly splits the data into training and validation parts.

Intuition first

Imagine testing a student using only one question.

The result may depend too much on that one question.

A better assessment uses several questions.

Cross-validation works similarly. It tests the model across several different splits of the data.

Cross-validation gives a more stable estimate of model performance than relying on one random split.

What is k-fold cross-validation?

In k-fold cross-validation, the data are divided into k parts called folds.

For example, in 5-fold cross-validation:

  1. split the data into 5 folds;
  2. train the model on 4 folds;
  3. validate it on the remaining fold;
  4. repeat until each fold has been used for validation;
  5. average the performance.
Fold 1 → validation
Fold 2 → validation
Fold 3 → validation
Fold 4 → validation
Fold 5 → validation

Each patient is used for validation once.

Mini conversation

Curious Learner:
“So cross-validation gives me many test sets?”

Prof Stat:
“Not exactly. It gives repeated internal validation splits within the development data.”

Dr Clinic:
“So it is still not external validation?”

Prof Stat:
“Correct. Cross-validation is internal validation, not proof that the model works elsewhere.”

Why cross-validation is useful

Cross-validation is useful when:

  • the dataset is not very large;
  • one train/test split may be unstable;
  • we want to compare models;
  • we want to tune model settings;
  • we want a better internal estimate of performance.

It helps us avoid judging a model based on one lucky or unlucky split.

Medical example

Suppose we are developing a diabetes prediction model.

A single train/test split may give:

AUC = 0.81

But another split may give:

AUC = 0.74

This variation happens because each split contains a different set of patients.

Cross-validation gives performance across multiple splits, which is more informative.

Common mistake

Common mistake: doing feature selection before cross-validation

If you select predictors using the full dataset before cross-validation, information from the validation folds has already influenced model development.

This creates data leakage.

Feature selection, scaling, imputation, and preprocessing should happen inside the cross-validation process.

Clinical reality check

Clinical reality check

Cross-validation can improve internal validation, but it does not replace external validation.

A model may perform well during cross-validation and still fail in a different hospital, country, time period, or patient group.

Key takeaway

Cross-validation helps estimate model performance more reliably than a single split.

But it must be done carefully.

The full modelling workflow, including preprocessing and feature selection, should be repeated inside each fold.

Reflection question

Why is it unsafe to select predictors using the full dataset before cross-validation?

Show suggested answer Because the validation folds influence predictor selection before the model is evaluated. This means the validation data are no longer truly unseen, and performance estimates may be too optimistic.
