Elastic Net

Big idea

Elastic net combines ridge regression and lasso regression.

Ridge shrinks coefficients but usually keeps all predictors.

Lasso shrinks some coefficients to zero and performs feature selection.

Elastic net sits between them.

It can shrink coefficients, select predictors, and behave more stably when predictors are correlated.

Learner: Ridge keeps everything. Lasso selects predictors. What if I want something in between?

Dr. Clinic: So it can select predictors but not behave as sharply as lasso?

1. Ridge and lasso recap

Ridge regression:

Shrinks coefficients
Usually keeps all predictors
Useful with correlated predictors

Lasso regression:

Shrinks coefficients
Can set some coefficients to zero
Useful for feature selection

Elastic net combines these ideas.

It can be useful when we want both stability and some feature selection.

2. Why elastic net exists

Lasso can be unstable when predictors are highly correlated.

For example, if three biomarkers measure a similar biological process, lasso may select one and discard the others.

In another dataset, it may select a different one.

Elastic net can handle correlated groups more smoothly.

It may keep related predictors together while still applying shrinkage.

3. Alpha

Elastic net uses a mixing parameter often called alpha.

A common interpretation is:

alpha = 0      ridge regression
alpha = 1      lasso regression
0 < alpha < 1  elastic net

So alpha controls whether the model behaves more like ridge or more like lasso.

4. Lambda

Elastic net also uses lambda.

Lambda controls the overall strength of shrinkage.

Small lambda  → less shrinkage
Large lambda  → more shrinkage

So elastic net has two tuning ideas:

alpha  → balance between ridge and lasso
lambda → strength of regularisation

Both matter.

Scene: tuning two knobs

Learner: Ridge has lambda. Lasso has lambda. Elastic net has alpha and lambda?

Prof. Stat: Correct. Alpha controls the ridge-lasso mixture. Lambda controls how strong the penalty is.

Dr. Clinic: More tuning means more responsibility.

Prof. Stat: Exactly. Tuning must be done inside the validation process.

5. Why correlated predictors matter

Biomedical predictors are often correlated.

Examples:

  • genes in the same pathway;
  • neighbouring methylation sites;
  • related proteins;
  • repeated clinical measurements;
  • imaging features from the same region.

If predictors are correlated, lasso may choose one and ignore others.

Elastic net can allow groups of related predictors to contribute.

This may produce a more stable model.

6. Elastic net in omics

Omics data are often high-dimensional and correlated.

For example:

300 participants
20,000 gene expression features

Many genes may move together because they belong to the same pathway.

Elastic net can be useful because it can select features while also keeping grouped information more stable than lasso alone.

7. Elastic net in clinical prediction

Elastic net may also be useful in clinical prediction when we have:

  • many candidate predictors;
  • correlated clinical measures;
  • biomarker panels;
  • repeated measurements;
  • electronic health record features.

It is especially useful when we want a balance between model simplicity and stability.

8. Elastic net versus ridge

Elastic net may be preferred over ridge when:

  • some feature selection is desired;
  • many predictors are likely irrelevant;
  • a smaller predictor set is preferred;
  • clinical measurement burden matters.

Ridge may be preferred when:

  • many predictors each carry small useful signals;
  • removing predictors is not important;
  • stability is the main goal.

9. Elastic net versus lasso

Elastic net may be preferred over lasso when:

  • predictors are correlated;
  • lasso feature selection is unstable;
  • grouped biological signals are expected;
  • high-dimensional omics data are used.

Lasso may be preferred when:

  • a very sparse model is desired;
  • predictors are not strongly correlated;
  • simplicity is the main goal.

10. Tuning and validation

Elastic net tuning must be handled carefully.

An honest workflow is:

1. Split data into training and test data.
2. Use cross-validation inside training data to choose alpha and lambda.
3. Fit the final elastic net model on training data.
4. Evaluate once on the untouched test data.

The test set should not be used repeatedly for tuning.

If it is, test performance becomes optimistic.

11. Interpretation

Elastic net-selected features are prediction features.

They are not automatically causal.

Before making biological or clinical claims, ask:

  • are the features stable?
  • are they biologically plausible?
  • are they measured reliably?
  • are they available at prediction time?
  • do they validate externally?
  • could they reflect batch effects?

12. Clinical example

Suppose we predict cancer recurrence using:

  • age;
  • tumour stage;
  • tumour grade;
  • treatment;
  • 5,000 gene expression features.

Ridge may keep all gene features.

Lasso may select a small set but be unstable.

Elastic net may select related groups of genes while shrinking coefficients.

This can be useful, but external validation is still needed.

Common mistake

Do not tune alpha and lambda using the final test set. Tuning should happen inside the training data or inside an inner validation loop.

Key takeaway

Elastic net combines ridge and lasso. It is especially useful when predictors are correlated and we want both shrinkage and some feature selection.
