Case Studies

This section contains applied medical machine learning case studies.

The aim is to connect the course concepts with realistic modelling workflows, clinical interpretation, figures, and common problems that occur in biomedical data analysis.

Each case study follows a practical structure:

  • clinical question;
  • dataset description;
  • outcome and predictors;
  • prediction time point;
  • model development in R;
  • performance evaluation;
  • figures;
  • clinical interpretation;
  • limitations;
  • reflection questions.

Current case study progression

Case Study 1: Diabetes Risk Prediction

This case study introduces a complete clinical prediction workflow using logistic regression.

It focuses on:

  • binary outcome definition;
  • train/test split;
  • logistic regression;
  • predicted probabilities;
  • confusion matrix;
  • sensitivity and specificity;
  • ROC curve and AUC;
  • calibration plot;
  • Brier score;
  • threshold interpretation;
  • clinical interpretation.

Case Study 2: Regularised Diabetes Risk Prediction

This case study extends Case Study 1 by comparing ordinary logistic regression with regularised regression methods.

It focuses on:

  • ridge regression;
  • lasso regression;
  • elastic net;
  • coefficient shrinkage;
  • feature selection;
  • AUC comparison;
  • Brier score comparison;
  • calibration comparison;
  • selected predictor interpretation;
  • why selected features are not automatically causal.

How to use the case studies

A good way to use each case study is:

  1. Read the clinical question.
  2. Understand the outcome and predictors.
  3. Run the related R script.
  4. Inspect the generated figures.
  5. Read the clinical interpretation.
  6. Answer the reflection questions.
  7. Revisit the related lecture, note, or exercise if needed.

Key message

The case studies are designed to show that medical machine learning is not only about fitting models.

A useful clinical prediction workflow must define the prediction question clearly, avoid leakage, evaluate discrimination and calibration, interpret thresholds carefully, and explain limitations honestly.
