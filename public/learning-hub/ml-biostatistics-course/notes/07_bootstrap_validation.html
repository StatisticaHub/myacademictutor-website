Bootstrap Validation in Clinical Prediction Modelling

Learning objectives

By the end of this note, you should be able to:

  1. Explain what apparent performance means.
  2. Explain why apparent performance is usually optimistic.
  3. Define optimism in the context of prediction model performance.
  4. Describe what bootstrap sampling means.
  5. Explain how bootstrap validation estimates optimism.
  6. Interpret optimism-corrected performance.
  7. Understand why bootstrap validation is useful in clinical prediction modelling.
  8. Distinguish bootstrap internal validation from external validation.

1. Why this note matters

In medical prediction modelling, performance estimated in the development dataset is usually too optimistic. A model may appear to discriminate well, classify patients accurately, or produce low prediction error simply because it has already been developed using the same data.

This is especially important in medical research because many datasets are small, outcomes may be rare, and the number of candidate predictors may be large. A model can easily learn dataset-specific patterns that do not apply to future patients.

Bootstrap validation is a method for estimating this optimism. It is widely used in clinical prediction modelling because it can make efficient use of the development data while still assessing how much performance may be exaggerated.

The practical message of this note is:

Bootstrap validation helps estimate how much a model’s apparent performance is likely to be too optimistic.

2. Apparent performance

Apparent performance is the performance of a model when it is evaluated on the same data used to develop it.

For example:

Fit the model on the full development dataset.
Evaluate the model on the same full development dataset.

This may give:

Apparent AUC = 0.86

The problem is that the model has already seen the data. It may have learned real patterns, but it may also have learned random noise, sample-specific patterns, or modelling choices that were selected because they worked well in this dataset.

Therefore, apparent performance is usually optimistic.

3. Why apparent performance is optimistic

Apparent performance is optimistic because model development is not a neutral process.

During development, analysts may:

  • try several models;
  • select predictors;
  • transform variables;
  • tune hyperparameters;
  • choose interactions;
  • remove outliers;
  • compare thresholds;
  • select the best-performing model.

Even if the final model is evaluated only once, the dataset may have influenced many modelling decisions.

This means apparent performance reflects both true signal and overfitting.

4. Optimism

Optimism is the amount by which apparent performance overestimates performance on new data.

Conceptually:

Optimism = apparent performance - expected performance on new data

For performance measures where higher is better, such as AUC or C-index, optimism is subtracted from apparent performance.

For example:

Apparent AUC = 0.86
Estimated optimism = 0.04
Optimism-corrected AUC = 0.82

The optimism-corrected estimate is usually more realistic than the apparent estimate.

5. What is bootstrap sampling?

Bootstrap sampling means sampling observations from the original dataset with replacement.

If the original dataset contains 500 patients, a bootstrap sample also contains 500 rows.

However:

  • some patients may appear more than once;
  • some patients may not appear at all;
  • the sample is a resampled version of the original dataset.

Sampling with replacement allows us to create many slightly different versions of the development dataset.

6. Why bootstrap sampling is useful

Bootstrap sampling helps us study how the model behaves when the development data change slightly.

Each bootstrap sample acts like a new version of the dataset. The model is fitted in the bootstrap sample and evaluated to estimate how much performance is inflated by fitting the model to that sample.

Repeating this many times gives an estimate of average optimism.

7. Basic bootstrap validation process

A simplified bootstrap validation process is:

  1. Fit the original model in the full development data.
  2. Calculate apparent performance.
  3. Draw a bootstrap sample.
  4. Fit the model in the bootstrap sample.
  5. Evaluate performance in the bootstrap sample.
  6. Evaluate the same bootstrap-fitted model in the original data.
  7. Estimate optimism for that bootstrap sample.
  8. Repeat many times.
  9. Average the optimism.
  10. Subtract average optimism from apparent performance.

This gives an optimism-corrected estimate.

8. Medical example: cancer recurrence prediction

Suppose we develop a model to predict cancer recurrence within 5 years.

The apparent performance is:

Apparent AUC = 0.84

Bootstrap validation estimates:

Optimism = 0.05

The optimism-corrected estimate is:

Optimism-corrected AUC = 0.84 - 0.05 = 0.79

This suggests that some of the apparent performance was due to overfitting.

The model may still be useful, but the corrected estimate is more cautious.

9. Bootstrap validation versus train/test split

A train/test split holds out part of the dataset for evaluation.

Bootstrap validation uses resampling and can allow the full dataset to contribute to model development.

This can be attractive in clinical prediction modelling because sample sizes are often limited.

For example, if there are only 200 patients and 40 events, holding out 20% of the data may leave too little data for model development and too few events for stable evaluation.

Bootstrap validation can be more efficient in such situations.

10. Bootstrap validation versus cross-validation

Both bootstrap validation and cross-validation are internal validation methods.

Cross-validation repeatedly divides the dataset into training and validation folds.

Bootstrap validation repeatedly samples with replacement and estimates optimism.

In clinical prediction modelling, bootstrap validation is often preferred for estimating optimism in a final model, especially when the sample size is limited.

Cross-validation is often used for model comparison and tuning.

Both methods must include the full modelling process to avoid biased results.

11. Bootstrap validation and the full modelling process

Bootstrap validation should include all major steps in model development.

If the original modelling process involved:

  • feature selection;
  • transformation selection;
  • tuning;
  • imputation;
  • variable filtering;
  • model comparison;

then these steps should be repeated inside the bootstrap process.

If not, the optimism estimate may be too small.

The bootstrap should reflect the real modelling process, not only the final fitted model.

12. Bootstrap validation and calibration

Bootstrap validation can assess more than discrimination.

It can also assess:

  • calibration slope;
  • calibration intercept;
  • Brier score;
  • calibration plots;
  • C-index;
  • overall performance.

This is important because a model may rank patients well but produce risk estimates that are too extreme.

A calibration slope less than 1 often suggests overfitting.

13. Calibration slope and shrinkage

When a model is overfitted, predictions may be too extreme.

High-risk patients may be predicted too high.

Low-risk patients may be predicted too low.

This often produces a calibration slope less than 1.

Bootstrap validation can help estimate the degree of overfitting and may support shrinkage or recalibration.

14. How many bootstrap samples?

There is no single universal number.

Common choices include:

100 bootstrap samples
200 bootstrap samples
500 bootstrap samples
1000 bootstrap samples

More samples usually give more stable estimates but require more computation.

For serious modelling work, several hundred bootstrap samples are commonly used.

15. Bootstrap validation is internal validation

Bootstrap validation uses the development dataset.

Therefore, it remains internal validation.

It gives information about expected performance in similar patients from the same source population.

It does not prove that the model will work in another hospital, country, laboratory system, or time period.

External validation is still needed before claiming broad generalisability.

16. What bootstrap validation does not solve

Bootstrap validation does not fix:

  • poor outcome definition;
  • poor predictor quality;
  • data leakage;
  • missing important predictors;
  • poor measurement quality;
  • unrepresentative data;
  • lack of external validation;
  • lack of clinical usefulness.

If the model is developed using future information or leakage, bootstrap validation may still give misleading results.

17. Common mistakes

17.1 Reporting only apparent performance

This can seriously exaggerate model performance.

17.2 Calling bootstrap validation external validation

Bootstrap validation is internal validation.

17.3 Not repeating feature selection

Feature selection should be included inside the bootstrap process.

17.4 Ignoring calibration

Optimism should be considered for calibration and overall error, not only AUC.

17.5 Using too few bootstrap samples

Very few bootstrap samples may give unstable estimates.

18. Reporting bootstrap validation

A clear report should describe:

  • number of bootstrap samples;
  • performance measures evaluated;
  • apparent performance;
  • estimated optimism;
  • optimism-corrected performance;
  • whether feature selection was repeated;
  • whether tuning was repeated;
  • calibration assessment;
  • limitations.

Transparent reporting is important because bootstrap validation can be misunderstood if the workflow is unclear.

19. Worked example: hospital mortality model

Suppose we develop a model to predict 30-day hospital mortality.

The apparent AUC is:

0.88

Bootstrap validation estimates optimism as:

0.06

The optimism-corrected AUC is:

0.82

This suggests that the model is likely less impressive than the apparent performance suggested.

If the calibration slope is also low, the model may be overfitted and may require shrinkage or recalibration before further testing.

20. Summary

Bootstrap validation is a practical internal validation method for clinical prediction models. It estimates how much apparent performance may be exaggerated due to overfitting.

It is particularly useful when sample size is limited and when holding out a test set would be inefficient.

However, bootstrap validation is not external validation. It should be used as part of responsible model development, not as final proof of clinical usefulness.

Key terms

Term Meaning
Apparent performance Performance measured on the development data used to fit the model
Optimism Amount by which apparent performance overestimates expected new-data performance
Bootstrap sample Sample drawn from the original data with replacement
Sampling with replacement Sampling where observations can be selected more than once
Optimism-corrected performance Apparent performance adjusted by estimated optimism
Internal validation Validation using the same data source as model development
External validation Validation using independent data from another setting, time, or population
Calibration slope Measure of whether predictions are too extreme or not extreme enough
Overfitting Learning noise or dataset-specific patterns
Shrinkage Reducing model coefficients or predictions to improve generalisability

Key takeaway

Bootstrap validation estimates how much a model’s apparent performance is likely to be too optimistic. It gives a more realistic internal estimate of performance, but it does not replace external validation or clinical usefulness assessment.
