Feature Selection in Biomedical Data

Learning objectives

By the end of this note, you should be able to:

  1. Explain why feature selection is common in biomedical machine learning.
  2. Distinguish clinical, statistical, and model-based feature selection.
  3. Understand why feature selection can cause data leakage.
  4. Explain why feature selection must happen inside validation.
  5. Describe why selected features may be unstable.
  6. Explain why prediction feature selection does not prove causality.
  7. Recognise special risks in omics and biomarker studies.
  8. Apply a practical checklist before trusting selected features.

1. Why this note matters

Biomedical datasets often contain many possible predictors.

These may include:

  • clinical variables;
  • blood test measurements;
  • imaging features;
  • biomarkers;
  • gene expression values;
  • DNA methylation sites;
  • proteins;
  • metabolites;
  • electronic health record codes.

Feature selection is the process of choosing a subset of predictors.

It can make models simpler and more practical.

However, feature selection is also one of the easiest ways to accidentally create data leakage.

The practical message of this note is:

Feature selection can simplify a biomedical prediction model, but if done incorrectly, it can seriously bias model performance.

2. Why feature selection is used

Feature selection may be used to:

  • reduce overfitting;
  • simplify the model;
  • reduce measurement burden;
  • improve interpretability;
  • remove noisy predictors;
  • identify candidate biomarkers;
  • create a clinically feasible predictor set.

In medicine, a model using fewer predictors may be easier to implement.

For example, a model requiring five routinely measured blood tests may be more practical than a model requiring 100 specialised biomarkers.

3. Feature selection is not automatically good

Feature selection can help, but it can also harm.

It may remove useful predictors.

It may select unstable predictors.

It may select variables that work only in one dataset.

It may create data leakage if performed before validation.

Therefore, feature selection should be treated as part of the modelling process, not as a harmless preprocessing step.

4. Clinical feature selection

Clinical feature selection uses subject-matter knowledge.

For example, a diabetes risk model may include:

  • glucose;
  • BMI;
  • age;
  • family history;
  • blood pressure.

These predictors are clinically plausible and usually available.

Clinical feature selection helps ensure that the model uses predictors that are meaningful, measurable, and available at the intended prediction time.

5. Statistical feature selection

Statistical feature selection uses the data to select predictors.

Examples include:

  • univariable p-value screening;
  • correlation filtering;
  • stepwise regression;
  • lasso selection;
  • recursive feature elimination;
  • model-based variable importance.

These methods may be useful, but they must be used carefully.

If they use the full dataset before validation, leakage occurs.

6. Model-based feature selection

Model-based feature selection uses a fitted model to identify useful predictors.

Examples include:

  • lasso non-zero coefficients;
  • random forest variable importance;
  • gradient boosting importance;
  • recursive feature elimination.

These methods identify predictors that help prediction in a specific dataset and modelling workflow.

They do not automatically identify causal factors.

7. Feature-selection leakage

Feature-selection leakage occurs when validation or test data influence predictor selection.

An incorrect workflow is:

1. Use the full dataset to select predictors.
2. Split into training and test sets.
3. Fit the model using selected predictors.
4. Evaluate on the test set.

This is biased because the test set helped choose the predictors.

The test performance may look better than it really is.

8. Correct feature selection workflow

A safer workflow is:

1. Split data into training and test sets.
2. Select features using training data only.
3. Fit the model using training data only.
4. Evaluate once on the untouched test set.

For cross-validation:

Feature selection must happen inside each training fold.

This means that each validation fold remains genuinely unseen.

9. Why this matters so much in biomedical data

Biomedical datasets often contain many predictors.

For example:

n = 300 participants
p = 450,000 methylation sites

With so many predictors, some will appear associated with the outcome by chance.

If the full dataset is used to select features before validation, the validation performance can become seriously inflated.

This is one of the most common problems in omics prediction studies.

10. Omics feature selection

Omics feature selection may involve:

  • selecting top genes;
  • selecting CpG sites;
  • selecting proteins;
  • selecting metabolites;
  • selecting genomic variants;
  • selecting pathway scores.

These features may be biologically interesting.

However, if selected incorrectly, they may simply reflect noise, batch effects, or cohort-specific patterns.

11. Batch effects and feature selection

Batch effects are technical differences unrelated to biology.

Examples include:

  • laboratory batch;
  • sequencing run;
  • methylation array plate;
  • sample processing date;
  • scanner;
  • technician;
  • hospital site.

If batch is associated with outcome, feature selection may select variables that reflect batch rather than disease biology.

This can produce excellent internal performance and poor external performance.

12. Feature stability

A selected feature should ideally be stable.

Stability means that the same or similar features are selected across:

  • bootstrap samples;
  • cross-validation folds;
  • repeated analyses;
  • external datasets.

If selected features change dramatically across resamples, interpretation should be cautious.

The model may still have some predictive ability, but the specific selected feature list may not be reliable.

13. Correlated predictors and instability

Biomedical predictors are often correlated.

For example:

  • nearby methylation sites may move together;
  • genes in the same pathway may be correlated;
  • proteins in the same system may be related;
  • clinical measurements may reflect the same underlying condition.

When predictors are correlated, feature selection may choose one predictor from a group and ignore others.

In another dataset, a different predictor from the same group may be selected.

This can make interpretation unstable.

14. Prediction versus discovery

Feature selection for prediction is not the same as biological discovery.

A predictor may help prediction because it is correlated with the outcome.

That does not prove it causes the outcome.

Similarly, a biologically important predictor may not be selected if it does not improve prediction in that dataset.

Always distinguish:

Prediction question: Does this feature help predict?
Discovery question: Is this feature biologically or causally important?

15. Medical example: diabetes prediction

In a diabetes prediction model, feature selection may identify:

  • glucose;
  • BMI;
  • age;
  • pedigree.

These predictors are clinically plausible.

However, even here, selection should be validated.

A feature selected in one dataset may not perform similarly in another population.

16. Biomedical example: methylation prediction

Suppose we want to predict exposure status from DNA methylation data.

The dataset contains:

300 participants
450,000 CpG sites

If we select the top 100 CpG sites using the full dataset and then cross-validate a classifier, performance will be biased.

The correct workflow is to select CpG sites inside each training fold.

External validation is also important before making strong claims.

17. Feature selection and clinical practicality

A selected predictor should be practical.

Ask:

  • Is it available at prediction time?
  • Is it routinely measured?
  • Is it affordable?
  • Is it reliable?
  • Does it require specialised technology?
  • Can it be used in the target healthcare setting?

A statistically selected feature may not be clinically useful if it cannot be measured in practice.

18. Practical checklist

Before trusting selected features, ask:

  1. Were features selected inside validation?
  2. Was the final test set untouched?
  3. Are selected features stable?
  4. Are they clinically or biologically plausible?
  5. Are they measured reliably?
  6. Are they available at the intended prediction time?
  7. Could batch effects explain them?
  8. Do they validate externally?
  9. Do they improve calibration or only AUC?
  10. Are claims limited to what the study design supports?

19. Common mistakes

Common mistakes include:

  • selecting features before train/test split;
  • selecting features before cross-validation;
  • using the test set to refine feature selection;
  • interpreting selected features as causal;
  • reporting only selected features without stability checks;
  • ignoring batch effects;
  • ignoring clinical feasibility.

These mistakes can make a model look more convincing than it really is.

20. Reporting feature selection

A clear report should describe:

  • all candidate predictors;
  • feature selection method;
  • when feature selection was performed;
  • whether it was inside validation;
  • number of selected features;
  • stability of selected features if assessed;
  • clinical or biological interpretation;
  • validation performance;
  • limitations.

Transparency is essential for judging reliability.

21. Summary

Feature selection is common in biomedical machine learning.

It can make models simpler and more practical.

However, feature selection must be included inside the validation process.

Selected features should be interpreted as prediction features first, not automatically as causal or biologically validated discoveries.

Key terms

Term Meaning
Feature selection Choosing a subset of predictors
Clinical feature selection Predictor selection based on domain knowledge
Statistical feature selection Predictor selection based on data-driven criteria
Model-based selection Feature selection based on fitted model behaviour
Leakage Information from validation or test data influencing model development
Feature-selection leakage Leakage caused by selecting predictors before validation
Stability Consistency of selected predictors across resamples
Batch effect Technical variation unrelated to biological signal
Omics High-dimensional molecular biomedical data
Prediction Estimating outcomes in new observations
Discovery Identifying biological mechanisms or associations
