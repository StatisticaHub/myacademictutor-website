Ridge Regression

Learning objectives

By the end of this note, you should be able to:

  1. Explain why ordinary regression can overfit when there are many predictors or limited outcome events.
  2. Describe the idea of coefficient shrinkage.
  3. Explain how ridge regression differs from ordinary logistic regression.
  4. Understand why ridge regression usually keeps all predictors in the model.
  5. Explain why ridge regression is useful when predictors are correlated.
  6. Describe the role of the tuning parameter lambda.
  7. Understand why ridge regression still requires validation and calibration assessment.
  8. Recognise when ridge regression may be preferable to lasso regression.

1. Why this note matters

In medical prediction modelling, ordinary regression can become unstable when there are too many predictors for the available sample size or number of outcome events.

This is common in clinical and biomedical datasets.

For example, we may have:

200 patients
35 outcome events
25 candidate predictors

Ordinary logistic regression may fit the development data well, but some coefficients may be too large, unstable, or strongly affected by random noise.

Ridge regression helps by shrinking coefficients towards zero.

The practical message of this note is:

Ridge regression improves stability by reducing overly large coefficients, especially when predictors are correlated or the model is at risk of overfitting.

2. Ordinary regression and instability

Ordinary logistic regression estimates coefficients that maximise the fit to the data.

This works well when:

  • the sample size is large;
  • the number of predictors is modest;
  • predictors are not highly correlated;
  • there are enough outcome events;
  • the relationships are not too noisy.

However, medical datasets often violate these conditions.

Clinical prediction studies may have:

  • small sample sizes;
  • rare outcomes;
  • many candidate predictors;
  • correlated clinical measurements;
  • noisy biomarker data;
  • missing values;
  • site-specific patterns.

In such settings, ordinary regression may overfit.

3. What overfitting looks like

Overfitting means the model learns patterns that are too specific to the development dataset.

An overfitted model may show:

High training performance
Lower validation performance
Poor external performance

For example:

Training AUC = 0.90
Test AUC = 0.72
External validation AUC = 0.64

This suggests that the model learned some unstable patterns that did not generalise.

Ridge regression tries to reduce this instability.

4. What ridge regression does

Ridge regression adds a penalty to the model-fitting process.

Ordinary regression says:

Fit the data as well as possible.

Ridge regression says:

Fit the data well, but avoid unnecessarily large coefficients.

The penalty discourages large coefficients.

As a result, ridge regression produces a model that is usually less extreme and more stable.

5. Coefficient shrinkage

Shrinkage means pulling coefficients towards zero.

For example, ordinary logistic regression may estimate:

glucose coefficient = 1.20
BMI coefficient = 0.80
age coefficient = 0.45

Ridge regression may shrink these to:

glucose coefficient = 0.95
BMI coefficient = 0.62
age coefficient = 0.33

The predictors are still in the model, but their effects are less extreme.

This can improve performance in new patients.

6. Why shrinkage helps

Large coefficients can appear because the model is fitting random variation in the training data.

This is especially likely when:

  • the sample size is small;
  • predictors are correlated;
  • there are many predictors;
  • outcome events are limited.

By shrinking coefficients, ridge regression reduces the model’s sensitivity to random noise.

It may fit the training data slightly less perfectly, but it can perform better on new data.

This is an example of the bias-variance trade-off.

7. Bias-variance trade-off

Ridge regression introduces some bias by shrinking coefficients.

However, it reduces variance.

This means the model becomes less sensitive to small changes in the training data.

The trade-off is:

slightly more bias
less variance
better generalisation

In prediction, this can be a good trade-off.

A model that fits the training data perfectly is not useful if it fails on new patients.

8. Ridge usually keeps all predictors

Ridge regression usually does not set coefficients exactly to zero.

This means all predictors usually remain in the model.

For example, if the original model includes 20 predictors, ridge regression will usually keep all 20, but shrink their coefficients.

This is different from lasso regression, which can remove predictors by setting their coefficients to zero.

Therefore, ridge is usually not considered a feature selection method.

9. Why ridge is useful with correlated predictors

Medical predictors are often correlated.

Examples include:

  • BMI and waist circumference;
  • systolic and diastolic blood pressure;
  • tumour size and tumour stage;
  • related blood biomarkers;
  • gene expression features from the same biological pathway.

When predictors are correlated, ordinary regression coefficients can become unstable.

One predictor may receive a large positive coefficient, while a correlated predictor receives a small or even opposite coefficient.

Ridge regression stabilises this by shrinking coefficients.

It allows correlated predictors to share information more smoothly.

10. Medical example: diabetes prediction

Suppose we build a diabetes prediction model using:

  • glucose;
  • BMI;
  • age;
  • insulin;
  • blood pressure;
  • diabetes pedigree function.

Some of these predictors may be correlated.

For example, BMI and insulin resistance-related measures may contain overlapping information.

Ordinary logistic regression may estimate unstable coefficients if the sample size is limited.

Ridge regression can produce a more stable prediction model.

However, the ridge model still needs validation.

11. Medical example: cancer prognosis

Suppose we predict cancer recurrence using:

  • age;
  • tumour size;
  • tumour stage;
  • tumour grade;
  • receptor status;
  • treatment;
  • biomarkers.

Many predictors may be related.

For example, tumour size and stage are connected.

Ridge regression can help stabilise the model when several predictors contribute overlapping information.

This may improve predictions for new patients.

12. The tuning parameter lambda

Ridge regression uses a tuning parameter often called lambda.

Lambda controls how strongly coefficients are shrunk.

Small lambda  → weak penalty
Large lambda  → strong penalty

If lambda is too small, ridge behaves more like ordinary regression and may still overfit.

If lambda is too large, coefficients may be shrunk too much and the model may underfit.

13. Choosing lambda

Lambda is usually chosen using cross-validation.

A common workflow is:

1. Define candidate lambda values.
2. Fit ridge models across those values.
3. Use cross-validation to estimate performance.
4. Choose a lambda that gives good validated performance.

The test set should not be used repeatedly to choose lambda.

If the test set is used for tuning, it becomes part of model development and performance may become optimistic.

14. Ridge regression and calibration

Ridge regression can reduce overfitting, which may improve calibration.

However, ridge does not guarantee good calibration.

After fitting a ridge model, we should still assess:

  • AUC;
  • calibration plot;
  • calibration intercept;
  • calibration slope;
  • Brier score;
  • clinical usefulness.

A model can be regularised and still poorly calibrated.

15. Ridge versus lasso

Ridge and lasso are both regularisation methods.

Method Main behaviour Feature selection?
Ridge Shrinks coefficients Usually no
Lasso Shrinks some coefficients to zero Yes

Ridge may be preferred when:

  • predictors are correlated;
  • many predictors may contain small amounts of information;
  • stability is more important than sparsity;
  • we do not need a small predictor list.

Lasso may be preferred when:

  • we want a smaller model;
  • some predictors are likely irrelevant;
  • feature selection is a major goal.

16. What ridge regression does not solve

Ridge regression does not solve all modelling problems.

It does not fix:

  • data leakage;
  • poor outcome definition;
  • predictors measured after the prediction time;
  • poor data quality;
  • unrepresentative samples;
  • lack of external validation;
  • lack of clinical usefulness.

Ridge is a modelling tool, not a complete study design.

17. Reporting ridge regression

A clear ridge regression report should describe:

  • outcome definition;
  • prediction time point;
  • candidate predictors;
  • sample size;
  • number of outcome events;
  • missing data handling;
  • whether predictors were standardised;
  • lambda selection method;
  • validation method;
  • discrimination;
  • calibration;
  • clinical interpretation.

Without these details, readers cannot judge whether the model is reliable.

18. Summary

Ridge regression is a regularisation method that shrinks coefficients towards zero.

It is useful when ordinary regression may overfit.

It is especially helpful when predictors are correlated or when many predictors may each contribute small amounts of information.

Ridge usually keeps all predictors in the model, so it is not mainly a feature selection method.

Like all medical prediction models, ridge regression still requires careful validation, calibration assessment, and clinical interpretation.

Key terms

Term Meaning
Ridge regression Regularised regression method that shrinks coefficients towards zero
Shrinkage Pulling coefficients towards smaller values
Lambda Tuning parameter controlling penalty strength
Overfitting Learning noise or dataset-specific patterns rather than generalisable signal
Correlated predictors Predictors that carry overlapping information
Bias-variance trade-off Trade-off between model simplicity and sensitivity to the training data
Calibration Agreement between predicted risks and observed risks
Internal validation Validation using data from the same source as the development data
External validation Validation using independent data from another setting, time, or population
