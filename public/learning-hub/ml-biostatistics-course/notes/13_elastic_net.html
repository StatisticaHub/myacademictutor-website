Elastic Net

Learning objectives

By the end of this note, you should be able to:

  1. Explain why elastic net combines ridge and lasso penalties.
  2. Describe the role of the alpha parameter.
  3. Describe the role of the lambda parameter.
  4. Explain why elastic net is useful when predictors are correlated.
  5. Understand how elastic net can perform shrinkage and feature selection.
  6. Distinguish elastic net from ridge and lasso.
  7. Explain why elastic net tuning must be done inside validation.
  8. Describe biomedical situations where elastic net may be useful.

1. Why this note matters

Ridge and lasso are both useful regularisation methods, but each has limitations.

Ridge regression is stable with correlated predictors, but it usually keeps all predictors.

Lasso regression can perform feature selection, but it may be unstable when predictors are strongly correlated.

Elastic net combines ridge and lasso.

The practical message of this note is:

Elastic net is useful when we want both coefficient shrinkage and some feature selection, especially when predictors are correlated.

2. Ridge and lasso recap

Ridge regression shrinks coefficients towards zero.

It usually keeps all predictors in the model.

Lasso regression also shrinks coefficients, but it can shrink some coefficients exactly to zero.

This means lasso can remove predictors from the model.

Elastic net combines these two approaches.

3. What elastic net does

Elastic net uses a mixture of ridge and lasso penalties.

It can:

  • shrink coefficients;
  • set some coefficients to zero;
  • handle correlated predictors better than lasso alone;
  • produce a model that is more stable than pure lasso in some settings.

This makes it useful in biomedical prediction, especially with high-dimensional and correlated data.

4. The alpha parameter

Elastic net uses a mixing parameter often called alpha.

A common interpretation is:

alpha = 0   ridge regression
alpha = 1   lasso regression
0 < alpha < 1   elastic net

Alpha controls the balance between ridge-like and lasso-like behaviour.

If alpha is closer to 0, the model behaves more like ridge.

If alpha is closer to 1, the model behaves more like lasso.

5. The lambda parameter

Elastic net also uses lambda.

Lambda controls the overall strength of regularisation.

Small lambda  → weak penalty
Large lambda  → strong penalty

So elastic net has two important tuning ideas:

alpha  → type of penalty mixture
lambda → strength of penalty

Both affect the final model.

6. Why elastic net helps with correlated predictors

Biomedical predictors are often correlated.

Examples include:

  • genes in the same biological pathway;
  • neighbouring DNA methylation sites;
  • related proteins;
  • repeated clinical measurements;
  • imaging features from the same region.

Lasso may select one predictor from a correlated group and ignore the others.

Elastic net can allow groups of related predictors to contribute more smoothly.

This may improve stability.

7. Elastic net and grouped information

Suppose five genes are highly correlated because they belong to the same pathway.

Lasso may select only one gene.

Elastic net may keep several genes from the pathway, while still shrinking their coefficients.

This can be useful when the biological signal is distributed across related features.

However, this does not prove that all retained genes are causal.

8. Elastic net in omics

Elastic net is commonly useful in omics prediction because omics data often have:

  • many predictors;
  • high correlation;
  • limited sample size;
  • noisy measurements;
  • many weak signals;
  • high risk of overfitting.

Examples include:

gene expression prediction
DNA methylation prediction
proteomic risk models
metabolomic classification

Elastic net provides a compromise between ridge and lasso.

9. Elastic net versus ridge

Elastic net may be preferable to ridge when:

  • we want some feature selection;
  • some predictors are likely irrelevant;
  • the final model should be smaller;
  • measurement burden matters.

Ridge may be preferable when:

  • many predictors each carry small amounts of information;
  • removing predictors is not important;
  • stability is more important than sparsity.

10. Elastic net versus lasso

Elastic net may be preferable to lasso when:

  • predictors are strongly correlated;
  • feature selection by lasso is unstable;
  • groups of related predictors may be useful;
  • high-dimensional biomedical features are present.

Lasso may be preferable when:

  • a very sparse model is desired;
  • predictors are not highly correlated;
  • simplicity is a major goal.

11. Tuning elastic net

Elastic net tuning should be done carefully.

A simple workflow is:

1. Define candidate alpha values.
2. Define candidate lambda values.
3. Use cross-validation in the training data.
4. Select tuning values.
5. Fit the final model in the training data.
6. Evaluate once on the untouched test data.

The test set should not be used repeatedly to choose alpha or lambda.

12. Nested cross-validation

When many tuning values are explored, nested cross-validation can be helpful.

The inner loop tunes alpha and lambda.

The outer loop estimates model performance.

This avoids using the same validation data for both model selection and performance estimation.

Nested cross-validation is especially useful when many models are compared.

13. Medical example: cancer recurrence

Suppose we predict cancer recurrence using:

  • age;
  • stage;
  • grade;
  • treatment;
  • 5,000 gene expression features.

Ridge may keep all 5,000 features.

Lasso may select a small number of features but be unstable.

Elastic net may select a group of related genes while shrinking coefficients.

This may provide a more stable prediction model.

But external validation is still essential.

14. Medical example: diabetes prediction

In a smaller clinical dataset, elastic net may not be necessary if only a few predictors are used.

For example, if we use only glucose, BMI, age, and blood pressure, ordinary logistic regression or ridge may be enough.

Elastic net becomes more useful when the number of candidate predictors increases or predictors are strongly correlated.

15. Interpretation of selected features

Elastic net-selected features should be interpreted cautiously.

They may be:

  • true biological signals;
  • proxies for other variables;
  • correlated with selected pathways;
  • artefacts of preprocessing;
  • dataset-specific patterns;
  • selected due to random variation.

Prediction feature selection is not the same as biological discovery.

16. Elastic net and calibration

Elastic net may reduce overfitting, but it does not guarantee good calibration.

After fitting an elastic net model, we should assess:

  • discrimination;
  • calibration plot;
  • calibration slope;
  • calibration intercept;
  • Brier score;
  • decision curve analysis if linked to action.

Regularisation is not a substitute for validation.

17. Common mistakes

Common mistakes include:

  • tuning alpha and lambda on the final test set;
  • selecting features before validation;
  • interpreting selected features as causal;
  • reporting only AUC;
  • ignoring calibration;
  • ignoring feature stability;
  • failing to externally validate high-dimensional models.

These mistakes can make results look stronger than they really are.

18. Reporting elastic net models

A clear report should describe:

  • outcome definition;
  • candidate predictors;
  • preprocessing;
  • standardisation;
  • alpha selection;
  • lambda selection;
  • validation design;
  • selected predictors;
  • discrimination;
  • calibration;
  • feature stability if relevant;
  • external validation if available.

19. Summary

Elastic net combines ridge and lasso.

It can shrink coefficients and perform feature selection.

It is especially useful when predictors are correlated, which is common in biomedical data.

Like ridge and lasso, elastic net requires careful validation and cautious interpretation.
