Penalised Regression for High-Dimensional Data

Learning objectives

By the end of this note, you should be able to:

  1. Explain what high-dimensional data means.
  2. Describe why ordinary regression can fail when the number of predictors is much larger than the number of observations.
  3. Explain how penalised regression helps control overfitting.
  4. Compare ridge, lasso, and elastic net in high-dimensional settings.
  5. Understand the role of tuning parameters.
  6. Explain why nested validation may be needed.
  7. Recognise the risks of batch effects and preprocessing leakage.
  8. Interpret high-dimensional prediction models cautiously.

1. Why this note matters

High-dimensional biomedical data are now common in medical research.

Examples include:

  • gene expression data;
  • DNA methylation data;
  • proteomics;
  • metabolomics;
  • microbiome data;
  • imaging radiomics;
  • large electronic health record feature sets.

These datasets may contain thousands or even hundreds of thousands of predictors.

The number of patients may be much smaller.

This creates a major risk of overfitting.

The practical message of this note is:

Penalised regression makes high-dimensional prediction possible, but trustworthy results require careful validation, leakage prevention, and cautious interpretation.

2. What high-dimensional means

High-dimensional data means the number of predictors is large.

Often, the number of predictors is much larger than the number of observations.

This is commonly written as:

p >> n

where:

p = number of predictors
n = number of observations

For example:

n = 300 participants
p = 450,000 CpG methylation sites

This is a high-dimensional problem.

3. Why high-dimensional data are difficult

When there are many predictors, some will appear related to the outcome by chance.

The model may find patterns that are not real.

This is especially risky when:

  • sample size is small;
  • outcome events are limited;
  • predictors are noisy;
  • predictors are correlated;
  • batch effects are present;
  • feature selection is done before validation;
  • many models are tried and only the best result is reported.

High-dimensional prediction requires careful study design.

4. Why ordinary regression can fail

Ordinary regression may fail when there are too many predictors.

Problems include:

  • unstable coefficients;
  • overfitting;
  • perfect or near-perfect separation;
  • inability to estimate all parameters;
  • excellent training performance but poor validation performance.

For example, ordinary logistic regression cannot safely estimate 20,000 gene coefficients from 300 patients.

Penalised regression helps by constraining the model.

5. What penalised regression does

Penalised regression adds a penalty to the model-fitting process.

The model is asked to:

predict the outcome well
and
avoid excessive model complexity

This helps control overfitting.

Common penalised methods include:

  • ridge regression;
  • lasso regression;
  • elastic net.

Each method handles high-dimensional data differently.

6. Ridge regression in high-dimensional data

Ridge regression shrinks all coefficients towards zero.

It is useful when:

  • many predictors may each carry small amounts of information;
  • predictors are correlated;
  • stability is important;
  • removing predictors is not the main goal.

However, ridge usually keeps all predictors.

This means it may not produce a simple list of selected biomarkers.

7. Lasso regression in high-dimensional data

Lasso regression can shrink many coefficients to zero.

This makes it useful for feature selection.

For example, from:

20,000 gene expression features

lasso may select:

30 genes

This can be useful for building a smaller model.

However, selected features may be unstable, especially when predictors are correlated.

8. Elastic net in high-dimensional data

Elastic net combines ridge and lasso.

It can shrink coefficients and select predictors.

It is often useful when predictors are correlated.

This is common in omics because genes, CpG sites, proteins, and metabolites often move together in biological groups.

Elastic net may be more stable than lasso alone in these settings.

9. Comparing ridge, lasso, and elastic net

Method Main behaviour Useful when
Ridge Shrinks all coefficients Many correlated predictors may contribute
Lasso Shrinks some coefficients to zero A sparse predictor set is desired
Elastic net Combines ridge and lasso Predictors are numerous and correlated

There is no universally best method.

The best method depends on the prediction task, data structure, and validation results.

10. Tuning parameters

Penalised regression methods require tuning.

Important parameters include:

lambda: strength of penalty
alpha: ridge-lasso mixing in elastic net

Tuning should be done using training data or inner cross-validation.

The final test set should not be repeatedly used to choose tuning parameters.

11. Why nested validation may be needed

High-dimensional workflows often involve many choices:

  • feature filtering;
  • preprocessing;
  • penalty type;
  • lambda value;
  • alpha value;
  • model comparison;
  • threshold choice.

If the same validation data are used to make these choices and report final performance, performance may be optimistic.

Nested validation separates model selection from performance estimation.

Inner loop: tune model and select features
Outer loop: estimate model performance

This is especially useful when many tuning choices are explored.

12. Preprocessing leakage

High-dimensional data usually require preprocessing.

Examples include:

  • normalisation;
  • transformation;
  • filtering;
  • batch correction;
  • PCA;
  • feature scaling;
  • missing data handling.

If preprocessing is learned from the full dataset before validation, leakage can occur.

For example:

1. Perform PCA using all samples.
2. Cross-validate a model using the PCA components.

This is biased because the validation folds influenced the PCA components.

Preprocessing must happen inside the validation process.

13. Batch effects

Batch effects are technical differences unrelated to biology.

Examples include:

  • sequencing batch;
  • methylation array plate;
  • laboratory technician;
  • processing date;
  • scanner type;
  • hospital site.

If batch is associated with outcome, the model may learn batch differences instead of disease biology.

This can produce excellent internal performance but poor external performance.

14. High-dimensional prediction and external validation

External validation is especially important in high-dimensional prediction.

A model may perform well in one dataset because of:

  • cohort-specific patterns;
  • platform-specific measurements;
  • batch effects;
  • local preprocessing choices;
  • population differences.

External validation tests whether the model works in a different setting.

Without external validation, strong claims should be avoided.

15. Calibration in high-dimensional models

High-dimensional models should not be evaluated only using AUC.

They also require calibration assessment.

A model may rank patients well but produce risk estimates that are too high or too low.

For clinical prediction, calibration matters because predicted probabilities may guide decisions.

Assessment should include:

  • calibration plot;
  • calibration slope;
  • calibration intercept;
  • Brier score;
  • clinical usefulness where relevant.

16. Interpretation of selected features

Selected features from high-dimensional penalised regression should be interpreted cautiously.

They may represent:

  • true biological signal;
  • correlated proxies;
  • technical artefacts;
  • batch effects;
  • cohort-specific patterns;
  • random noise.

Prediction performance and biological mechanism are different questions.

A selected feature should be considered a candidate for further study, not definitive proof.

17. Medical example: methylation prediction

Suppose we want to predict exposure status using DNA methylation data.

The dataset contains:

300 participants
450,000 CpG sites

A penalised regression model may select a small number of CpG sites.

Before trusting the result, we should ask:

  • Was feature selection inside validation?
  • Were batch effects handled properly?
  • Was preprocessing done inside resampling?
  • Was calibration assessed?
  • Was external validation performed?
  • Are selected CpG sites biologically plausible?
  • Are the findings stable across resamples?

18. Medical example: cancer gene expression prediction

Suppose we predict cancer recurrence using gene expression data.

The dataset contains:

500 patients
20,000 genes
80 recurrence events

Ordinary regression is not appropriate.

Penalised regression may help.

However, the model must still be validated carefully.

A high internal AUC is not enough.

We need calibration, external validation, and cautious interpretation of selected genes.

19. Practical checklist

Before trusting a high-dimensional penalised model, ask:

  1. What is the exact prediction question?
  2. What is the prediction time point?
  3. How many patients are available?
  4. How many outcome events are available?
  5. How many candidate predictors are available?
  6. Was preprocessing done inside validation?
  7. Was feature selection done inside validation?
  8. Was tuning performed without touching the test set?
  9. Were batch effects assessed?
  10. Was calibration assessed?
  11. Was external validation performed?
  12. Are selected features stable?
  13. Are biological claims appropriately cautious?

20. Reporting high-dimensional penalised models

A clear report should describe:

  • sample size;
  • number of outcome events;
  • number of candidate predictors;
  • preprocessing workflow;
  • feature filtering;
  • batch handling;
  • penalised method used;
  • tuning method;
  • validation design;
  • discrimination;
  • calibration;
  • external validation if available;
  • selected features if relevant;
  • limitations of biological interpretation.

Transparent reporting is essential because high-dimensional models are easy to overstate.

21. Common mistakes

Common mistakes include:

  • using ordinary regression with too many predictors;
  • selecting features before validation;
  • preprocessing using the full dataset;
  • tuning on the test set;
  • ignoring batch effects;
  • reporting only AUC;
  • interpreting selected features as causal;
  • claiming clinical usefulness without external validation.

These mistakes can make a weak model look strong.

22. Summary

High-dimensional biomedical prediction is challenging because the number of predictors may be far larger than the number of patients.

Penalised regression methods help by shrinking coefficients and controlling model complexity.

Ridge shrinks all coefficients.

Lasso can select predictors.

Elastic net combines ridge and lasso.

However, these methods do not remove the need for careful validation, leakage prevention, calibration assessment, and cautious interpretation.

Key terms

Term Meaning
High-dimensional data Data with many predictors, often more predictors than observations
p » n Number of predictors much larger than number of observations
Penalised regression Regression with a penalty to control model complexity
Ridge regression Penalised method that shrinks coefficients
Lasso regression Penalised method that can select predictors
Elastic net Penalised method combining ridge and lasso
Lambda Tuning parameter controlling penalty strength
Alpha Tuning parameter controlling ridge-lasso balance
Nested validation Validation separating tuning from performance estimation
Batch effect Technical variation unrelated to biological signal
Preprocessing leakage Leakage caused by learning preprocessing from validation or test data
External validation Testing a model in independent data

Key takeaway

Penalised regression is essential for many high-dimensional biomedical prediction problems, but high performance is only credible when tuning, preprocessing, feature selection, validation, and calibration are handled without leakage.
