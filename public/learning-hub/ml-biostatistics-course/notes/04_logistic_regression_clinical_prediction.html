Logistic Regression for Clinical Prediction

Learning objectives

By the end of this note, you should be able to:

  1. Explain why logistic regression is one of the most important baseline models in medical machine learning.
  2. Understand the difference between linear probability modelling and logistic regression.
  3. Interpret probabilities, odds, log-odds, odds ratios, and predicted risks.
  4. Fit logistic regression conceptually as a binary clinical prediction model.
  5. Explain why logistic regression can be used for both inference and prediction.
  6. Understand how logistic regression connects to clinical risk prediction.
  7. Recognise common modelling issues such as overfitting, separation, poor calibration, and inappropriate predictor selection.
  8. Explain why logistic regression should often be compared with more complex machine learning models.

1. Introduction

Logistic regression is one of the most important models in medical statistics, epidemiology, and clinical prediction modelling. Although it is often taught as a classical statistical method, it is also a supervised learning method when used to predict a binary outcome.

Many medical outcomes are binary. A patient may develop diabetes or not. A patient may be readmitted within 30 days or not. A tumour may recur within 5 years or not. A patient may respond to treatment or not. A participant may be classified as a case or control. These are all examples where logistic regression can be used.

In machine learning courses, logistic regression is sometimes treated as a simple baseline model before moving to random forests, support vector machines, boosting, or neural networks. In biostatistics, however, logistic regression deserves much deeper attention. It is interpretable, clinically familiar, statistically well understood, and often performs surprisingly well for structured tabular medical data. In many clinical prediction problems, a well-specified logistic regression model can be competitive with more complex machine learning methods, especially when sample sizes are modest and predictors are mostly structured clinical variables (Steyerberg, 2019; James et al., 2023).

This is why logistic regression is a good first practical model for this course. It connects classical medical statistics with machine learning and provides a foundation for later topics such as penalised regression, calibration, decision curve analysis, and clinical prediction model reporting.

2. Where logistic regression fits in supervised learning

Logistic regression is used when the outcome is binary. In supervised learning notation, we usually write:

Y = f(X) + error

For logistic regression:

Y = 0 or 1

where:

  • Y = 1 means the event occurs;
  • Y = 0 means the event does not occur.

Examples:

Clinical question Outcome coded as 1 Outcome coded as 0
Diabetes prediction Developed diabetes Did not develop diabetes
Readmission prediction Readmitted within 30 days Not readmitted
Cancer recurrence prediction Recurrence within 5 years No recurrence
Treatment response Responded Did not respond
Disease classification Case Control

The model uses predictors such as age, sex, BMI, blood pressure, biomarkers, tumour features, or methylation markers to estimate the probability that Y = 1.

This estimated probability is usually written as:

P(Y = 1 | X)

This means:

The probability that the event occurs, given the predictors.

For example:

P(readmission within 30 days | age, diagnosis, comorbidities, lab values) = 0.35

This would mean the model estimates a 35% probability of 30-day readmission for that patient.

3. Why not use ordinary linear regression?

Before understanding logistic regression, it helps to understand why ordinary linear regression is not ideal for binary outcomes.

Suppose we try to predict disease status using a linear regression model:

Disease = β0 + β1(age) + β2(BMI) + error

The problem is that linear regression can produce predicted values below 0 or above 1. But probabilities must lie between 0 and 1.

For example, a linear model might predict:

Predicted probability = 1.18

or:

Predicted probability = -0.12

These values are not meaningful probabilities.

A second issue is that the relationship between predictors and risk is often not linear on the probability scale. A small increase in a risk factor may have little effect at very low baseline risk, but a larger effect around intermediate risk. Logistic regression handles this by modelling the log-odds of the outcome rather than the probability directly.

4. Probability, odds, and log-odds

Logistic regression is based on three related ideas:

  1. probability;
  2. odds;
  3. log-odds.

Probability

A probability is the chance that an event occurs.

If the probability of disease is 0.20, this means:

Probability = 0.20 = 20%

In a group of 100 similar patients, approximately 20 would be expected to develop the outcome if the model is well calibrated.

Odds

Odds are another way of expressing risk.

Odds = probability / (1 - probability)

If the probability is 0.20:

Odds = 0.20 / 0.80 = 0.25

This means the odds are 0.25 to 1.

If the probability is 0.50:

Odds = 0.50 / 0.50 = 1

If the probability is 0.80:

Odds = 0.80 / 0.20 = 4

So probability and odds are related, but they are not the same.

Log-odds

Logistic regression models the log of the odds.

log-odds = log(p / (1 - p))

This is also called the logit transformation.

logit(p) = log(p / (1 - p))

The logit transformation maps probabilities between 0 and 1 onto the whole real number line. This makes it possible to use a linear model for the transformed outcome.

5. The logistic regression model

The logistic regression model can be written as:

logit(p) = β0 + β1X1 + β2X2 + ... + βpXp

where:

  • p is the probability of the outcome;
  • β0 is the intercept;
  • β1, β2, ..., βp are regression coefficients;
  • X1, X2, ..., Xp are predictors.

For example, a simple logistic regression model for diabetes might be:

logit(p) = β0 + β1(age) + β2(BMI) + β3(HbA1c)

This model estimates the probability of diabetes using age, BMI, and HbA1c.

The important point is that the model is linear on the log-odds scale, not on the probability scale.

After estimating the log-odds, we can convert back to probability using:

p = exp(logit) / (1 + exp(logit))

This ensures that predicted probabilities always lie between 0 and 1.

6. Interpreting coefficients

In logistic regression, coefficients are interpreted on the log-odds scale. This can feel unintuitive at first.

If a coefficient is positive, higher values of the predictor are associated with higher odds of the outcome.

If a coefficient is negative, higher values of the predictor are associated with lower odds of the outcome.

For example:

β_age = 0.04

This means that each one-unit increase in age is associated with an increase of 0.04 in the log-odds of the outcome, holding other predictors constant.

Because log-odds are difficult to interpret directly, coefficients are often exponentiated to obtain odds ratios.

Odds ratio = exp(β)

If:

β_age = 0.04

then:

exp(0.04) = 1.04

This means each one-year increase in age is associated with approximately 4% higher odds of the outcome, assuming other predictors are held constant.

7. Odds ratios: useful but easy to misunderstand

Odds ratios are common in medical statistics and epidemiology. However, they are often misunderstood.

An odds ratio is not the same as a risk ratio, especially when the outcome is common.

For rare outcomes, odds ratios and risk ratios may be similar. For common outcomes, odds ratios can exaggerate the impression of effect size compared with risk ratios.

For example, if an odds ratio is 2.0, this does not always mean the risk has doubled. It means the odds have doubled.

This distinction matters in inference. In prediction, odds ratios may be less central because the main goal is not necessarily to interpret each coefficient. Instead, we often care about predicted risks, discrimination, calibration, and clinical usefulness.

Still, interpretability is one reason logistic regression remains attractive in clinical prediction. Clinicians and researchers often understand odds ratios more easily than the internal structure of a random forest or gradient boosting model.

8. Logistic regression for inference vs prediction

Logistic regression can be used for two different purposes:

  1. inference;
  2. prediction.

This is important because the same model form can be used differently depending on the research aim.

Logistic regression for inference

In inference, we may ask:

Is BMI associated with diabetes after adjustment for age, sex, and family history?

Here, the focus is on the coefficient for BMI, its odds ratio, confidence interval, p-value, and whether confounding has been addressed.

The model may be written as:

diabetes ~ BMI + age + sex + family history

The main interest is the association between BMI and diabetes.

Logistic regression for prediction

In prediction, we may ask:

Can we predict a person’s probability of developing diabetes within 10 years?

Here, the focus is not one coefficient. The focus is the model’s predictive performance.

We care about:

  • predicted probabilities;
  • discrimination;
  • calibration;
  • sensitivity and specificity;
  • clinical usefulness;
  • validation on new data.

The same logistic regression model may be used, but the evaluation and interpretation are different.

This distinction is central to clinical prediction modelling (Steyerberg, 2019; Collins et al., 2024).

9. Example from your own study interests: breast cancer prediction

Your previous breast cancer survival dashboard focused on time-to-event methods such as Kaplan-Meier curves, log-rank tests, Cox proportional hazards models, proportional hazards diagnostics, penalised Cox models, random survival forests, and calibration.

That is mainly a survival modelling framework. However, logistic regression can still be relevant if the research question is converted into a fixed-time binary prediction problem.

For example:

Outcome:
Death within 5 years after diagnosis: yes/no

or:

Outcome:
Recurrence within 5 years: yes/no

Predictors might include:

  • age at diagnosis;
  • tumour stage;
  • tumour grade;
  • lymph node status;
  • receptor status;
  • treatment group;
  • baseline biomarker values.

A logistic regression model could estimate:

P(5-year recurrence = yes | baseline clinical and tumour characteristics)

This would provide a predicted probability of recurrence within 5 years.

However, there is an important limitation. If patients have different follow-up times or censoring is present, converting survival data into a binary outcome may discard information or introduce bias. In that case, Cox models or survival machine learning methods may be more appropriate.

This example shows an important biostatistical principle:

The model should match the outcome structure.

If the outcome is genuinely time-to-event with censoring, survival methods are usually more appropriate. If the outcome is a clearly observed binary outcome at a fixed time point, logistic regression may be appropriate.

10. Example from your own study interests: EWAS and methylation

Your current research interests include epidemiology, high-dimensional biomedical data, and DNA methylation. Logistic regression can appear in this area in several ways.

Suppose we have DNA methylation data and want to classify disease status:

Outcome:
Disease status: case/control

Predictors might include:

CpG methylation values

A logistic regression model could be used to predict disease status from methylation markers.

However, methylation data are high-dimensional. A dataset may contain hundreds of thousands of CpG sites and far fewer participants.

Example:

n = 800 participants
p = 450,000 CpG sites

A standard logistic regression model cannot be fitted safely with so many predictors. It would be extremely prone to overfitting and may not even be statistically estimable.

In this setting, penalised logistic regression methods such as lasso or elastic net are more appropriate because they shrink coefficients and can perform variable selection. Even then, feature selection must be done inside the resampling process to avoid data leakage (Hastie et al., 2009; James et al., 2023).

This connects directly with your EWAS-style work. In an EWAS, the goal may be association testing CpG by CpG. In a prediction problem, the goal is different: we want to predict disease, exposure, or phenotype status in new individuals. The validation strategy must match that prediction goal.

11. Example: predicting exposure status in pregnancy research

Suppose a study investigates whether maternal exposures during pregnancy are related to offspring DNA methylation. In an association-focused EWAS, the question might be:

Which CpG sites are associated with maternal exposure?

That is primarily an inference or association question.

A prediction version of the question might be:

Can DNA methylation patterns predict whether an offspring was exposed to a particular maternal exposure during pregnancy?

Here, the binary outcome could be:

Exposure status: exposed vs unexposed

Predictors might be methylation values at selected CpG sites.

A logistic regression model could be considered if the number of predictors is small. If the number of CpGs is large, penalised logistic regression or another high-dimensional method would be needed.

This example is useful because it shows that the same biological setting can lead to different modelling goals:

Question Goal Possible method
Which CpGs are associated with exposure? Association/inference EWAS regression models
Can CpGs predict exposure status? Prediction Penalised logistic regression, random forest, boosting
Does exposure causally affect methylation? Causal/epidemiological DAG-informed regression, sensitivity analyses

This is why it is important to define the question before choosing the model.

12. Choosing predictors in logistic regression

In clinical prediction, predictor choice should be guided by the intended use of the model.

A predictor should generally be:

  • available at the prediction time point;
  • measured reliably;
  • clinically meaningful or operationally available;
  • not a consequence of the outcome;
  • not a leakage variable;
  • feasible to collect in the intended setting.

For example, if predicting 30-day readmission at discharge, discharge medications may be available and appropriate. But if predicting risk at admission, discharge medications are future information and should not be used.

Similarly, if predicting mortality at cancer diagnosis, recurrence status during follow-up should not be included because it occurs after baseline and is closely tied to the outcome.

A model can look very accurate if it includes future information, but such a model is not useful in real practice.

13. Events per predictor and sample size

Logistic regression can overfit when there are too many predictors relative to the number of outcome events.

In older teaching, a common rule of thumb was at least 10 events per predictor. However, modern sample size thinking for prediction models is more nuanced and depends on outcome prevalence, expected model performance, number of predictors, and acceptable optimism (Riley et al., 2019; Steyerberg, 2019).

The basic principle remains:

A model with many predictors and few outcome events is at high risk of overfitting.

For example:

Total patients = 200
Outcome events = 20
Candidate predictors = 30

This is a risky setting for ordinary logistic regression. The model may fit the development data too closely and perform poorly in new patients.

Possible solutions include:

  • reducing the number of predictors;
  • using penalised logistic regression;
  • increasing sample size;
  • using clinical knowledge to pre-specify predictors;
  • applying internal validation;
  • avoiding data-driven variable selection without validation.

14. Separation and unstable estimates

A special problem in logistic regression is called separation.

Separation occurs when a predictor or combination of predictors perfectly separates outcome groups.

For example, suppose all patients with a particular biomarker value develop the outcome, and no patients without that value develop the outcome. The model may try to estimate an infinitely large coefficient.

This can lead to:

  • very large standard errors;
  • unstable coefficients;
  • convergence warnings;
  • extreme predicted probabilities;
  • poor generalisation.

Separation is more likely when the dataset is small or the outcome is rare.

Possible solutions include penalised logistic regression, Firth correction, reducing model complexity, or collecting more data.

15. Logistic regression and calibration

One reason logistic regression is popular in clinical prediction is that it often produces reasonably calibrated probabilities when the model is well specified and validated.

However, logistic regression is not automatically well calibrated.

Poor calibration can occur if:

  • important predictors are missing;
  • relationships are nonlinear but modelled as linear;
  • interactions are ignored;
  • the model is overfitted;
  • the model is applied to a different population;
  • outcome prevalence differs between development and validation settings.

Calibration should be assessed using calibration plots, calibration-in-the-large, calibration slope, and the Brier score where appropriate (Steyerberg, 2019).

For example, if a logistic regression model predicts 30% risk for a group of patients, but only 10% experience the outcome, the model is overestimating risk. This could lead to unnecessary treatment or monitoring.

16. Logistic regression and discrimination

Discrimination measures how well the model separates patients who experience the outcome from those who do not.

For binary outcomes, discrimination is commonly assessed using the ROC curve and AUC.

If a logistic regression model has:

AUC = 0.82

this suggests that the model has good ability to rank patients by risk.

However, AUC alone is not enough. The model may discriminate well but still be poorly calibrated.

For clinical prediction, both discrimination and calibration should be reported.

17. Model specification: linearity and interactions

Standard logistic regression assumes that continuous predictors have a linear relationship with the log-odds of the outcome.

This may not always be realistic.

For example, the relationship between age and risk may be nonlinear. Risk may increase slowly at younger ages and more rapidly at older ages. Similarly, BMI may have a nonlinear relationship with diabetes risk.

Possible ways to handle nonlinear effects include:

  • transforming variables;
  • using splines;
  • categorising variables carefully, though this often loses information;
  • using more flexible models;
  • including clinically justified interaction terms.

Interactions may also matter.

For example, the effect of a biomarker may differ by sex, treatment group, or disease stage. However, adding many interactions increases model complexity and can lead to overfitting if the sample size is limited.

18. Variable selection: be careful

Many beginners fit a logistic regression model by adding many variables and then using p-values to remove predictors step by step.

This can be problematic in prediction modelling.

Stepwise variable selection can lead to:

  • unstable predictor sets;
  • biased coefficient estimates;
  • overfitting;
  • optimistic performance estimates;
  • poor reproducibility;
  • exaggerated importance of selected predictors.

For prediction, it is often better to:

  • pre-specify clinically important predictors;
  • use penalisation;
  • validate the full modelling process;
  • report how predictors were selected;
  • avoid selecting variables using the test set.

In high-dimensional settings such as methylation or gene expression, variable selection must be done inside cross-validation or another resampling framework to avoid leakage.

19. Logistic regression vs machine learning models

More complex machine learning models may sometimes outperform logistic regression, especially when there are nonlinear relationships or complex interactions.

Examples include:

  • random forests;
  • gradient boosting;
  • support vector machines;
  • neural networks.

However, complex models are not automatically better.

In medical data, logistic regression may perform similarly to more complex models when:

  • the dataset is small or moderate in size;
  • the predictors are structured clinical variables;
  • relationships are approximately linear on the log-odds scale;
  • strong clinical predictors are already known;
  • the outcome is not extremely complex.

Therefore, logistic regression should often be treated as a serious baseline model.

A useful principle is:

Do not use a more complex model unless it improves clinically meaningful performance after proper validation.

20. Reporting a logistic regression prediction model

A logistic regression prediction model should be reported clearly enough that another researcher could understand and reproduce the modelling process.

A report should include:

  • clinical prediction question;
  • target population;
  • prediction time point;
  • outcome definition;
  • candidate predictors;
  • missing data handling;
  • sample size and number of events;
  • model specification;
  • predictor coding;
  • nonlinear terms or interactions;
  • validation strategy;
  • discrimination;
  • calibration;
  • clinical usefulness where relevant;
  • limitations;
  • model equation or sufficient details for implementation.

TRIPOD+AI emphasises transparent reporting of prediction model studies, including those using regression and machine learning approaches (Collins et al., 2024). PROBAST+AI provides a framework for judging risk of bias and applicability (Moons et al., 2025).

21. Practical workflow

A good first logistic regression workflow for clinical prediction might look like this:

  1. Define the clinical prediction question.
  2. Define the target population.
  3. Define the prediction time point.
  4. Define the binary outcome clearly.
  5. Identify candidate predictors available at prediction time.
  6. Explore missing data and data quality.
  7. Split data into training and test sets or define resampling strategy.
  8. Fit logistic regression as a baseline model.
  9. Generate predicted probabilities.
  10. Evaluate discrimination using ROC AUC.
  11. Evaluate calibration using calibration plots.
  12. Examine sensitivity and specificity at clinically relevant thresholds.
  13. Compare with more complex models only after the baseline is understood.
  14. Report the model transparently.

This workflow will be implemented in R in the practical part of the course.

22. Summary

Logistic regression is a foundational model for binary clinical prediction. It estimates the probability of an outcome using a linear model on the log-odds scale.

In medical statistics, logistic regression is important because it is interpretable, familiar, relatively stable, and often competitive for tabular clinical data. It can be used for both inference and prediction, but the interpretation and evaluation differ depending on the goal.

For prediction, logistic regression should be evaluated using discrimination, calibration, sensitivity, specificity, predictive values, and validation on new data. Accuracy alone is not enough.

Logistic regression is also a bridge to more advanced methods. Penalised logistic regression extends it to settings with many predictors, and its limitations help motivate tree-based models, boosting, and survival machine learning.

The main message is:

Logistic regression is not just an old statistical method. In biostatistics, it is a strong, interpretable, and clinically meaningful baseline for supervised machine learning.
