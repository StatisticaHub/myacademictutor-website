Lasso Regression

Learning objectives

By the end of this note, you should be able to:

  1. Explain what lasso regression does.
  2. Describe how lasso performs feature selection.
  3. Explain how lasso differs from ridge regression.
  4. Understand the role of the tuning parameter lambda.
  5. Explain why lasso-selected predictors should not automatically be interpreted as causal.
  6. Recognise why lasso can be unstable when predictors are correlated.
  7. Explain why lasso feature selection must happen inside validation.
  8. Describe how lasso can be used in biomedical prediction.

1. Why this note matters

Medical and biomedical datasets often contain many candidate predictors.

Some predictors may be useful. Others may be weak, redundant, noisy, unavailable in practice, or not clinically meaningful.

Lasso regression is attractive because it can shrink some coefficients exactly to zero.

This means lasso can produce a smaller prediction model.

The practical message of this note is:

Lasso regression can perform feature selection, but selected features must be validated and interpreted cautiously.

2. What lasso regression does

Lasso regression is a regularisation method.

It adds a penalty to the model-fitting process.

This penalty encourages some coefficients to become exactly zero.

If a predictor has a coefficient of zero, it is effectively removed from the model.

This is why lasso is often used for feature selection.

3. Lasso and coefficient shrinkage

Like ridge regression, lasso shrinks coefficients.

However, lasso can shrink some coefficients all the way to zero.

For example, suppose ordinary logistic regression gives:

glucose coefficient = 1.10
BMI coefficient = 0.70
pressure coefficient = 0.15
triceps coefficient = 0.05

Lasso may produce:

glucose coefficient = 0.90
BMI coefficient = 0.50
pressure coefficient = 0
triceps coefficient = 0

In this example, pressure and triceps are removed from the model.

4. Why lasso is useful

Lasso can be useful when:

  • there are many candidate predictors;
  • only some predictors are expected to be useful;
  • a simpler model is desired;
  • the clinical measurement burden matters;
  • the model needs to be easier to explain;
  • the dataset is too small for ordinary regression with all predictors.

In medical prediction, a smaller model may be easier to implement.

For example, a clinical tool requiring five predictors may be more practical than a tool requiring fifty.

5. Lasso as feature selection

Feature selection means choosing a subset of predictors.

Lasso performs feature selection automatically by shrinking some coefficients to zero.

This can help create a sparse model.

A sparse model uses only a subset of available predictors.

However, sparse does not automatically mean better.

A smaller model still needs validation.

6. The tuning parameter lambda

Lasso uses a tuning parameter called lambda.

Lambda controls how much shrinkage is applied.

Small lambda  → less shrinkage, more predictors retained
Large lambda  → more shrinkage, fewer predictors retained

Different lambda values can lead to different selected predictors.

Cross-validation is commonly used to choose lambda.

7. Choosing lambda carefully

Lambda should be chosen using training data or internal validation.

The final test set should not be used repeatedly to choose lambda.

An honest workflow is:

1. Split data into training and test sets.
2. Use cross-validation within the training set to choose lambda.
3. Fit the lasso model using the chosen lambda.
4. Evaluate once on the untouched test set.

This keeps the test set honest.

8. Lasso versus ridge

Ridge and lasso both shrink coefficients, but they behave differently.

Method Main behaviour Feature selection?
Ridge Shrinks coefficients towards zero Usually no
Lasso Shrinks some coefficients exactly to zero Yes

Ridge is often useful when many predictors each contribute small information.

Lasso is often useful when we want a smaller predictor set.

9. Lasso and correlated predictors

Lasso can be unstable when predictors are strongly correlated.

For example, suppose two biomarkers measure a similar biological process.

Lasso may select biomarker A and remove biomarker B.

In another sample, it may select biomarker B and remove biomarker A.

This does not necessarily mean one is truly more important.

It may simply reflect instability due to correlation.

10. Stability of selected predictors

Because lasso selection can be unstable, it is useful to check whether selected predictors are stable across:

  • cross-validation folds;
  • bootstrap samples;
  • repeated analyses;
  • external datasets.

If selected predictors change dramatically across resamples, biological interpretation should be cautious.

The model may still predict reasonably well, but the selected feature list may not be reliable.

11. Prediction versus causation

Lasso is a prediction tool.

A selected predictor may be:

  • causally related to the outcome;
  • correlated with a causal variable;
  • a proxy for disease severity;
  • a technical artefact;
  • selected by chance;
  • specific to one dataset.

Therefore, lasso selection does not prove causality.

For causal interpretation, we need additional design, assumptions, and analysis.

12. Feature-selection leakage

Lasso can easily cause data leakage if used incorrectly.

An incorrect workflow is:

1. Run lasso on the full dataset.
2. Keep the selected predictors.
3. Perform cross-validation using only those predictors.
4. Report cross-validated performance.

This is biased because the validation folds helped select predictors.

The performance estimate may be too optimistic.

13. Correct lasso validation workflow

A correct cross-validation workflow is:

For each fold:
    1. Use only the training fold.
    2. Select lambda using only the training fold.
    3. Fit the lasso model using only the training fold.
    4. Predict the validation fold.
    5. Store validation performance.

Feature selection must happen inside the validation process.

This is especially important in biomedical data with many predictors.

14. Lasso in omics

Lasso is often used in omics research.

Examples include:

  • gene expression prediction;
  • methylation-based prediction;
  • proteomic biomarker panels;
  • metabolomic risk prediction.

For example:

300 participants
20,000 gene expression features

Lasso may select a smaller set of genes.

But selected genes must be validated.

They should not automatically be treated as biological discoveries.

15. Medical example: diabetes prediction

Suppose we predict diabetes using clinical variables.

Lasso may remove weak predictors and keep variables such as:

  • glucose;
  • BMI;
  • age;
  • pedigree.

This can produce a simpler model.

However, we still need to assess:

  • AUC;
  • calibration;
  • Brier score;
  • clinical usefulness;
  • external validation.

A lasso-selected model is not automatically clinically useful.

16. Medical example: cancer biomarkers

Suppose we predict cancer recurrence using 200 candidate biomarkers.

Lasso may select 12 biomarkers.

Before claiming these biomarkers are clinically important, ask:

  • Are they selected consistently?
  • Are they biologically plausible?
  • Are they measurable in routine practice?
  • Do they validate externally?
  • Could they reflect batch effects?
  • Do they improve calibration and clinical usefulness?

17. Limitations of lasso

Lasso has several limitations:

  • selected features may be unstable;
  • correlated predictors can cause arbitrary selection;
  • selected features are not automatically causal;
  • performance may be optimistic if validation is incorrect;
  • a sparse model may underfit if many predictors each have small effects.

Elastic net may be useful when correlated predictors are common.

18. Reporting lasso regression

A clear report should describe:

  • outcome definition;
  • candidate predictors;
  • preprocessing steps;
  • missing data handling;
  • lambda selection method;
  • whether feature selection was inside validation;
  • selected predictors;
  • discrimination;
  • calibration;
  • external validation if available;
  • limitations of selected feature interpretation.

19. Summary

Lasso regression is a regularisation method that can perform feature selection.

It creates sparse models by shrinking some coefficients to zero.

This can be useful in medical prediction when we want a smaller model.

However, selected predictors must be validated and interpreted carefully.

Lasso is a prediction method, not automatic causal discovery.

Feature-selection leakage Leakage caused by selecting predictors before validation

Key takeaway

Lasso regression can create a smaller prediction model by shrinking some coefficients to zero. However, lasso-selected features should be interpreted as prediction features first, not automatically as causal or clinically proven predictors.
