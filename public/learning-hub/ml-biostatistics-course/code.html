R Code

This section contains reproducible R scripts for the course.

The aim is not only to run models, but to understand the full medical prediction workflow:

  • defining the outcome and predictors;
  • preparing the data;
  • splitting data into training and test sets;
  • fitting prediction models;
  • evaluating discrimination and calibration;
  • interpreting results clinically;
  • understanding feature selection and regularisation;
  • producing reproducible figures and outputs.

Most scripts use the Pima Indians Diabetes dataset so that the examples connect naturally across the course.

How to use the scripts

You can run any script from the terminal using:

Rscript path/to/script.R

For example:

Rscript r/06_cross_validation.R

or:

Rscript r/15_penalised_regression_high_dimensional_data.R

or:

Rscript r/C1_diabetes_risk_prediction_case_study.R

You can also open the scripts in RStudio and run them section by section.

Required R packages

Install the required packages once before running the scripts:

install.packages(c(
  "tidyverse",
  "mlbench",
  "pROC",
  "broom",
  "glmnet"
))

These packages are used for:

Package Purpose
tidyverse Data manipulation and plotting
mlbench Example datasets
pROC ROC curves and AUC
broom Tidy model summaries
glmnet Ridge, lasso, and elastic net models

What each script teaches

01_setup.R

Introduces the basic R setup for the course.

Use this script to check that R is working and that required packages can be loaded.

02_validation_overfitting_leakage.R

Shows why training performance can be misleading.

This script compares training and test performance, then demonstrates how an artificial leakage variable can make a model look unrealistically strong.

Main ideas:

  • training performance;
  • test performance;
  • overfitting;
  • artificial data leakage;
  • why predictors must be available at the prediction time.

03_supervised_learning_basics.R

Introduces the basic supervised learning workflow.

Main ideas:

  • outcome and predictors;
  • binary classification;
  • train/test split;
  • logistic regression as a first model;
  • predicted probabilities;
  • ROC curve and AUC.

04_logistic_regression_clinical_prediction.R

Uses logistic regression as a clinical prediction model.

Main ideas:

  • logistic regression for binary outcomes;
  • predicted probabilities;
  • odds ratios;
  • clinical interpretation;
  • test-set performance;
  • why statistical significance is not the same as prediction performance.

05_classification_metrics_in_medicine.R

Shows why accuracy alone is not enough in medical prediction.

Main ideas:

Compares ordinary logistic regression with regularised models.

11_ridge_regression.R

Focuses specifically on ridge regression.

This script compares ordinary logistic regression with ridge regression for diabetes prediction.

Main ideas:

  • coefficient shrinkage;
  • ridge penalty;
  • alpha = 0;
  • lambda.min;
  • ordinary logistic regression versus ridge regression;
  • why ridge usually keeps predictors;
  • why ridge may help with correlated predictors.

12_lasso_regression.R

Focuses specifically on lasso regression.

This script fits a lasso model and prints the predictors retained by the model.

Main ideas:

  • lasso penalty;
  • alpha = 1;
  • coefficient shrinkage;
  • coefficients shrunk to zero;
  • feature selection;
  • sparse prediction models;
  • why selected predictors are not automatically causal.

13_elastic_net.R

Focuses on elastic net regression.

This script fits elastic net models using different alpha values and compares their test AUC.

Main ideas:

  • elastic net;
  • alpha as the ridge-lasso mixing parameter;
  • alpha = 0 as ridge;
  • alpha = 1 as lasso;
  • intermediate alpha values;
  • comparing alpha values;
  • selecting features from the best elastic net model.

14_feature_selection_biomedical_data.R

Demonstrates feature selection in biomedical data.

This script compares a leakage-prone feature selection workflow with a more honest workflow where feature selection is performed using training data only.

Main ideas:

  • feature ranking;
  • univariable AUC;
  • feature selection;
  • data leakage;
  • training-only feature selection;
  • why feature selection must be part of validation;
  • selected features as prediction features, not causal findings.

15_penalised_regression_high_dimensional_data.R

Simulates a high-dimensional biomedical prediction problem.

This script creates a dataset with many more predictors than true signal features and compares ridge, lasso, and elastic net.

Main ideas:

  • high-dimensional data;
  • p much larger than n;
  • simulated biomedical predictors;
  • ridge regression;
  • lasso regression;
  • elastic net;
  • selected features;
  • why high-dimensional models need careful validation;
  • why selected features are not automatically biologically confirmed.

Reproducibility note

The scripts are written to support reproducible learning.

Where appropriate, scripts should:

  • set a random seed;
  • clearly define the outcome;
  • clearly define the positive class;
  • clearly define predictors;
  • avoid data leakage;
  • separate training and test data;
  • keep validation honest;
  • keep feature selection inside validation where appropriate;
  • avoid using the final test set repeatedly for tuning;
  • save figures or outputs to the correct folders where relevant;
  • include comments explaining the clinical interpretation.

The goal is not only to run R code, but to understand how each modelling step supports reliable medical prediction.
