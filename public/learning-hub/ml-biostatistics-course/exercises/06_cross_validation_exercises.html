Exercise 6: Cross-Validation

Learning aim

This exercise helps you understand why cross-validation is used in medical machine learning and how it differs from a single train/test split.

By the end of this exercise, you should be able to explain:

  • why a single train/test split can be unstable;
  • how k-fold cross-validation works;
  • why preprocessing must happen inside the validation process;
  • why cross-validation is internal validation, not external validation.

Question 1: Concept check

Explain in your own words why a single train/test split may give an unstable estimate of model performance.

Show suggested answer A single train/test split depends on which patients happen to be placed in the training set and which patients happen to be placed in the test set. If the test set is unusually easy, performance may look too good. If the test set is unusually difficult, performance may look too poor. This is especially important when the dataset is small or the outcome is uncommon. Cross-validation reduces this problem by evaluating the model across multiple splits.

Question 2: Medical interpretation

You develop a diabetes prediction model.

One train/test split gives:

AUC = 0.82

Another train/test split gives:

AUC = 0.74

What does this suggest?

Show suggested answer This suggests that the performance estimate may be sensitive to the particular split of the data. The difference could be due to: - small sample size; - different outcome distributions in the test sets; - different patient characteristics in each split; - model instability. Cross-validation may provide a more stable internal estimate of performance by evaluating the model across several splits.

Question 3: True or false

Feature selection can be performed on the full dataset before cross-validation because cross-validation will still provide an unbiased performance estimate.

Show suggested answer False. If feature selection is performed on the full dataset before cross-validation, information from the validation folds has already influenced the modelling process. This creates data leakage. Feature selection should happen inside each training fold, so that the validation fold remains genuinely unseen.

Question 4: Applied reasoning

Suppose you are using 5-fold cross-validation.

How many times is each patient used for validation?

Show suggested answer Each patient is used for validation once. In 5-fold cross-validation: 1. the data are divided into five folds; 2. the model is trained on four folds; 3. the remaining fold is used for validation; 4. the process repeats until each fold has served as the validation fold.

Question 5: Short answer

Why does cross-validation not replace external validation?

Show suggested answer Cross-validation is a form of internal validation. It estimates model performance within the same dataset or source population. External validation tests the model in a different dataset, such as another hospital, time period, country, or patient population. A model may perform well during cross-validation but still perform poorly in a new clinical setting.

Question 6: Workflow problem

A researcher follows this workflow:

1. Use the full dataset to select the top 10 predictors.
2. Perform 10-fold cross-validation using only those 10 predictors.
3. Report the average cross-validated AUC.

What is wrong with this workflow?

Show suggested answer The problem is that predictor selection was performed before cross-validation using the full dataset. This means the validation folds influenced which predictors were selected. As a result, the validation folds are no longer truly independent, and the cross-validated AUC may be too optimistic. A better approach would be to repeat predictor selection inside each training fold.

Question 7: Clinical reasoning

A model developed for hospital readmission has good cross-validated performance.

Before using it in a new hospital, what additional evaluation would you want?

Show suggested answer You would want external validation in data from the new hospital or a similar external setting. This is important because hospitals may differ in: - patient population; - disease prevalence; - coding systems; - admission and discharge practices; - available predictors; - follow-up procedures. Good internal validation does not guarantee good external performance.

Reflection

In a medical prediction study, what could go wrong if the cross-validation process is not carefully designed?

Show reflection answer Poorly designed cross-validation can lead to overly optimistic results. Problems may include: - preprocessing before splitting; - feature selection before cross-validation; - repeated measurements from the same patient appearing in both training and validation folds; - outcome information leaking into predictor construction; - tuning the model repeatedly until the best result is found. In medical machine learning, this can make a model appear useful when it may fail on real patients.
