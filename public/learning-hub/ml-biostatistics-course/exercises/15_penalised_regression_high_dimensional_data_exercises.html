Suggested answers

Answer 1

High-dimensional data contain many predictors, often far more predictors than observations. This is written as p >> n, where p is the number of predictors and n is the number of observations.

Answer 2

Examples include gene expression, DNA methylation, proteomics, metabolomics, microbiome data, radiomics, and large electronic health record feature sets.

Answer 3

Ordinary regression can fail because there are too many coefficients to estimate, leading to unstable estimates, overfitting, perfect separation, and poor generalisation.

Answer 4

Penalised regression adds a penalty to the model-fitting process to control model complexity and reduce overfitting.

Answer 5

Method Main behaviour Useful when
Ridge Shrinks all coefficients Many correlated predictors may each contribute
Lasso Shrinks some coefficients to zero A sparse selected feature set is desired
Elastic net Combines ridge and lasso Predictors are numerous and correlated

Answer 6

Situation Possible method

A sparse biomarker panel is desired Lasso

Feature stability is more important than sparsity Ridge or elastic net
A compromise between ridge and lasso is needed Elastic net

Answer 7

There are far more predictors than participants: 450,000 CpG sites and only 300 participants. This is a p >> n problem.

Answer 8

Ordinary logistic regression cannot safely estimate so many coefficients from so few observations. It would overfit or fail to estimate stable coefficients.

Answer 9

Lasso may be attractive because it can shrink many coefficients to zero and select a smaller set of CpG sites.

Answer 10

Elastic net may be preferable because neighbouring CpG sites are often correlated. Elastic net can handle correlated predictors more smoothly than pure lasso.

Answer 11

lambda controls the strength of the penalty.

Answer 12

alpha controls the balance between ridge-like and lasso-like behaviour.

Answer 13

The test set becomes part of model development, so the final test performance may be optimistic.

Answer 14

Nested cross-validation separates model tuning from performance estimation. The inner loop tunes the model, while the outer loop estimates performance.

Answer 15

It is useful because high-dimensional workflows involve many choices, such as feature selection, preprocessing, and tuning. Nested validation helps avoid optimistic performance estimates.

Answer 16

The validation folds influenced the PCA components because PCA was performed using the full dataset. This creates preprocessing leakage.

Answer 17

PCA or preprocessing should be learned inside each training fold and then applied to the validation fold.

Answer 18

Examples include PCA, normalisation, scaling, feature filtering, batch correction, imputation, and outcome-based feature selection.

Answer 19

A batch effect is technical variation unrelated to biology.

Answer 20

Examples include sequencing run, methylation plate, lab technician, processing date, scanner, hospital site, or assay platform.

Answer 21

With many features, the model may find technical patterns that are associated with outcome in the development data but do not generalise externally.

Answer 22

AUC measures discrimination but not calibration or clinical usefulness. A model may rank patients well but give inaccurate risk probabilities.

Answer 23

Additional assessments include calibration plot, calibration slope, calibration intercept, Brier score, decision curve analysis, external validation, and feature stability.

Answer 24

No. Selected genes may be causal, but they may also be proxies, correlated features, batch artefacts, or selected by chance. Prediction selection is not causal proof.

Answer 25

The AUC may be optimistic, especially if preprocessing, tuning, or feature selection were not properly nested. Calibration and external validation are missing.

Answer 26

They should report preprocessing details, feature selection procedure, tuning method, validation design, calibration, Brier score, selected features, feature stability, batch handling, and external validation if available.

Answer 27

A better plan:

1. Define prediction question and time point.
2. Split data or define outer cross-validation folds.
3. Within each training fold, perform preprocessing and feature filtering.
4. Tune elastic net using inner cross-validation.
5. Evaluate predictions in the outer validation fold.
6. Assess AUC, calibration, Brier score, and clinical usefulness.
7. Check feature stability.
8. Validate externally if possible.
9. Interpret selected genes cautiously.
