Part I: True or false

Statement True / False
Feature selection can reduce measurement burden.  
Feature selection always improves external validation performance.  
Feature selection should be done inside validation.  
Selected features are automatically causal.  
Batch effects can influence selected features.  
Feature stability is important for interpretation.  

Suggested answers

Answer 1

Feature selection is the process of choosing a subset of predictors from a larger set of candidate predictors for use in a model.

Answer 2

Feature selection may reduce model complexity, reduce measurement burden, improve practicality, remove noisy predictors, and make the model easier to communicate.

Answer 3

Feature selection can cause leakage, select unstable predictors, remove useful predictors, identify dataset-specific patterns, or encourage overinterpretation of selected features.

Answer 4

Clinical feature selection uses subject knowledge and clinical practicality. Statistical feature selection uses data-driven criteria such as p-values, correlations, or model-based importance.

Answer 5

Method Type
Selecting predictors based on prior clinical knowledge Clinical
Selecting predictors with lowest p-values Statistical
Selecting predictors retained by lasso Model-based/statistical
Selecting variables based on random forest importance Model-based
Selecting predictors because they are routinely available in hospital records Clinical/practical
Selecting top genes by association with outcome Statistical

Answer 6

Model-based feature selection identifies variables useful for prediction in a specific dataset and model. It does not prove that these variables cause the outcome.

Answer 7

The test set influenced feature selection because the full dataset was used before splitting. This creates leakage and may inflate test performance.

Answer 8

Correct workflow:

1. Split into training and test sets.
2. Select features using training data only.
3. Fit the model using training data only.
4. Evaluate once on the untouched test set.

Answer 9

Validation or test data have already influenced model development, so the evaluation is no longer independent. Random patterns from the validation/test data can be selected and make performance look better.

Answer 10

B is correct. Feature selection must happen inside each training fold so that the validation fold remains unseen.

Answer 11

Omics data contain many thousands of features. Some will appear related to the outcome by chance. If selected before validation, those chance signals can inflate performance.

Answer 12

A batch effect is technical variation unrelated to biology. Examples include sequencing run, methylation array plate, lab technician, scanner, processing date, or hospital site.

Answer 13

Feature selection may identify methylation sites that distinguish plates rather than disease status. Because plate is confounded with outcome, selected features may reflect technical artefacts.

Answer 14

The model may perform well internally but fail in external data where the batch structure is different.

Answer 15

Feature stability means selected features are consistent across resamples, folds, bootstrap samples, or external datasets.

Answer 16

It suggests the selected biomarker list is unstable. The model may not provide a reliable set of biomarkers for interpretation.

Answer 17

When predictors are correlated, several variables contain similar information. A selection method may choose one in one sample and another in a different sample.

Answer 18

No. A selected feature may be causal, correlated with a causal factor, a proxy, a batch artefact, or selected by chance.

Answer 19

No. A biologically important predictor may not improve prediction enough to be selected, or it may be redundant with other predictors.

Answer 20

The main problem is feature-selection leakage. The full dataset was used to select genes before cross-validation.

Answer 21

A corrected plan:

1. Define cross-validation folds.
2. Within each training fold, perform gene selection.
3. Fit the prediction model using selected genes in that training fold.
4. Predict the held-out validation fold.
5. Repeat across folds.
6. Estimate performance from held-out predictions.
7. Use external validation if possible.

Answer 22

They should ask:

  • Were genes selected inside validation?
  • Are selected genes stable?
  • Are they biologically plausible?
  • Could batch effects explain them?
  • Do they validate externally?
  • Are they measured reliably?
  • Do they improve calibration or only AUC?
  • Are claims limited to prediction rather than causation?
